## 用途

这不是网站默认前台文案，也不是直接恢复 8 大块长报告。  
这是一组为 **价值研究增强模式** 准备的专业输出提示词资产，后续可用于：

- `观象` 的价值分析增强解释
- `/research?render_profile=value` 的 LLM 增强版
- 内部离线价值研究生成

## 使用边界

必须坚持：

- Research-Only
- 不输出投资建议
- 不输出目标价
- 不输出合理估值结论
- 不输出买入/增持/减持
- 不把卖方评级当成系统结论

必须替换或删除的风险内容：

- “基金经理投资决策”
- “投资逻辑”
- “推荐/回避/谨慎”
- “Alpha 收益”
- “目标价/合理估值/预期收益空间”

## 0. 共享信源分级与数据清洗约束

源节点：`175309`

```md
# Shared Source Rules
1. 一级信源（核心）：
   - 政府、监管机构官网
   - 交易所与上市公司正式公告
2. 二级信源（高信度）：
   - 头部券商
   - 主流财经终端或公告聚合平台
3. 三级信源（一般建议）：
   - 官媒与严肃财经媒体
4. 废弃信源：
   - 股吧、论坛、自媒体、个人博客、营销号
   - 含“网友称”“据传”“爆料”等表述的信息
   - 域名或片段中出现 `guba`、`weibo`、`zhihu` 等噪音来源

# Shared Data Hygiene
1. 优先使用最新可得数据，但必须在文本中明确日期归属。
2. 多年份财务数据必须先完成口径清洗，避免把 SQL 杂质、空值、占位符带入正文。
3. 所有金额统一为“亿元”，保留 2 位小数；若原始数据无法可靠换算，则改做定性描述。
4. 若搜索信息与公告、正式财报冲突，以公告与财报为准。
5. 缺乏可靠来源时，宁可少写，不要补写主观推断。

# Shared Output Discipline
1. 所有模块默认 Research-Only。
2. 只允许输出事实、结构、变化与公开信息，不输出目标价、评级、买卖建议。
3. 若引用公开机构观点，必须明确它只是外部观点，不代表系统判断。
```

## 不进入主链的 Prompt

以下内容只做档案留存，不进入当前网站主链：

- `179899` 合理估值计算
- `113460` 估值分析报告

原因：

- 含目标价
- 含投资建议
- 含买入/增持/减持
- 含前瞻预测结论
