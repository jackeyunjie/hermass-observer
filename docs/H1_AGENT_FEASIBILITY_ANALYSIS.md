## 第一章：H1 Agent 定义

### 1.1 H1 Agent 的构成

H1 Agent 是一个完整的全局观察器，包含五周期状态：

```text
H1 Agent:
  MN1@H1_view  — 月线结构在 H1 视角下的状态
  W1@H1_view   — 周线结构在 H1 视角下的状态
  D1@H1_view   — 日线结构在 H1 视角下的状态
  H4@H1_view   — 4小时结构在 H1 视角下的状态
  H1@H1_view   — 小时结构在 H1 视角下的状态
```

**基准价规则**：所有 position 计算使用 H1 close。

### 1.2 H1 Agent 与 D1 Agent 的区别

| 维度 | D1 Agent | H1 Agent |
|------|----------|----------|
| 构成周期 | MN1/W1/D1 | MN1/W1/D1/H4/H1 |
| position 基准 | D1 close | H1 close |
| 更新频率 | 每日 | 每小时 |
| ef_count 范围 | 0-3 | 0-5 |
| 数据量（相对） | 1× | 24× |

### 1.3 H1 Agent 的更新频率

```text
A 股交易时间：09:30-11:30, 13:00-15:00

H1 bar 闭合时间点：
  10:30（第一小时）
  11:30（第二小时）
  14:00（第三小时，午后第一小时）
  15:00（第四小时，收盘）

每日 4 根 H1 bar，全年约 960 根（240 交易日 × 4）
```

## 第二章：A 股制度约束分析

### 2.1 T+1 制度的影响

**规则**：买入当日不可卖出，当日买入的股票最早次日才能卖出。

**对 H1 Agent 的影响**：

```text
场景：H1 Agent 在 10:30 发出买入信号
  → 可以买入 ✓
  → 但买入后当日无法卖出 ✗
  → 如果 11:30 H1 Agent 发出卖出信号
  → 无法执行，只能等到次日

结果：H1 的高频优势被 T+1 抵消
  - H1 可以捕捉更精细的入场时机
  - 但无法捕捉 H1 级别的出场时机
  - 出场必须等到次日 D1 收盘或更晚
```

**结论**：T+1 制度下，H1 Agent 的"双向高频交易"价值大幅降低。

### 2.2 涨跌停限制的影响

**规则**：A 股主板涨跌幅限制 10%，创业板/科创板 20%。

**对 H1 Agent 的影响**：

```text
场景 1：H1 Agent 检测到突破信号
  → 但价格已涨停
  → 无法买入
  → H1 信号不可执行

场景 2：H1 Agent 检测到止损信号
  → 但价格已跌停
  → 无法卖出
  → H1 止损无法执行

结果：H1 级别的突破/止损信号可能因涨跌停而不可执行
  - 信号频率越高，遇到涨跌停的概率越大
  - H1 信号的"可执行率"低于 D1 信号
```

### 2.3 交易成本的影响

| 成本项 | D1 级别 | H1 级别（估算） |
|--------|--------|----------------|
| 佣金 | 0.025% | 0.025%（不变） |
| 印花税 | 0.1%（卖出） | 0.1%（不变） |
| 滑点 | 0.05-0.1% | 0.1-0.2%（波动更大） |
| 交易次数/年 | 50-100 次 | 200-400 次（4×） |
| **年化交易成本** | **1-2%** | **4-8%（4×）** |

**结论**：H1 级别的频繁交易，年化成本是 D1 的 4 倍左右。

### 2.4 结论：A 股 T+1 制度下 H1 Agent 的价值评估

| 价值维度 | 评估 | 说明 |
|----------|------|------|
| 入场时机优化 | 有限 | H1 可以优化入场，但 T+1 限制出场 |
| 止损灵敏度 | 有限 | H1 可以更早发现风险，但无法当日止损 |
| 盘中监控 | 有价值 | H1 可以作为 D1 信号的"提前预警" |
| 交易频率 | 负面 | 成本上升，可执行率下降 |
| 信息噪音比 | 负面 | H1 级别假突破更多 |

**综合评估**：A 股 T+1 制度下，H1 Agent 作为"交易信号源"的价值有限；作为"盘中监控工具"有一定价值。

## 第三章：H1 Agent 的潜在价值场景

### 3.1 盘中监控（不做交易，只做观察）

**场景**：H1 State 变化作为 D1 信号的"提前预警"

```text
示例：
  09:30 开盘，D1 Agent 显示 D1@D1 = C（扩张趋势，未突破）
  10:30 H1 Agent 显示 D1@H1 = E（H1 视角下日线结构已突破）
  
  预警输出：
    "盘中监控：688010 福光股份
     H1 Agent 检测到 D1 结构突破（D1@H1 = E）
     D1 Agent 收盘确认概率：高
     建议：关注今日收盘是否确认 D1@D1 = E"
```

**价值**：信息增量，但不直接产生交易信号。

### 3.2 止损优化

**场景**：H1 跌破支撑位作为更灵敏的止损触发

```text
示例：
  持仓中，D1 Agent 显示 D1@D1 = E（健康）
  14:00 H1 Agent 显示 D1@H1 = -C（H1 视角下已跌破支撑）
  
  预警输出：
    "盘中预警：持仓股 688010 福光股份
     H1 Agent 检测到 D1 结构走弱（D1@H1 = -C）
     D1 Agent 尚未确认（D1@D1 = E）
     建议：若收盘确认 D1@D1 转负，次日考虑止损"
```

**价值**：可能降低 1-2% 的平均止损幅度（提前预警，次日早行动）。

### 3.3 入场时机优化

**场景**：D1 信号触发后，等 H1 也进入有利状态再执行

```text
示例：
  昨日 D1 Agent 显示 D1@D1 = E（信号触发）
  今日开盘 H1 Agent 显示 H1@H1 = -C（小时结构偏弱）
  
  决策：
    "D1 信号已触发，但 H1 结构偏弱
     建议：等待 H1@H1 转正后再执行入场
     避免'信号日追高'"
```

**价值**：可能提升 0.5-1% 的平均入场质量（避免追高）。

### 3.4 A 股系统的 H1 Agent 定位

**场景**：A 股 T+1 制度下的 H1 Agent 应用

| 市场 | 制度 | H1 Agent 价值 |
|------|------|--------------|
| A 股 | T+1 | 监控为主，交易价值有限 |

**结论**：H1 Agent 对 A 股价值有限，建议暂不实现。

## 第四章：成本-收益分析

### 4.1 实现成本

| 模块 | 工作量 | 说明 |
|------|--------|------|
| H1 数据采集 | 3-5 天 | 5 分钟线聚合为小时线，需处理盘中数据 |
| H1 SR 关键位计算 | 2-3 天 | H1 级别的分形计算，参数可能需调整 |
| H1 State 计算模块 | 3-5 天 | 五周期状态计算，复用 p116_core.py 逻辑 |
| H1 Agent 与 D1 Agent 集成 | 2-3 天 | 状态缓存联动、冲突处理 |
| 测试和验证 | 3-5 天 | 与 D1 Agent 的差异验证 |
| **总计** | **13-21 天（约 2-3 周）** | |

### 4.2 运行成本

| 成本项 | D1 Agent | H1 Agent | 倍数 |
|--------|----------|----------|------|
| 日数据量 | 5,000 只 × 1 bar | 5,000 只 × 4 bar | 4× |
| 存储（Foundation DB） | ~3.8 GB/日 | ~15 GB/日 | 4× |
| 计算时间 | ~10 分钟 | ~40 分钟 | 4× |
| 状态缓存更新 | 每日 1 次 | 每日 4 次 | 4× |

### 4.3 预期收益

| 收益场景 | 预期效果 | 置信度 |
|----------|----------|--------|
| 盘中监控预警 | 信息增量，不直接产生收益 | 高 |
| 止损优化（提前 1-2%） | 降低平均止损幅度 | 中 |
| 入场优化（提升 0.5-1%） | 提升平均入场质量 | 中 |
| 直接交易信号 | T+1 限制下不可行 | — |

### 4.4 ROI 评估

```text
投入：2-3 周开发 + 4× 运行成本
产出：信息增量 + 可能的 0.5-2% 质量提升

A 股 T+1 市场 ROI：低
  - 投入明确（2-3 周 + 4× 成本）
  - 产出有限（监控为主，无法直接交易）
  - 建议：暂不实现，用替代方案

其他市场 T+0 市场 ROI：高（但本系统不做）
  - 投入相同
  - 产出明确（H1 信号可立即执行）
  - 说明：本系统专注 A 股，T+0 市场不在范围内
```

## 第五章：结论与建议

### 5.1 推荐方案（含优先级）

**方案 A（推荐）：暂不实现 A 股 H1 Agent，采用替代方案**

```text
替代方案：D1 Agent + H1 辅助指标（非独立 Agent）
  - 在 D1 Agent 中增加 H1 参考字段：
    h1_trend_direction: H1 级别的趋势方向（不计算完整 State）
    h1_position_vs_d1_sr: H1 close 与 D1 SR 的关系
  - 不新建 H1 Agent，只做辅助判断
  - 开发成本：2-3 天
  - 运行成本：几乎无增加
```

**方案 B（次选）：仅实现 H1 盘中监控（不用于交易信号）**

```text
实现范围：
  - H1 数据采集和 State 计算
  - 仅输出监控预警，不接入策略信号链路
  - 不计算 H4/H1 的 SR（简化，只监控 D1/W1/MN1@H1_view）
  
开发成本：1-2 周
适用场景：盘中实时监控、D1 信号提前预警
```

**方案 C（已废弃）：MT5 平台的 H1 Agent**

```text
状态：已废弃
原因：本系统专注 A 股市场，不再涉及 MT5
说明：MT5 相关文档已归档，仅作历史参考
```

### 5.2 不推荐的方案及理由

| 方案 | 不推荐理由 |
|------|-----------|
| 完整 H1 Agent + A 股交易信号 | T+1 制度下无法执行 H1 出场信号，高频优势被抵消 |
| H1 Agent 替代 D1 Agent | D1 Agent 已覆盖 A 股主要交易场景，H1 信息噪音比更低 |
| M30/M15 Agent | 数据量 48-96×，噪音远超信息增益，A 股完全不需要 |

### 5.3 分阶段实施路径（如果决定做）

```text
Phase 1（1-2 周）：H1 辅助指标
  - 在 D1 Agent 中增加 H1 趋势方向参考
  - 不新建 Agent，最小成本验证价值

Phase 2（2-3 周）：H1 监控模式
  - 实现 H1 数据采集和简化 State 计算
  - 仅输出监控预警，不接入交易链路
  - 验证盘中监控的实际价值

Phase 3（待定）：A 股 H1 交易信号
  - 仅在 T+1 制度取消或变相 T+0（如 ETF）场景下考虑
```
