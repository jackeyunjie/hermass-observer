# 一、系统定位

## 这不是推荐系统，是环境识别系统

```
系统回答一个问题：
"当前市场环境，适合哪个策略？"
```

### 三不原则

| 原则 | 说明 |
|------|------|
| **不预测涨跌** | 系统只描述环境，不预测方向 |
| **不给买卖建议** | 所有输出都是观察，不是指令 |
| **不编造胜率** | 统计数据来自本地历史验证，标注样本量 |

# 二、核心链路

```
State 底座         策略信号          环境匹配           输出
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ MN1/W1/D1 │──→│ VCP      │──→│ 最佳适配   │──→  │ 每日总报  │
│ 三周期    │    │ 2560     │    │ 适配     │     │ 策略提醒  │
│ 4-bit编码 │    │ Bollinger│    │ 弱适配   │     │ 首席报告  │
│ E/F 状态  │    │          │    │ 待观察   │     │          │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
```

### 三条策略一句话

| 策略 | 核心逻辑 | 最佳环境 |
|------|----------|----------|
| **VCP** | 弹簧压缩后释放，捕捉趋势新生 | 收缩后刚脱离 |
| **2560** | 趋势歇脚时跟上，回踩均线确认 | 趋势行进 + 行业共振 |
| **Bollinger** | 趋势跑步时跟踪，波动突破加速 | 趋势延展 + 波动转折 |

# 七、敢用的数据验证

## VCP：路径假设通过

```
核心发现：收缩后释放路径 > 静态状态匹配

近 20 日经历收缩后释放：
  20d 平均超额：+1.67%
  样本量：43,259

对比 — 当前扩张但无收缩前兆：
  20d 平均超额：+0.45%
  → 路径组表现 3.7 倍于非路径组
```

## 2560：已固化规则

```
State 组合：E/E/F + E/F/F + E/F/E
市场匹配口径：

  full_match（个股+State+行业ETF共振）：62 例
  stock_only（个股+State，ETF缺失）：     8 例
  market_unsupported（ETF不支持）：      97 例
```

## Bollinger：vol=0 更稳

```
违反直觉的发现：

  D1 volatility_bit=0（波动稳定）：20d 超额 +0.59%
  D1 volatility_bit=1（波动活跃）：20d 超额 -0.49%

结论：Bollinger 的最佳环境不是"已经波动很大"
      而是"刚从稳定进入波动"的转折点
```

# 九、试用指引

## 每天 5 分钟

```
Step 1：打开 public/strategy_reminder_latest.html
        （策略提醒，完整信号清单）

Step 2：打开 public/daily_research_brief_latest.html
        （每日总报，环境匹配报告）

Step 3：浏览"最佳适配聚焦表"
        这是当日置信度最高的信号
```

## 关注什么

| 关注点      | 怎么看                               |
|--------    |--------                             |
| 最佳适配信号 | 优先浏览，这是环境最匹配的信号           |
| 环境标签     | 看"波动稳定"、"三周期共振"等标签理解背景  |
| 本地验证数字 | 有具体百分比的信号比"待校准"更有参考价值   |
| 适配度等级   | 最佳适配 > 适配 > 弱适配 > 待观察        |

## "待校准"是诚实，不是缺陷

```
系统不说"胜率 55%"来吸引眼球。
系统说"待校准"——因为还没有积累足够的前向观察样本来验证。

当样本积累到 3-6 个月，校准层会自动验证：
适配度排序是否真的与历史收益方向一致。

校准通过前，适配度基于逻辑匹配（策略理论 + State 环境）。
校准通过后，适配度基于逻辑 + 数据双重确认。
```
