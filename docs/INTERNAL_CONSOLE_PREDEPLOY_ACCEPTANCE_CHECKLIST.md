# Internal Console Predeploy Acceptance Checklist

目标：在部署到服务器前，用一份**产品视角**清单确认当前网站已经达到“内部可用、可演示、可继续迭代”的状态。

这份清单不关注 Git 提交范围，而关注：

- 首页是否足够低门槛
- 市场/行业/研究/执行/回测五条主路径是否通
- 多周期观察、概率与盈亏比、风险控制这条主线是否真正体现在页面里

---

## 1. 本轮验收目标

当前网站不是“策略信号大全”，而是：

- 多周期相互影响观察
- 单周期内周期运行观察
- 概率与盈亏比验证
- 风险控制先于投入

部署前，必须确认页面传达的是这条主线，而不是：

- 某几个策略名的堆叠
- 单一买点提示
- 只会显示技术表格的控制台

---

## 2. 首页验收

访问：

- `/?mode=direction`
- `/?mode=research`
- `/?mode=execution`

必须确认：

1. 页面标题与首屏文案
- 标题为 `Hermass 多周期观察台`
- 首屏副文案明确提到：
  - 多周期相互影响观察
  - 单周期运行观察
  - 概率与盈亏比验证

2. 低门槛入口
- 首页存在 `从这里开始`
- 至少包含 4 个入口：
  - 先判断现在能不能做
  - 先知道看哪些方向
  - 直接看今天该跟踪什么
  - 先研究一只票

3. 模式切换
- `方向 / 研究 / 执行` 三种模式切换可用
- 三种模式的：
  - headline
  - 当前结论
  - 为什么
  - 先做什么
  - 第一次使用建议
  都不是同一套文案重复改标题

4. 术语说明
- 首页必须解释：
  - `State`
  - `EF`
  - `RR`
  - `Cron`

5. 首页不应误导
- 不应把 `VCP / 2560 / 布林强盗` 放在首屏中心
- 不应让用户一进来就面对大段调试输出

---

## 3. 市场页验收

访问：

- `/market`

必须确认：

1. 有“现在看什么 / 暂时少看什么”
- 不只是统计数字
- 要有结论层

2. 有市场阶段说明
- 不是只展示 breadth、phase 指标名
- 应能读出：
  - 当前更适合等待 / 试错 / 顺风跟踪
  - 当前是局部改善还是全面进攻

3. 有策略降权提醒
- 明确说明：
  - 策略不是常量工具
  - 先看大节奏，再看哪类路径值得提高权重

4. 不应只突出单一策略
- 页面主语应是环境、结构、阶段
- 不是“今天 VCP 最强，所以去做 VCP”

---

## 4. 行业页验收

访问：

- `/industry`

必须确认：

1. 页面主语是方向，不是股票清单
- 应先看到行业/产业链方向
- 而不是一上来就是个股列表

2. 行业页用于“缩圈”
- 帮用户减少关注面
- 不应鼓励同时看很多行业

3. 如果接了信号样本
- 信号只是辅助说明该方向里有什么
- 不应抢过“方向选择”本身

---

## 5. 研究页验收

访问：

- `/research?stock_code=000021.SZ`

必须确认：

1. 有 `决策收束`
- 当前结论
- 为什么
- 下一步

2. 有 `多因子共振判断`
- 资金流确认
- 资金背离
- 板块承接
- 真假突破
- 持续性

3. 页面不是只展示策略名
- 可以有 `策略视图`
- 但核心应是：
  - 多周期结构
  - 当前阶段位置
  - 风险边界
  - 证据充分度

4. 页面能降级但不崩
- 当基础资料缺失时：
  - 页面仍能打开
  - 会明确提示缺口
  - 不会 500

5. 页面不应伪装完整
- 如果 industry_state / financial_trend 缺失
- 必须明确显示缺失，不编造成结论

---

## 6. 执行页验收

访问：

- `/watchlist`

必须确认：

1. 页面主语是“处理顺序”
- 优先队列
- 观察队列
- 常规积压队列

2. 主列已经降到“共振路径”
- 不再用原始 `vcp / ma2560 / bollinger_bandit` 作为主显示

3. 每个样本都应尽量回答：
- 当前 State 是什么
- D1 支撑在哪里
- 资金流是否确认
- 是否存在资金背离
- 板块是否承接
- 更像真突破还是假突破
- 持续性如何
- 当前应该等待、试错、跟踪还是降级

4. 页面弱化单一策略名
- 路径可以显示
- 但真正影响判断的是：
  - 结构
  - 位置
  - 概率
  - 盈亏比

5. 高风报比样本
- 必须显示中文化后的共振路径
- 不再直接显示英文策略 id

---

## 7. 回测页验收

访问：

- `/backtest`

必须确认：

1. 页面能打开且能运行
- 至少 30 天窗口可提交
- 页面不 500

2. 页面强调验证，不夸大预测
- 回测页必须明确：
  - 同样的 E/F，不同位置与阶段的赔率和概率不同
  - 回测结果是环境/路径验证，不是直接投入建议

3. 有最小 tearsheet
- 指标卡
- 净值曲线
- 交易统计
- 运行日志

4. 空结果可解释
- 若 `0 笔交易`
- 页面要明确解释：
  - 可能因为窗口短
  - 条件严
  - 当前环境本就不活跃

---

## 8. 运行稳定性验收

必须确认：

1. 主要路由可打开
- `/?mode=direction`
- `/?mode=research`
- `/?mode=execution`
- `/market`
- `/industry`
- `/watchlist`
- `/research?stock_code=000021.SZ`
- `/backtest`

2. 页面有数据回退提示
- 当 `latest` 为空时
- 页面会明确显示当前使用的是最近非空日

3. 研究页有降级提示
- 当基础资料缺失或 formatter 降级时
- 页面显式说明，不静默失败

4. 回测页不因导入路径报错
- 之前的 `signal.quality_score` 路径问题必须已消除

---

## 9. 文案主线验收

部署前要最后确认：

1. 不再把网站定义成“策略集合”
- 不以策略名为中心

2. 明确网站本质
- 多周期相互影响观察
- 单周期运行观察
- 概率与盈亏比验证

3. 风险控制优先
- 页面不应给出直接仓位建议
- 页面只应给：
  - 等待确认
  - 小额试错
  - 标准跟踪
  - 重点关注

4. 不把强结构误导成确定盈利
- 强结构只能提高先验
- 不等于结果保证

---

## 10. 通过标准

如果以下条件同时满足，则可以进入服务器部署阶段：

1. 首页三模式已经足够低门槛
2. 五条主路径都能正常打开
3. 研究/执行页面已体现多因子共振判断
4. 回测页可运行且不会误导用户
5. 文案主线已经统一到：
   - 多周期观察
   - 概率与盈亏比
   - 风险控制

如果还有以下任一问题，则继续本地收口，不部署：

- 页面仍大量暴露英文策略 id
- 首屏仍像运维控制台
- 研究页或回测页仍存在 500/空白/不可解释的降级
- 执行页仍主要靠策略名而不是共振与位置判断
