## 1. 目标与边界

### 1.1 AI 助手做什么

- **翻译层**：把 State 编码、ef_count、适配度、资金流信号翻译成用户能理解的中文
- **解释层**：说明「为什么当前是这个判断」，引用具体数据来源
- **导航层**：每次回答结束后，给用户明确的下一步页面入口
- **降级层**：数据缺失或过旧时，诚实降级而不是伪装完整

### 1.2 AI 助手不做什么

| 不做 | 原因 |
|------|------|
| 买入 / 卖出 / 加仓 / 减仓 建议 | 合规边界 |
| 目标价、止损价、止盈价 | 合规边界 |
| 必涨 / 必跌 / 稳赚 等确定性判断 | 系统不预测涨跌 |
| 收益承诺 | 合规边界 |
| 仓位建议（几成仓、多少手） | 合规边界 |
| 编造不存在的数据 | 所有数据必须有来源 |
| 把低频背景数据说成当日结论 | 违反 DATA_FRESHNESS_POLICY |
| 自由 SQL 查询、自由策略生成 | Phase 2 保留 |
| 多 Agent 长链推理 | Phase 2 保留 |

## 3. 统一回答结构（6 段）

每次回答**固定 6 段**，不随问题类型变化：

```
1. 当前结论     — 1-2 句话，直接回答用户问题
2. 为什么       — 引用具体数据，说明判断依据
3. 多周期环境   — 说明 MN1 / W1 / D1 的相互关系，回答当前大背景
4. 单周期位置   — 说明当前日线/短周期在运行的哪个阶段
5. 暂时不用看什么 — 告诉用户当前可以忽略什么，降低信息压力
6. 下一步去哪里  — 1-2 个页面跳转入口
```

### 3.1 结构约束

- 每段之间用空行分隔
- 「当前结论」必须是完整中文句子，不以数字开头
- 「为什么」必须引用具体数据（数字或 State 编码），不得用「根据分析」等空话
- 「多周期环境」必须体现大周期与中周期是否同步，不允许省略为泛泛的“趋势较好”
- 「单周期位置」必须说明当前是等待确认、刚突破、活跃推进、中段延展或高位衰竭中的哪一类
- 「暂时不用看什么」不允许写成「暂时不用看 XXX 策略」— 只能说环境和数据类型
- 「下一步去哪里」必须给出具体页面名称和理由

## 4. 每类问题的字段需求

### 4.1 市场问答

**输入字段**（Codex 从已有模块读取）：

| 字段 | 来源 | 类型 | 当数据缺失时 |
|------|------|------|-------------|
| `market_phase` | `web/main.py` 聚合 | `{"phase": "...", "breadth": ..., "phase_label": "..."}` | 跳过市场阶段话术，只保留 ef2 统计 |
| `daily_snapshot` | `outputs/daily_snapshot.json` | `{"market": {"ef2_count": ..., "ef2_pct": ..., "stocks": ...}, "ef_dist": {...}}` | 无法回答市场类问题，触发降级 |
| `market_assets_state` | `outputs/market_assets_state/` | ETF State + 行业 ETF 覆盖 | 跳过 ETF 辅助确认 |
| `macro_chain_prior` | `outputs/macro_chain_prior/` | 宏观评分 + 象限 | 不参与主判断，仅作为背景提一句 |

**回答生成规则**：

```
市场阶段 + ef2 广度 → 判断「当前是局部改善还是全面进攻」
  ef2_pct > 15% → 偏进攻环境
  ef2_pct 8-15% → 震荡选择环境
  ef2_pct < 8% → 偏等待环境

策略降权提醒（每次市场回答必须包含）：
  - 策略不是常量工具
  - 当前阶段应优先看哪类路径（收缩释放 / 趋势推进 / 波动扩张）
  - 不够条件时不要强行套用

多周期段要求：
  - 必须指出月线 / 周线 / 日线是否同步
  - 必须指出当前主导节奏是大周期扩张、周线跟随，还是仅日线局部修复

单周期段要求：
  - 必须指出当前更接近等待、试错还是顺风推进
```

### 4.2 行业问答

**输入字段**：

| 字段 | 来源 | 类型 |
|------|------|------|
| `industry_rotation` | `outputs/industry_rotation/` | 行业轮动数据 + 方向缩圈 |
| `industry_position_summary` | `outputs/industry_chain/industry_position_*` | 行业景气 + 产业链位置 |
| `market_assets_state` | `outputs/market_assets_state/` | 行业 ETF State |
| `sector_resonance` | `industry_slice.detect_sector_resonance()` | 板块共振信号 |

**回答生成规则**：

```
先做方向缩圈 → 再指出当前共振行业 → 最后提示不用追的行业

行业回答不说「XX 行业强」——说「XX 行业当前处于什么结构位置」
行业回答不给排名——给分类（共振中 / 可观察 / 先少看）

多周期段要求：
  - 说明行业是在大周期共振下走强，还是仅短周期局部改善

单周期段要求：
  - 说明行业当前更像起势初期、扩散中段还是高位延展
```

### 4.3 个股问答

**输入字段**：

| 字段 | 来源 | 类型 | 说明 |
|------|------|------|------|
| `stock_code` | 用户输入 | `str` | 6 位代码，如 `000021` |
| `foundation_state` | `p116_foundation.duckdb` → `d1_perspective_state` | MN1/W1/D1 State + ef_count + ATR | 核心 |
| `unified_view` | `outputs/unified_view/` | 多因子共振（State / 资金流 / 板块承接 / 突破判断 / 持续性 / 阶段位置） | 辅助判断 |
| `research_evidence` | `external_research_evidence` | Quick / Deep / Evidence 三卡 | 辅助 |
| `reward_risk` | `outputs/reward_risk/` | RR 估算 | 辅助（周更，允许 7 天滞后） |
| `forward_observation` | `outputs/forward_observation/` | 历史同状态路径 | 辅助 |
| `moneyflow` | `outputs/moneyflow/` | 主力资金流 | 辅助（日更，允许 1 天滞后） |

**回答生成规则**：

```
1. 先看三周期 State 结构（MN1/W1/D1）→ 判断这只股票当前处于什么位置
2. 再看 ef_count → 共振强度
3. 再看 unified_view 的多因子 → 有没有资金确认 / 板块承接 / 假突破风险
4. 最后说 RR 参考（如果数据有效）

个股回答不说的：
  - 不说「建议关注」
  - 不说「可以买入」「值得买入」
  - 不说「目标价」「止损价」
  - 不说「现在就是买点」

个股回答说的：
  - 当前处于什么结构位置
  - 三周期之间的关系（共振 / 冲突 / 独立）
  - 历史同状态路径的行为特征（不是预测，是描述）
  - 当前数据能支撑到什么程度

多周期段要求：
  - 必须明确 MN1 / W1 / D1 是共振、冲突还是错位

单周期段要求：
  - 必须指出当前单周期是等待确认、刚突破、推进中段、高位延展、或疑似衰竭
```

### 4.4 导航问答

**输入字段**：

| 字段 | 来源 | 类型 |
|------|------|------|
| `page_context` | 请求参数 | `str` — 当前所在页面 |
| `daily_snapshot` | `outputs/daily_snapshot.json` | 市场摘要 |
| `watchlist_summary` | `web/main.py` 聚合 | 执行页摘要 |
| `market_phase` | `web/main.py` 聚合 | 市场阶段 |

**回答生成规则**：

```
根据当前页面上下文，告诉用户：
  - 这一页是用来做什么的
  - 当前应该先看这一页的什么
  - 看完这一页后下一步去哪

导航回答不是「系统说明书」— 是基于当前数据状态的任务指引

多周期段要求：
  - 至少一句说明“先看多周期环境，再看单周期位置”

单周期段要求：
  - 当前若已知环境偏等待/试错/推进，要在导航中体现
```

## 6. 合规规则

### 6.1 明确禁止的表述

以下措辞在任何情况下不得出现在回答中：

- 「建议买入」「建议卖出」「建议持有」「建议加仓」「建议减仓」
- 「目标价」「止损位」「止盈位」
- 「必涨」「必跌」「稳赚」「确定」「保证」
- 「仓位」「几成仓」「多少手」
- 「收益预期」「年化收益」「预期回报」
- 「强烈推荐」「重点推荐」「首选」

### 6.2 允许的表述

- 「当前处于…结构位置」
- 「历史上同状态路径平均表现为…（历史统计，不代表未来）」
- 「结构更适合…类策略的观察条件」
- 「数据来源：…，报告期：…」

### 6.3 免责后缀

每次回答末尾自动追加：

> 以上内容基于系统确定性数据生成，仅供研究参考，不构成投资建议。历史统计不代表未来表现。

## 7. 降级规则

### 7.1 数据缺失降级

| 场景 | 行为 |
|------|------|
| `daily_snapshot` 不存在 | 市场/行业/导航类：返回降级回答「当前市场数据暂未更新，请稍后再试。如需查看最新数据，可手动触发流水线。」 |
| Foundation DB 不存在 | 个股类：返回降级回答「当前暂无 {stock_code} 的 State 数据。请确认代码是否正确，或等待今日流水线完成。」 |
| `unified_view` 缺失 | 个股类：跳过辅助判断，只基于 State 结构和 ef_count 给出核心判断 |
| `moneyflow` 过期 | 个股类：不写「资金确认」相关话术 |
| `reward_risk` 缺失或过期 | 个股类：跳过 RR 参考，回答中不提及 |
| `macro_chain_prior` 过期 | 市场类：不参与主判断，仅在末尾提一句「宏观背景参考数据较旧，暂不作为当前判断依据」 |

### 7.2 数据过旧降级

按 `DATA_FRESHNESS_POLICY.md` 分层规则：

| 数据层 | 允许滞后 | 超期行为 |
|--------|---------|---------|
| 主判断层（Foundation DB / daily_snapshot / forward_observation） | 0 天 | 回答降级为「数据未更新」，不生成结论 |
| 辅助判断层（unified_view / industry_rotation / reward_risk） | 7 天 | 可展示，但必须标注来源日期 |
| 背景参考层（macro_chain_prior / industry_position） | 10 天 | 可展示，但不参与主判断话术 |

### 7.3 无法识别意图时的降级

```
当前结论：我没有完全理解你的问题，但可以告诉你现在能做什么。
为什么：Hermass 助手当前支持四类问题——市场环境、行业方向、个股研究、页面导航。
下一步去哪里：
  1. 先看市场 → 打开市场页
  2. 直接搜一只股票 → 打开研究页
  3. 告诉我股票代码，我帮你看

提示：你可以直接发 6 位代码（如 000021），或问「现在能不能做」。
```

## 8. 示例回答

### 8.1 市场类示例

**示例 1：震荡选择环境**

> **当前结论**
> 当前不适合全面铺开，更适合先做选择题。
>
> **为什么**
> 全市场 ef≥2 占比 11.2%（616/5508），处于 8-15% 的震荡选择区间。扩张广度在收窄而非加速——这是震荡偏强环境的典型特征，不是全面进攻信号。VCP 收缩释放路径在 C/D 环境下匹配度最高，当前更适合筛选完成收缩的标的，而非追突破。
>
> **暂时不用看什么**
> 暂时少看高位延展样本。不要把布林突破或普通均线金叉当成早期机会——当前环境下未经收缩确认的突破，历史统计上超额不明显。
>
> **下一步去哪里**
> 先看市场页确认整体结构，再去执行观察页筛候选。
>
> 以上内容基于系统确定性数据生成，仅供研究参考，不构成投资建议。

**示例 2：偏进攻环境**

> **当前结论**
> 当前处于偏进攻窗口，但需要分批而非一次铺满。
>
> **为什么**
> 全市场 ef≥2 占比 18.3%，三周期共振广度在加速而非收窄。大盘结构 MN1=E/F（牛市确认）+ W1=C/D（周线整理），这是 MA2560 趋势推进路径的最优匹配环境。但注意——ef2 上升速度如果过快，反而要警惕短期情绪过热。
>
> **暂时不用看什么**
> 暂时不用看 VCP 高离散度标的（多次收缩失败后仍在震荡区间内的）。当前环境下，趋势推进路径的效率高于蓄力路径。
>
> **下一步去哪里**
> 打开市场页确认阶段节奏，然后去执行观察页筛选趋势共振候选。
>
> 以上内容基于系统确定性数据生成，仅供研究参考，不构成投资建议。

**示例 3：偏等待环境**

> **当前结论**
> 当前更适合等待，不适合强做。
>
> **为什么**
> 全市场 ef≥2 占比仅 5.8%，处于收缩等待区间。三周期结构整体偏弱，大盘 MN1 处于 0-7 收缩区域。布林强盗和 MA2560 在收缩环境下历史 t-stat 均低于 1.0——信号不显著。VCP 在收缩环境中的表现也显著弱于 C/D 环境。
>
> **暂时不用看什么**
> 暂时不用看任何需要「趋势背景」的策略路径。当前不应把任何突破信号当成机会。
>
> **下一步去哪里**
> 打开市场页确认阶段判断。建议等待 ef2 占比回到 8% 以上再重新评估。
>
> 以上内容基于系统确定性数据生成，仅供研究参考，不构成投资建议。

### 8.2 行业类示例

**示例 1：有共振行业**

> **当前结论**
> 电子和机械设备当前处于板块共振状态，是今天方向缩圈后的主要观察区域。
>
> **为什么**
> 电子行业 22 只标的同日从 ef<2 跳至 ef≥2，触发板块级共振信号。机械设备 23 只共振。两个行业的 ETF State 均为扩张有趋势结构——行业层面的三周期环境与个股层面共振形成双向确认。
>
> **暂时不用看什么**
> 暂时先少看消费类和地产链行业——当前这些行业未出现板块级共振，个股信号多为孤立触发，不具备行业层面的一致性确认。
>
> **下一步去哪里**
> 打开行业页看完整缩圈方向，然后去执行观察页在共振行业内筛个股。
>
> 以上内容基于系统确定性数据生成，仅供研究参考，不构成投资建议。

**示例 2：无共振行业**

> **当前结论**
> 当前没有行业出现板块级共振信号，方向判断需要回到个股层面做。
>
> **为什么**
> 全市场 18 个主要行业中，无一触发同日 3+ 标的从 ef<2 跳至 ef≥2 的共振条件。行业 ETF State 分布显示多数行业处于分散震荡状态——当前没有「整个板块在动」的信号，更适合在个股层面逐只评估。
>
> **暂时不用看什么**
> 暂时不要用行业轮动逻辑来替代个股判断。在无共振环境下，行业归属只是标签，不是信号。
>
> **下一步去哪里**
> 打开执行观察页，从全市场候选池中按 State 结构和适配度逐只筛选。
>
> 以上内容基于系统确定性数据生成，仅供研究参考，不构成投资建议。

**示例 3：具体行业查询**

> **当前结论**
> 电子行业当前处于扩张有趋势结构，ef2 占行业比重 14.3%，高于全市场均值——属于当前相对集中的方向之一。
>
> **为什么**
> 行业 ETF State：MN1=F / W1=C / D1=E，ef_count=3（三周期共振）。行业景气度评分 6.2/10（来源：industry_position，更新至 2026-05-22）。板块共振状态：有（22 只标的本周触发共振）。
>
> **暂时不用看什么**
> 行业内部标的的离散度较高——不是所有电子股都处于同一结构节奏。不要用行业强来替代个股判断。

## 9. 输出 JSON 合同

### 9.1 接口定义

```
POST /api/chat/query
```

**请求体**：

```json
{
  "message": "现在能不能做",
  "page_context": "market",
  "stock_code": null
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `message` | `str` | 是 | 用户自然语言输入 |
| `page_context` | `str` | 否 | 当前页面路径标识：`market` / `industry` / `watchlist` / `research` / `backtest` / `index` |
| `stock_code` | `str` | 否 | 6 位代码（如 `000021`），仅个股问答需要 |

**响应体**：

```json
{
  "answer": "当前不适合全面铺开，更适合先做选择题。",
  "why": "全市场 ef≥2 占比 11.2%，处于 8-15% 的震荡选择区间。",
  "avoid": "暂时少看高位延展样本，不把所有突破都当成早期机会。",
  "next_actions": [
    {"label": "打开市场页", "url": "/market"},
    {"label": "打开执行观察", "url": "/watchlist"}
  ],
  "sources": ["daily_snapshot", "market_phase", "market_assets_state"],
  "freshness_note": "数据日期：2026-05-29，快照含 5508 只股票。"
}
```

### 9.2 字段详解

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `answer` | `str` | 是 | 「当前结论」段，1-2 句话 |
| `why` | `str` | 是 | 「为什么」段，引用具体数据 |
| `avoid` | `str` | 是 | 「暂时不用看什么」段 |
| `next_actions` | `list[dict]` | 是 | 每个 action 有 `label`（按钮文字）和 `url`（相对路径） |
| `sources` | `list[str]` | 是 | 本次回答使用的数据模块名 |
| `freshness_note` | `str` | 是 | 数据日期标注，格式「数据日期：YYYY-MM-DD，…」 |

### 9.3 降级响应

当数据缺失无法生成回答时，返回降级结构：

```json
{
  "answer": "当前市场数据暂未更新，请稍后再试。",
  "why": "daily_snapshot 缺失，无法获取最新 ef2 分布和市场阶段判断。",
  "avoid": "数据未就绪时，任何市场判断都不具备时效性。",
  "next_actions": [
    {"label": "打开市场页", "url": "/market"}
  ],
  "sources": [],
  "freshness_note": "数据日期：缺失。请确认流水线已完成。"
}
```

### 9.4 最小实现路径（Codex 视角）

Codex 实现 `/api/chat/query` 的最小流程：

```
1. 接收 message + page_context + stock_code
2. 调用意图分类（现有 intent_router.classify_intent 或 新增 fallback 规则）
3. 根据分类读取对应数据：
   - 市场 → daily_snapshot.json + market_phase 聚合
   - 行业 → industry_rotation + sector_resonance
   - 个股 → foundation db（DuckDB 查询）+ unified_view + research_evidence
   - 导航 → daily_snapshot 摘要 + page_context
4. 按回答合同生成 4 段（不调用 LLM，纯规则拼接）
5. 追加免责后缀 + 来源标注
6. 返回 JSON
```

## 10. Phase 2 保留项

以下能力明确保留给 Phase 2，**不在本版本中实现**：

| 保留项 | 原因 |
|--------|------|
| NL→SQL 自然语言查询 | 需要 LLM 依赖，增加故障面 |
| 自由策略生成 | 需要多 Agent 推理链 |
| 多 Agent 长链推理 | 需要推理协议设计 + 上下文管理 |
| AI 每日叙事生成 | 需要 LLM 调用 + 话术质量审核 |
| 对话记忆（多轮上下文） | 需要会话管理升级 |
| 历史相似环境嵌入检索 | 需要 embedding + 向量数据库 |
| 交易建议生成 | 合规边界——永不纳入 |
