# Predeploy Scope Freeze

日期：2026-05-29  
状态：生效  
适用范围：`Hermass 多周期观察台` 首次内部部署前冻结范围

---

## 1. 冻结目的

本次部署前的目标不是继续扩功能，而是把已经成型的产品主线稳定交付：

- 多周期相互影响观察
- 单周期运行位置观察
- 概率与盈亏比辅助判断
- 风险控制先于投入

本次冻结后，部署前仅允许：

- 修复阻塞访问的真实缺陷
- 收口文案与口径不一致
- 修复部署脚本与运行时提示
- 补充验收与交付文档

不再允许在主路径新增能力。

---

## 2. 本次上线包含什么

### 2.1 网站主路径

本次上线包含以下页面：

- 首页三模式：
  - `/?mode=direction`
  - `/?mode=research`
  - `/?mode=execution`
- 市场页：`/market`
- 行业页：`/industry`
- 执行页：`/watchlist`
- 研究页：`/research?stock_code=...`
- 回测页：`/backtest`

### 2.2 已包含的产品能力

- 首页低门槛入口：
  - 先判断现在能不能做
  - 先知道看哪些方向
  - 直接看今天该跟踪什么
  - 先研究一只票
- 首页三模式差异化文案与首次使用建议
- 市场页的环境判断、策略降权提示、当前重点/降权方向
- 行业页的方向缩圈能力
- 执行页的多因子共振观察：
  - State
  - D1 支撑提示
  - 资金流确认
  - 资金背离
  - 板块承接
  - 真/假突破
  - 持续性
  - 阶段位置
  - 投入层级
- 研究页的：
  - 决策收束
  - 证据充分度
  - 多因子共振判断
  - Quick / Deep / Evidence 三卡
- 回测页的最小验证闭环：
  - 参数表单
  - 结果指标卡
  - 最小 tearsheet
  - 运行日志
  - 空结果说明

### 2.3 已包含的运行时保护

- 当最新信号文件为空时，页面提示最近非空回退
- 当研究 evidence 降级时，页面提示 warning，而不是伪装完整
- 当基本面或研究格式化失败时，研究页不直接崩溃

## 3. 本次上线不包含什么

以下内容明确不纳入本次部署：

### 3.1 AI 原生增强层

- 自然语言到 SQL 查询层
- 多 Agent 推理链
- 历史相似环境嵌入检索
- AI 每日叙事生成主路径
- 任意自然语言策略生成

原因：

- 当前确定性底座已可用
- 部署前引入 LLM 依赖会新增故障面
- 这些能力属于 Phase 2 增强层，不是首发必需项

### 3.2 交易执行能力

- 自动下单
- 自动仓位决策
- 自动止损止盈执行
- 外部券商/交易终端联动

### 3.3 更重的产品化能力

- 登录/权限系统
- 团队角色管理
- 对外开放门户
- 完整策略实验室前台
- 自由脚本回测编辑器
- 图表型高级回测分析台

---

## 4. 本次首发产品定位

本次上线的对外/对内产品主线统一为：

> Hermass 是一个多周期观察与风险控制系统。  
> 它帮助交易者先判断当前环境，再缩圈方向，再筛样本，再决定等待、试错、跟踪或降级。

本次禁止把产品主线重新拉回：

- 固定几个策略名的展示台
- 单一买卖信号工具
- “AI 什么都能答”的聊天壳

---

## 5. 部署前允许的最后修改范围

部署前只允许继续做以下四类变更：

1. 修复真实 500 / 空页 / 路由不可访问问题
2. 修复首屏口径不一致、残留英文策略 id、错误文案
3. 修复部署脚本、Nginx/systemd、数据新鲜度提示
4. 补齐验收报告、交付说明、部署说明

超出以上范围的新增需求，统一转入 Phase 2。

## 6. Phase 2 候选项

以下能力进入后续阶段，不纳入本次首发：

### P2-AI

- AI 每日叙事生成
- NL→SQL 只读查询
- 多 Agent 推理链
- 历史相似环境检索

### P2-产品

- 策略实验室 `/strategy-lab`
- 新策略跟进页
- 更完整的回测 tearsheet
- 观察池与提醒账本联动

### P2-运维

- 服务器正式域名与 HTTPS
- 团队内部鉴权
- 轻量监控与运行日志面板

## 7. 冻结结论

从本文件生效起：

- 本次首发以“多周期观察台”完成度为唯一主线
- 所有新增 AI/策略实验想法一律不再进入本轮部署范围
- 部署前工作只围绕稳定性、口径一致性、验收与交付展开
