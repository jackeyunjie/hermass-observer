# Weekly Cognitive Recap Spec

版本：v0.1  
日期：2026-05-28  
范围：A 股 / Research-Only / 每周五收盘后复盘卡

## 1. 目标

每周认知复盘卡不评价用户“对错”，只帮助用户回看自己这一周关注了什么、这些标的发生了什么变化、系统是否在持续观察。

Phase 1 目标：

- 强化“关注轨迹可回看”
- 降低信息分散感
- 为后续认知检测累积结构化行为数据

## 2. 数据来源

Phase 1 只用本地数据：

- `outputs/cognitive/behavior_*.json`
- 飞书消息形成的 `BehaviorEvent`
- `d1_perspective_state`
- `industry_state / sector_resonance`

## 3. Phase 1 输出内容

每周五收盘后，生成一张简版复盘卡：

1. 本周问过哪些股票 / 行业
2. 这些标的本周 `State` 变化了什么
3. 其中哪些进入了更强共振，哪些转入确认或走弱

不做：

- 盈亏归因
- 买卖对错判断
- 个性化性格分析

## 4. 输出格式

建议保持短：

```text
本周关注回顾：你本周重点问了 4 只股票、2 个行业。
其中 000021 从 E/E/C 走到 E/E/F，电子行业共振维持增强；600519 从 E/E/E 进入确认阶段。
```

可追加一段简短系统视角：

```text
系统观察：你本周更关注强趋势延续标的，关注轨迹集中在电子与消费两个方向。
```

限制：

- 3 到 6 行
- 不做“你应该”
- 不做交易建议

## 5. 用户范围

Phase 1 先做两种模式：

1. 群级周报  
   面向群内整体关注轨迹汇总

2. 用户级周报  
   面向单个 `user_id` 的行为账本摘要

优先实现顺序：

- 先做群级
- 再做用户级

## 6. 建议落点

建议新增：

```text
scripts/build_weekly_cognitive_recap.py
```

可选再加：

```text
scripts/send_weekly_cognitive_recap_to_lark.sh
```

## 7. 暂不做

- 系统判断 vs 实际走势量化评分
- 用户认知等级打分
- 交易收益对照
- 自动生成长篇周报
