## 逐章详细映射

### 一、基本结论 → `discard`

**原功能**：
- 为基金经理提供高浓度决策摘要
- 提炼行业投资逻辑（2-3个核心逻辑）
- 提炼公司投资逻辑（2-3个核心逻辑）
- 列出主要风险（资本市场风险+公司个体风险）

**核心价值**：无独立研究价值，是后续章节的摘要合集

**为何废弃**：
1. **合规冲突**："投资逻辑"一词直接服务于投资决策，与 Research-Only 边界冲突
2. **角色冲突**：原prompt要求"为忙碌的基金经理提供决策摘要"，当前系统不服务基金经理
3. **内容冗余**：所有信息均可从二、三、四章提取，无需独立章节
4. **强结论导向**：要求"每一点先用4-8个字的加粗小标题概括"，容易形成"推荐"印象

**替代方案**：
- 快速问答卡的"结论摘要"行（1-2句话，无投资倾向）
- 深度研究卡的模块化展示，不生成摘要式结论

### 五、盈利预测 → `downgrade`

**原功能**：
- 生成2025E-2027E盈利预测表格（营收、利润、EPS、ROE）
- 分析增长质量（营收驱动因素、成本费用管控、净利润增长质量）
- 给出盈利预测结论（高速成长期/稳健成熟期/业绩拐点期）

**核心价值**：未来业绩趋势判断

**为何降级**：
1. **数据稳定性**：EPS一致预期数据稳定性有限，当前系统缺乏完整数据源
2. **前瞻性风险**："预计""有望"等前瞻性判断容易越界
3. **合规边界**：盈利预测本质是为投资决策提供依据

**降级方案**：

| 原功能 | 降级后 | 目标位置 |
|--------|--------|---------|
| 2025E-2027E预测表格 | 近4个季度EPS趋势 | `evidence.financial_trend.period_rows` |
| 增长质量分析 | 历史营收/利润趋势描述 | `deep card` 财务趋势模块 |
| 盈利预测结论 | 去除"高速成长期"等定性判断 | 不输出 |

**输出示例（降级后）**：
```
盈利趋势观察（基于历史数据）：
  近4个季度EPS：2.39 → 2.62 → 2.58 → 2.45
  趋势方向：波动下行
  营收增速：3.3%（最新报告期）

  注意：以上为历史数据的趋势描述，不构成未来盈利预测。
```

### 六、估值分析 → `downgrade`

**原功能**：
- 行业估值对比（PE/PB/市值排名表格）
- 计算合理估值（成长型：EPS×CAGR；价值型：EPS×PE×ROE调节系数）
- 估值分析结论（预期收益空间+投资建议）

**核心价值**：估值水平参考

**为何降级**：
1. **合规高风险**：合理估值计算涉及主观假设，不同假设得出不同结论
2. **角色风险**：prompt要求"首席证券分析师（CFA执业者）"角色，暗示专业投资判断
3. **公式化投资建议**："预期收益空间>20%：买入"属于明确投资建议

**降级方案**：

| 原功能 | 降级后 | 目标位置 |
|--------|--------|---------|
| 行业估值对比表格 | 保留，作为参考数据 | `evidence.valuation_reference` |
| 合理估值计算 | 废弃，不输出 | — |
| 投资建议 | 废弃，不输出 | — |
| PE/PB/行业均值 | 保留，标注"仅供参考" | `deep card` 估值参考模块 |

**输出示例（降级后）**：
```
估值参考（仅供参考，不构成建议）：
  当前PE(TTM)：35.2（行业均值42.3）
  当前PB：5.8
  历史分位：PE处于近3年65%分位

  可比公司估值：
    兆易创新：PE 28.5
    北京君正：PE 55.2

  注意：估值受市场情绪、流动性、宏观环境等多重因素影响，
  历史分位不代表未来走势。
```

### 七、市场观点 → `downgrade`

**原功能**：
- 筛选3-5份代表性券商研报
- 提取核心投资逻辑、结论与评级、目标价
- 归纳市场共识与分歧
- 给出首席分析师深度综述

**核心价值**：公开市场信息汇总

**为何降级**：
1. **滞后性**：券商观点有滞后性，目标价经常不准确
2. **角色风险**："首席证券分析师"角色容易让输出带有权威判断色彩
3. **共识/分歧判断**："市场共识"属于主观判断，非事实描述

**降级方案**：

| 原功能 | 降级后 | 目标位置 |
|--------|--------|---------|
| 研报筛选与提取 | 保留，仅做信息汇总 | `evidence.market_views` |
| 核心投资逻辑 | 改为"机构关注要点" | `deep card` 市场观点模块 |
| 评级与目标价 | 仅作为 `evidence.market_views` 的可选原始字段收录，不进入默认卡片输出 | `evidence.market_views`（可选） |
| 市场共识与分歧 | 废弃，不输出主观判断 | — |
| 首席分析师综述 | 废弃，不输出 | — |

**输出示例（降级后）**：
```
公开市场观点（数据来源：巨潮资讯，截至2026-05-28）：
  机构关注要点：
    - 3家机构关注产能释放进度
    - 2家机构关注海外订单可持续性
    - 1家机构提示原材料成本压力

  注意：以上为公开信息中的机构关注方向汇总，不代表系统判断。
```

> **边界说明**：`evidence.market_views` 可选收录原始评级/目标价数据（作为第三方公开信息字段），但默认不进入 quick/deep card 输出。如需展示，必须显式标注"券商评级基于各自模型，不代表本系统观点"。

### 八、投资建议 → `discard`

**原功能**：
- 投资价值总结（3-4句话总结性价比）
- 估值与目标价（引用第六章结论）
- 最终建议（投资评级+核心理由）

**核心价值**：无研究价值，纯投资指令

**为何废弃**：
1. **合规边界**：AI助手不给出买入/卖出判断
2. **角色冲突**：prompt要求"基金经理投资顾问"角色，直接服务于投资决策
3. **明确投资建议**："投资评级：[买入/增持/中性/减持]"属于明确投资建议
4. **与Research-Only冲突**：当前架构明确不生成投资建议

**替代方案**：
- 不输出任何替代内容
- 用户如问"该不该买"，回答"以上为基于公开数据的研究观察，不构成投资建议。投资决策应由投资者独立做出。"

## 总结

| 处置方式 | 章节数 | 说明 |
|---------|--------|------|
| `keep` | 1 | 二、公司概况 |
| `split` | 2 | 三、行业分析 + 四、公司分析 |

- 事实性描述保留（公司概况、财务数据、行业结构）
- 分析框架拆散按需触发（行业分析、公司分析）
- 前瞻性判断降级为历史观察（盈利预测、估值分析）
- 投资建议完全废弃（基本结论、投资建议）
