市场 vs 个股：双路径赚钱框架

先判断今天更适合自上而下还是自下而上。赚钱不是死盯一只票，而是在对的环境里用对的进入方式。

路径选错，后面全错

路径一：先看市场，再选个股

适合顺风或共振较强的日子。先缩圈，再深挖，不要一开始就埋进个股细节。

  1. 市场环境判断 看宽度、活跃性、风险偏好和资金面，确认今天是进攻日还是防守日。
  2. 方向识别 只盯少数承接最好的行业，不在弱方向里找侥幸样本。
  3. 机会池收敛 从推荐页挑局部最强对象，再进入研究页做裁决。
  4. 输出 市场观点 + 方向配置 + 少数重点对象。
互相验证

路径二：聚焦个股，反推验证

适合震荡、偏弱或外部线索驱动的日子。先研究样本，再回头校验它是不是逆风里的假机会。

  1. 样本本体研究 看结构、估值、增长、题材、资金与位置。
  2. 给出裁决 明确继续、等待还是回避，别把材料堆给用户自己拼。
  3. 补环境校验 用市场页和行业页确认它是不是孤立脉冲。
  4. 输出 买卖区间、风险边界、何时回来看。

研究裁决卡：赚钱不是看资料，是看结论

研究页第一屏必须先告诉你这只票应该怎么处理，然后再展开证据。三种结论足够覆盖绝大多数动作。

先结论，后证据
继续跟踪

能继续

  • 多周期结构至少两层站得住。
  • 资金确认或行业承接至少有一层成立。
  • 研究页要给出下一步动作：继续下钻证据，还是创建观察任务。
等条件

先别急

  • 现在更像观察样本，不是成熟机会。
  • 页面必须写清等待什么：资金确认、行业承接，还是 1-3 天站稳。
  • 等条件比强行下判断更赚钱，因为它直接过滤掉低质量试错。
暂不继续

先回避

  • 出现资金背离、跌回关键支撑、或结构明确失效。
  • 研究页要写清楚失效条件，不许只说“风险较大”。
  • 回避不是错过机会，而是保护注意力和本金。

赚钱六步法

把页面使用顺序固定下来，系统才能变成赚钱工具，而不是信息游乐场。

Playbook
1
界定问题

今天先问路径，不先问“买哪只”。先识别环境，再决定入口。

2
收集信号

市场页看环境，推荐页看收敛后的对象池，不平均用力。

3
分析研究

研究页给结论、等待条件、失效条件，而不是只堆资料。

4
形成观点

把对象定为“继续 / 等条件 / 暂不继续”，才算完成一次判断。

5
执行跟踪

创建观察任务，把提醒、有效期、复看时点写清楚。

6
复盘优化

回看哪些判断赚钱，哪些判断浪费注意力，持续调参。

观察账本：赚钱靠复看，不靠记忆

真正的观察对象必须带着时间感存在。不是“收藏”，而是告诉你什么时候回来、看什么、什么情况下直接删除。

Watchlist
回来看的时点

今天刚触发、这两天要复看、还是几天内到期，必须一眼看到。

等待条件

不是“继续关注”，而是写清楚等待资金确认、行业承接还是结构继续展开。

失效条件

跌回支撑、资金背离扩大、或突破失败，任何一条触发都要降级或删除。

下一次提醒理由

系统要告诉你为什么要回来，而不是让你自己想“这票我是不是看过”。

赚钱核心

Hermass 最值钱的不是数据多，而是让你少做三件亏钱的事。

少在逆风里强行找全面进攻机会 逆风日也会有机会，但那是局部、精选、证据化的机会，不是普涨幻想。
少把阅读误当成决策 看完一屏屏材料，不等于知道下一步该买、该等还是该撤。
少让机会失联 真正赚钱的对象通常不是当天看完当天结束，而是后面几天被持续正确跟踪。

最容易亏钱的六个误用

如果用户这样用系统，再漂亮的页面也赚不到钱。这一页要反过来教用户怎么避坑。

Do / Don’t
误用 表现 正确用法
把市场当开关 一看大盘弱，就以为今天完全不能研究个股。 市场只是背景音，决定你先从市场切还是先从线索切。
把推荐池当购物车 看到一堆候选就想全看，注意力被打散。 先看哪一组，再挑其中少数对象，不平铺几十只。
把研究页当资料库 阅读很多，但最后没有明确动作。 先看裁决卡，再决定要不要展开证据层。
把观察账本当收藏夹 对象越来越多，但没有复看节奏。 每个观察对象都必须有等待条件、失效条件和回来时点。
先问路径再问个股 知道今天该从哪进，后面所有页面都会更有效。 首页先看路由，顺风看市场，逆风看局部，迷茫时问观象。
先裁决再执行 继续、等待、回避三分法会显著降低冲动试错。 不给结论不下单，不写观察条件不创建观察任务。
这份秘籍的核心

赚钱不是更多信息，而是更短的决策链路。路径、裁决、观察、复盘，这四层必须闭环。

建议入口

首页负责导航，市场页负责定环境，研究页负责出裁决，观察账本负责驱动复看。

最重要动作

今天先决定从哪条路进；看到研究结论后，再决定要不要创建观察。

一句话收束

先走对路，再盯对票，最后用纪律把正确判断变成复利。
