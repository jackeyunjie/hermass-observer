## 2. 阶段判定：按 5 级优先级匹配

### 2.1 判定输入汇总

| 指标 | 值 | 备注 |
|------|-----|------|
| pool_size | 223 | 当前全三 E/F 池规模 |
| pool_5d | ≥ +35.2% | 单日即达，5 日只会更高 |
| pool_20d | ≥ +35.2% | 仅 1 日数据 |
| vol_ratio | 1.00（EF 池内） | 全市场数据缺失 |
| vol_ratio_5d_avg | 未知 | 历史数据缺失 |
| dispersion | 0.0645（代理） | 完整数据缺失 |
| release_density | 6.87%（广义）/ 0.60%（严格） | 建议采用广义 |

### 2.2 逐级匹配

#### 优先级 1：risk_release（风险释放）

条件：`pool_5d <= -20%` AND `vol_ratio > vol_5d_avg + 0.10`

- pool_5d = +35.2%（正增长，非负增长）
- **不匹配**

#### 优先级 2：contraction（收缩期）

条件：`pool_size < 50` OR (`pool_20d <= -30%` AND `pool_5d <= -5%`)

- pool_size = 223（远大于 50）
- pool_5d = +35.2%（正增长）
- **不匹配**

#### 优先级 3：emergence（趋势新生）

条件：`release_density >= 0.05` AND `pool_5d > 0.10` AND `pool_size >= 50`

| 子条件 | 值 | 满足？ |
|--------|-----|--------|
| release_density >= 0.05 | 0.0687（广义）/ 0.0060（严格） | **是（广义）/ 否（严格）** |
| pool_5d > 0.10 | ≥ +35.2% | **是** |
| pool_size >= 50 | 223 | **是** |

**判定：若采用广义 release_density，满足 emergence 条件**

#### 优先级 4：extension（趋势延展）

条件：`vol_ratio >= 0.45` OR `dispersion >= 0.25`

| 子条件 | 值 | 满足？ |
|--------|-----|--------|
| vol_ratio >= 0.45 | 1.00（EF 池内）/ 未知（全市场） | **可能** |
| dispersion >= 0.25 | 0.0645 | **否** |

**判定：若 EF 池内 vol_ratio 可代理全市场，则满足 extension 条件**

#### 优先级 5：progression（趋势行进）

条件：`pool_size >= 80` AND `pool_5d between -10% and +15%` AND `vol_ratio < 0.45`

| 子条件 | 值 | 满足？ |
|--------|-----|--------|
| pool_size >= 80 | 223 | **是** |
| pool_5d ∈ [-10%, +15%] | ≥ +35.2% | **否**（超出上限） |
| vol_ratio < 0.45 | 1.00 | **否** |

**不匹配**

### 2.3 判定结果

```
┌─────────────────────────────────────────────────────────────────┐
│  优先级 1: risk_release    → 不匹配 (pool 正增长)                │
│  优先级 2: contraction     → 不匹配 (pool=223, 极大)             │
│  优先级 3: emergence       → 匹配   (release_density=6.87%       │
│                              pool_5d=+35%, pool=223)            │
│  优先级 4: extension       → 待确认 (vol_ratio 数据不完整)       │
│  优先级 5: progression     → 不匹配 (pool_5d 过高, vol 过高)     │
└─────────────────────────────────────────────────────────────────┘
```

**判定结论：趋势新生（emergence）**

**置信度评估**：
- 核心支撑：池规模单日 +35%、释放密度 6.87%、79 只股票三周期共振仅 2 天（新生特征）
- 数据限制：5 日/20 日历史数据缺失、全市场波动率缺失、行业离散度代理值偏低
- 潜在矛盾：EF 池内波动率 100% 偏高，若全市场也如此，则 extension 特征也存在
- **综合置信度：0.70**（中高，受数据完整性限制）

## 6. 数据限制与改进建议

### 6.1 当前数据限制

| 限制项 | 影响 |
|--------|------|
| 仅有 1 日前史数据 | pool_change_5d/20d 无法精确计算 |
| state_ef 仅含 EF=3 股票 | 全市场波动率无法计算 |
| 无 market_assets_state 数据 | 行业 EF 离散度无法精确计算 |
| transition 文件含多日期 | 需按日期过滤，当日数据提取复杂 |

### 6.2 框架逻辑验证结论

| 验证项 | 结论 |
|--------|------|
| 指标定义清晰度 | 良好，四个指标均有明确计算公式 |
| 阈值合理性 | 基本合理，但 release_density 的严格定义（abs(from)<8）可能过严 |
| 优先级逻辑 | 合理，从极端风险到稳定趋势逐级排除 |
| 数据可行性 | 全部指标可从现有 State 缓存计算，无需新增数据源 |
| 阶段区分度 | 良好，emergence 与 progression 的区分（pool_5d > 10% vs -10%~15%）有效 |

### 6.3 框架改进建议

1. **release_density 定义放宽**
   - 当前：`abs(from_score) < 8`（仅 mild 收缩态）
   - 建议：增加广义定义 `from_score < 0`（任何收缩态到扩张态）
   - 理由：强收缩态（-E、-F）到扩张态的转换是更强烈的 emergence 信号

2. **volatility_ratio 计算范围**
   - 当前：从 state_ef 计算，但 cache_scope 限制导致样本偏差
   - 建议：从 foundation_db 直接查询全市场 D1 state，确保覆盖全部 5506 只股票

3. **pool_change_rate 历史数据**
   - 当前：依赖 p116_daily_all_three_ef 历史文件
   - 建议：确保该文件至少保留 20 日历史，或从 DuckDB 历史表查询

4. **置信度计算**
   - 当前：公式较复杂，依赖 current_phase_days
   - 建议：简化公式，在数据缺失时降低置信度而非报错
