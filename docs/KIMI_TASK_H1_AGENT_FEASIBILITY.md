# Kimi 任务：A 股系统 H1 Agent 可行性分析

## 任务概述

分析 A 股系统是否需要引入 H1（小时线）Agent。A 股 T+1 制度下 H1 Agent 价值有限，需要基于数据和逻辑给出明确结论。

## 产出文件

`docs/H1_AGENT_FEASIBILITY_ANALYSIS.md`

## 文档结构要求

```
第一章：H1 Agent 定义
  1.1 H1 Agent 的构成（MN1/W1/D1/H4/H1 @ H1_view）
  1.2 H1 Agent 与 D1 Agent 的区别
  1.3 H1 Agent 的更新频率（每小时）

第二章：A 股制度约束分析
  2.1 T+1 制度的影响
    - 买入当日不可卖出
    - H1 信号触发后最早 T+1 才能执行
    - H1 的高频优势被 T+1 抵消
  2.2 涨跌停限制的影响
    - 涨停无法买入
    - 跌停无法卖出
    - H1 级别的突破信号可能因涨跌停而不可执行
  2.3 交易成本的影响
    - 佣金、印花税、滑点
    - H1 级别的频繁交易成本更高
  2.4 结论：A 股 T+1 制度下 H1 Agent 的价值评估

第三章：H1 Agent 的潜在价值场景
  3.1 盘中监控（不做交易，只做观察）
    - H1 State 变化作为 D1 信号的"提前预警"
    - H1 进入 E/F 时提示"D1 可能在收盘时也进入 E/F"
  3.2 止损优化
    - H1 跌破支撑位作为更灵敏的止损触发
    - 比等到 D1 收盘止损更早
  3.3 入场时机优化
    - D1 信号触发后，等 H1 也进入有利状态再执行
    - 避免"信号日追高"
  3.4 A 股系统的 H1 Agent 定位
    - A 股 T+1 制度下的 H1 Agent 应用范围
    - 本系统专注 A 股，不涉及港股/美股/MT5

第四章：成本-收益分析
  4.1 实现成本
    - H1 数据采集（5分钟线聚合为小时线）
    - H1 State 计算模块
    - H1 Agent 与 D1 Agent 的集成
    - 测试和验证
    - 预估：2-3 周开发工作量
  4.2 运行成本
    - H1 数据量是 D1 的 6-8 倍
    - Foundation DB 体积增大
    - 每日计算时间增加
  4.3 预期收益
    - 盘中监控：信息增量，但不直接产生交易信号
    - 止损优化：可能降低 1-2% 的平均止损幅度
    - 入场优化：可能提升 0.5-1% 的平均入场质量
  4.4 ROI 评估

第五章：结论与建议
  5.1 推荐方案（含优先级）
  5.2 不推荐的方案及理由
  5.3 分阶段实施路径（如果决定做）

附录：A 股 T+1 制度对 H1 Agent 的约束分析
```

## 约束

- 基于数据和逻辑分析，不做无依据的推测
- 明确给出"做"或"不做"的建议，不要模棱两可
- 如果建议"做"，必须给出优先级和实施路径
- 如果建议"不做"，必须给出替代方案
- 全中文撰写
- 引用 A 股 T+1 制度的具体规则
