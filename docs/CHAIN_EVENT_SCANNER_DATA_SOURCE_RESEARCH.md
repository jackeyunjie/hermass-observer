# 产业链事件扫描器 — 数据源调研报告

版本：v1.0
日期：2026-05-23
状态：调研完成
关联文档：`docs/CHAIN_EVENT_SCANNER_SPEC.md`

---

## 执行摘要

本报告对产业链事件扫描器三大接口的实际可用性进行了系统调研。结论如下：

| 接口 | 当前可用性 | 推荐优先级 | 接入难度 |
|------|-----------|-----------|---------|
| iFinD Agent JSON | 部分可用（需人工导出） | P1 | 低 |
| 政策公告 RSS/API | 部分可用（RSSHub + 自研爬虫） | P2 | 中 |
| chain_dynamics 异常值检测 | 不可用（表为空，数据源需自建） | P3 | 高 |

**核心发现**：规范中假设的自动化数据流在当前环境下均无法直接实现。扫描器的可行路径是：以 iFinD Agent 人工导出 JSON 为事件主源，以 RSSHub 政策路由为辅助，以期货/行业协会数据为 chain_dynamics 的渐进填充源。

## 1. iFinD Agent JSON 接口调研

### 1.1 当前状态

**不可用为自动化接口**，原因如下：

1. **无程序化导出通道**：iFinD 智能体广场（Kimi Agent）的输出是交互式对话，不支持 API 调用或自动 JSON 导出。规范中假设的 `data/Kimi_Agent_股票服务升级/research/*.json` 路径实际不存在 JSON 文件——该目录下仅有 `.md` 文档（`stockpool_dim01_competition_business.md` 等），是 Agent 的设计文档而非运行时输出。

2. **Agent 类型不匹配**：现有 zip 包中的 Agent 为：
   - `Kimi_Agent_股票服务升级` — 股票池产品设计文档
   - `Kimi_Agent_A股VCP多周期策略` — 技术分析策略文档
   - `Kimi_Agent_A股布林强盗VCP多周期策略` — 技术分析策略文档
   均非规范中假设的"算力行业头部公司动态跟踪助手"或"每日热点快讯简报"。

3. **现有事件雷达状态**：`scripts/ifind_event_radar.py` 已实现对 iFinD Agent 导出 JSON 的解析，但 `outputs/event_digest/ifind_event_digest.duckdb` 中 `company_events` 和 `news_briefs` 表均为 **0 条记录**，说明从未成功导入过 Agent 输出。

### 1.2 可行路径

| 方案 | 说明 | 可行性 |
|------|------|--------|
| A. 人工导出 + 自动解析 | 用户在 iFinD 终端中手动复制 Agent 输出，保存为 JSON，由 `ifind_event_radar.py` / `chain_event_scanner.py` 解析入库 | **可行，当前唯一可用路径** |
| B. iFinD HTTP API | iFinD 提供 HTTP API（`ftwc.51ifind.com`），但主要用于行情/财务数据，未确认支持 Agent 输出或行业新闻流 | 需进一步验证 |
| C. iFinD Python SDK | `THS_iFinDPy` 可获取新闻、公告数据，但非 Agent 解释层输出 | 可行，但数据为原始事实层，非事件摘要层 |

### 1.3 推荐方案

**采用方案 A（人工导出）+ 方案 C（SDK 新闻数据）作为 fallback**：

- **主源**：用户每日在 iFinD 终端运行"行业动态跟踪"类 Agent，将输出保存为规范格式的 JSON（`events` / `news_briefs` / `announcements` 数组），放入 `data/ifind_agent_exports/YYYYMMDD/` 目录，由扫描器自动解析。
- **Fallback**：通过 `THS_iFinDPy` 的 `THS_News` 或 `THS_BD` 接口获取原始新闻/公告，用关键词规则提取产业链事件（自动化但质量较低）。

### 1.4 输出格式评估

iFinD Agent 的输出格式不固定，取决于 Agent 类型。规范中假设的字段（`events[].title`, `events[].summary`, `events[].type`）需要与具体 Agent 的输出结构对齐。建议定义一个**标准化导入格式**：

```json
{
  "source_agent": "算力行业头部公司动态跟踪助手",
  "export_date": "2026-05-23",
  "events": [
    {
      "title": "事件标题",
      "summary": "事件摘要",
      "date": "2026-05-23",
      "type": "policy",
      "subtype": "补贴政策",
      "affected_chains": ["ai_compute"],
      "impact_direction": "positive"
    }
  ]
}
```

## 2. 政策公告 RSS/API 源调研

### 2.1 中国政府官方源

| 来源 | RSS/API 状态 | 内容类型 | 更新频率 | 接入方式 |
|------|-------------|---------|---------|---------|
| **国务院政策文件库** (`sousuo.www.gov.cn`) | 无原生 RSS，RSSHub 已适配路由 `/gov/zhengce/govall/:advance?` | 国务院文件、国令、国发 | 不定期 | RSSHub 实例或自研爬虫 |
| **国家统计局** (`stats.gov.cn`) | 有原生 RSS (`/wzgl/rss/`) | 统计数据发布 | 月度 | 直接 RSS 订阅 |
| **国家发改委** (`ndrc.gov.cn`) | 无原生 RSS | 政策文件、规划 | 不定期 | 网页爬虫 |
| **工信部** (`miit.gov.cn`) | 无原生 RSS | 产业政策、行业标准 | 不定期 | 网页爬虫 |
| **财政部** (`mof.gov.cn`) | 无原生 RSS | 税收政策、财政公告 | 不定期 | 网页爬虫 |
| **商务部** (`mofcom.gov.cn`) | 无原生 RSS | 贸易政策、反倾销公告 | 不定期 | 网页爬虫 |
| **生态环境部** (`mee.gov.cn`) | 无原生 RSS | 环保政策、限产通知 | 不定期 | 网页爬虫 |

**关键发现**：中国政府网站普遍**不提供结构化 API**。RSSHub 社区已适配部分路由（如 `/gov/zhengce/govall`），但覆盖度有限，且依赖 RSSHub 公共实例的稳定性。

### 2.2 第三方聚合服务

| 服务 | 政策推送能力 | API 状态 | 费用 | 可行性 |
|------|------------|---------|------|--------|
| **企查查** | 企业关联政策监控 | 有开放平台 API (`open.qcc.com`)，但侧重企业信息，非宏观政策流 | 付费 | 低 |
| **天眼查** | 企业关联政策监控 | 有开放平台 API (`open.tianyancha.com`)，同样侧重企业信息 | 付费 | 低 |
| **人民数据库** (`data.people.com.cn`) | 国家政策聚合 | 有数据平台，需申请账号 | 付费/机构 | 中 |
| **八爪鱼采集器** | 通用网页采集 | 工具型，非 API | 付费 | 中（可作为爬虫基础设施） |

**结论**：第三方服务均**不直接提供产业链级别的政策事件流**。企查查/天眼查的政策监控是"企业视角"（某企业受到某政策影响），而非"产业链视角"（某政策影响某产业链）。

### 2.3 推荐方案

**分层接入策略**：

1. **Tier 1（立即可用）**：部署 RSSHub 私有实例，订阅以下路由：
   - `/gov/zhengce/govall/:advance?` — 国务院政策文件库
   - `/gov/stats/rss` — 国家统计局 RSS
   - 按需添加各部委路由（RSSHub 社区持续扩展中）

2. **Tier 2（短期实现）**：自研爬虫补充关键部委：
   - 工信部政策发布页 (`miit.gov.cn/jgsj/wgj/wjfb/`)
   - 发改委政策文件库 (`ndrc.gov.cn/xxgk/wjk/`)
   - 商务部公告 (`mofcom.gov.cn/article/b/`)
   爬虫输出标准化 JSON，由扫描器统一消费。

3. **Tier 3（中期规划）**：接入付费政策数据库（如人民数据库 API），作为高可信度政策源的补充。

## 3. chain_dynamics 异常值检测数据源调研

### 3.1 当前状态

`chain_dynamics` 表当前为 **0 条记录**。规范中设计的异常值检测逻辑（检测 `turning_up` / `turning_down` 或 15% 以上波动）**无数据可运行**。

### 3.2 公开数据源评估

#### 3.2.1 期货价格数据（最可行）

| 数据源 | 覆盖品种 | 数据类型 | 频率 | 接入难度 |
|--------|---------|---------|------|---------|
| **AKShare 期货接口** | 碳酸锂(LC)、工业硅(SI)、多晶硅(PS)、铜(CU)、铝(AL)、原油(SC) 等 | 日线行情、库存、仓单、基差 | 日频 | 低（pip 安装即可） |
| **Tushare 期货接口** | 全商品期货 | 日线行情、持仓排名、仓单日报 | 日频 | 中（需 token） |
| **各交易所官网** | 对应交易所品种 | 库存、仓单、交割数据 | 日/周频 | 中（需爬虫） |

**可直接映射到产业链的期货品种**：

| 产业链 | 期货品种 | 交易所 | AKShare 接口 |
|--------|---------|--------|-------------|
| 新能源车链 | 碳酸锂 (LC) | 广州期货交易所 | `futures_contract_info_gfex()` + 行情接口 |
| 光伏链 | 工业硅 (SI)、多晶硅 (PS) | 广州期货交易所 | 同上 |
| 半导体/AI算力链 | 铜 (CU) | 上海期货交易所 | `futures_contract_info_shfe()` |
| 军工/有色 | 铝 (AL)、镍 (NI)、锡 (SN) | 上海期货交易所 | 同上 |

**关键限制**：期货价格反映的是**金融市场预期**，与产业链实际"产品均价/开工率/库存天数"存在偏差。期货数据适合作为**价格信号代理**，但不等同于行业统计数据。

#### 3.2.2 行业协会数据（高价值但无 API）

| 协会 | 数据类型 | API 状态 | 获取方式 |
|------|---------|---------|---------|
| **中国光伏行业协会 (CPIA)** | 产量、装机量、出口量、价格 | 无 API | 官网月度报告 PDF/网页 |
| **中国汽车工业协会 (CAAM)** | 汽车/新能源车销量、产量 | 无 API | 官网月度统计网页 |
| **中国半导体行业协会 (CSIA)** | 销售额、进出口 | 无 API | 官网新闻/报告 |
| **中国有色金属工业协会** | 有色产量、价格 | 无 API | 官网/杂志 |

**结论**：行业协会数据**质量高、产业链匹配度强**，但均为**非结构化发布**（PDF、网页文章），需要爬虫 + NLP 提取，接入成本高。

#### 3.2.3 商业数据服务商（高质量但付费）

| 服务商 | 数据类型 | API 状态 | 费用 |
|--------|---------|---------|------|
| **上海有色网 (SMM)** | 现货价格、库存、排产、开工率 | 有数据平台 (`data-pro.smm.cn`)，需申请 | 高（机构订阅） |
| **百川盈孚** | 价格、产量、库存、成本、利润、开工率 | 有数据服务，需申请 | 高（机构订阅） |
| **隆众资讯** | 能源化工产业链数据 | 有数据服务，需申请 | 高（机构订阅） |
| **Wind/同花顺 iFinD** | 行业数据库、产业链中心 | iFinD 终端内置 | 已有（终端费用） |

**关键发现**：iFinD 终端本身已包含**行业中心/产业链中心**模块，可查看"产品均价""开工率""库存天数"等指标。但这些数据的**程序化导出**能力未验证。如果 iFinD 的 Excel 导出或 THS_iFinDPy 支持行业数据查询，这将是 chain_dynamics 的最佳填充源。

### 3.3 推荐方案

**分阶段填充策略**：

| 阶段 | 时间 | 数据源 | 填充指标 | 精度 |
|------|------|--------|---------|------|
| **Phase 1** | 立即 | AKShare 期货数据 | 产品均价（期货结算价代理）、库存（仓单代理） | 中 |
| **Phase 2** | 1-2 周 | 行业协会官网爬虫 | 产量、销量、装机量 | 高 |
| **Phase 3** | 1 月 | iFinD 行业数据导出 | 开工率、库存天数、毛利率 | 高 |
| **Phase 4** | 持续 | SMM/百川盈孚 API（如采购） | 现货价、排产、供需缺口 | 最高 |

## 5. 接入路线图

### 5.1 短期（1-2 周）：最小可行扫描器

```
1. 定义 iFinD Agent 标准化导出 JSON Schema
2. 实现 chain_event_scanner.py 核心逻辑（解析 + 分类 + 入库）
3. 部署 RSSHub 实例，订阅国务院政策路由
4. 接入 AKShare 期货数据，填充 chain_dynamics 基础价格指标
5. 实现异常值检测 SQL，输出 turning_up / turning_down 信号
```

### 5.2 中期（1 月）：自动化增强

```
1. 开发部委官网爬虫（工信部、发改委、商务部）
2. 验证 iFinD THS_iFinDPy 行业数据查询能力
3. 开发行业协会爬虫（CPIA、CAAM）
4. 实现事件与期货价格波动的交叉验证
```

### 5.3 长期（按需）：商业数据接入

```
1. 评估 SMM/百川盈孚 API 订阅成本与 ROI
2. 接入付费行业数据，替换期货代理指标
3. 建立产业链数据质量评分体系
```

## 6. 风险与限制

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| iFinD Agent 无自动化导出 | 事件主源依赖人工，可能漏扫 | 定义每日导出 SOP，设置提醒；以 RSSHub + 爬虫为 fallback |
| 政府网站无结构化 API | 政策源采集脆弱，易受反爬 | 使用 RSSHub 官方路由优先；爬虫增加请求间隔和 User-Agent 轮换 |
| 期货价格 ≠ 产业链实际价格 | chain_dynamics 指标偏差 | 标注 `confidence < 1.0`；在文档中声明代理关系 |
| 行业协会数据分散 | 高价值数据获取成本高 | 优先爬取月度摘要数据，非逐日明细 |
| chain_dynamics 表为空 | 异常值检测无法运行 | Phase 1 先用期货数据填充，确保检测逻辑有数据可跑 |

---

## 7. 结论

产业链事件扫描器的**核心瓶颈不是技术实现，而是数据源的自动化获取**。在当前环境下：

1. **iFinD Agent JSON** 是最可靠的事件源，但需要**人工导出**介入。建议将其作为"高质量人工标注 + 自动解析"的混合模式运行。

2. **政策公告** 可通过 **RSSHub + 自研爬虫** 实现半自动化采集，覆盖国务院和主要部委。

3. **chain_dynamics** 需要**从零建设**。最快捷路径是通过 AKShare/Tushare 期货数据建立"价格代理"层，再逐步替换为行业真实指标。

4. **扫描器应先运行起来，再逐步优化**。即使只有 iFinD 人工导出 + RSSHub 政策流 + 期货价格代理，也足以支撑 S_event 分项的基础计算。
