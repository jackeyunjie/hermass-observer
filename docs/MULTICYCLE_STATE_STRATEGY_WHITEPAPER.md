## 第 1 章：State 底座原理

### 1.1 设计目标

传统多周期分析中，日线、周线、月线各自独立计算支撑阻力位和指标状态。由于各周期收盘日期不同（周五日线收盘值与该周周线收盘值可能不同），当对跨周期状态进行联合分析时，存在口径不一致问题。

State 底座的核心设计目标是：**用同一个价格数据点统一描述同一标的在不同时间尺度上的状态，消除跨周期口径差异。**

### 1.2 D1 视角统一原理

所有周期的 position 计算统一使用 **D1（日线）收盘价** 比较各自周期的支撑位（Support）和阻力位（Resistance）：

```text
MN1 position = D1 close vs MN1 SR
W1 position  = D1 close vs W1 SR
D1 position  = D1 close vs D1 SR
```

这意味着在任何一个交易日，MN1、W1、D1 三个周期的状态评估基于同一个价格数据点（当日日线收盘价），而非各自周期的收盘值。这保证了跨周期状态比较的一致性和可复现性。

### 1.3 三周期体系

| 周期 | 代号 | 含义 | 数据来源 |
|------|------|------|----------|
| 月线 | MN1 | 月度趋势和位置 | 由日线按自然月聚合生成 |
| 周线 | W1 | 周度趋势和位置 | 由日线按自然周聚合生成 |
| 日线 | D1 | 日度趋势和位置 | 原始日线数据 |

高周期（MN1、W1）的关键位采用分形理论计算，经确认延迟处理后，通过前向填充对齐至日线日期序列。这确保了在任何观察日期上引用的关键位均为已确认的历史数据，不存在未来数据泄露（look-ahead bias）。

### 1.4 4-bit 编码公式

每个周期的状态由以下公式计算：

```text
score = base + trend_bit * 4 + position_bit * 2 + volatility_bit
```

各分量定义：

| 分量 | 取值 | 含义 | 计算依据 |
|------|------|------|----------|
| base | 0 或 8 | 底座位：收缩或扩张 | 布林带宽在滑动窗口内的分位关系 |
| trend_bit | 0 或 1 | 趋势位：无方向或有方向 | 快速均线 vs 慢速均线（ADX/DI） |
| position_bit | 0 或 2 | 位置位：未突破或突破确认 | D1 收盘价 vs 该周期 SR 关键位 |
| volatility_bit | 0 或 1 | 波动位：波动稳定或波动活跃 | ATR% vs 滑动窗口历史均值 |

由此产生 0-15 共 16 个状态值，对应十六进制编码 0-F。

### 1.5 E/F 状态定义

| 状态码 | 十进制 | bit 拆解 | 完整含义 |
|--------|--------|----------|----------|
| E | 14 | 8+4+2+0 | 多向/扩张/有趋势/突破/波动稳定 |
| F | 15 | 8+4+2+1 | 多向/扩张/有趋势/突破/波动活跃 |

E 和 F 是当前系统的最高强度状态。当 MN1、W1、D1 三周期同时为 E 或 F 时，称为"三周期 E/F 共振"。`ef_count` 表示三周期中状态为 E 或 F 的周期数量。

### 1.6 位置优先符号裁决

当价格位置与趋势方向冲突时，系统采用位置优先规则：

```text
IF D1 close < 该周期支撑位 THEN sign = -（负向）
ELSE IF D1 close > 该周期阻力位 THEN sign = +（正向）
ELSE sign 由趋势方向决定
```

典型场景：某标的月线趋势仍为上升（快速均线高于慢速均线），但 D1 收盘价已跌破月线支撑位，此时 MN1 状态被确定为负值。这解决了传统方法中趋势指标滞后于价格运动导致状态符号误判的技术问题。

带负号的状态码示例：`-E`（-14）、`-C`（-12）、`-F`（-15）。研究脚本需要按"方向 + 绝对值 bit"解码，不能把负值当作普通 0-15。

### 1.7 常见状态速查

| 状态码 | 十进制 | base | trend | position | volatility | 含义 |
|--------|--------|------|-------|----------|------------|------|
| 0 | 0 | 收缩 | 平 | 未突破 | 稳定 | 收缩，无趋势 |
| 4 | 4 | 收缩 | 有方向 | 未突破 | 稳定 | 收缩，有趋势方向 |
| 8 | 8 | 扩张 | 平 | 未突破 | 稳定 | 刚扩张，无趋势 |
| A | 10 | 扩张 | 平 | 突破 | 稳定 | 扩张突破，无趋势 |
| C | 12 | 扩张 | 有方向 | 未突破 | 稳定 | 扩张趋势行进中 |
| D | 13 | 扩张 | 有方向 | 未突破 | 活跃 | 趋势行进，波动开始活跃 |
| E | 14 | 扩张 | 有方向 | 突破 | 稳定 | 趋势突破，波动平稳 |
| F | 15 | 扩张 | 有方向 | 突破 | 活跃 | 趋势突破，波动活跃 |

## 第 2 章：策略匹配方法论

### 2.1 核心思想

State 底座和策略模块承担不同职责：

- **State 底座**：识别先验概率更高的市场环境。它回答"当前环境适合哪类策略"。
- **策略模块**：产生精确、可复现的信号。它回答"策略规则是否触发"。
- **策略信号账本**：只记录策略模块输出的权威信号。
- **提醒层**：只消费账本和环境标签，不重新计算、不简化、不发明策略触发条件。

两者的结合方式是：State 提供场景过滤框架，策略提供精确触发。系统不追求预测，只追求在统计意义上更适合的环境识别。

### 2.2 搜索流程

策略-环境匹配的验证通过以下流程进行：

```text
全市场策略信号（strategy_signal_daily）
  × Foundation DB 中的 MN1/W1/D1 State
  → 按 MN1/W1/D1 State 组合分组
  → 统计各组未来 5/10/20/30/60 日收益和超额收益
  → 与全市场等权、沪深300、中证1000等基准对比
  → 输出最优候选组合
```

关键脚本：

- `scripts/search_2560_optimal_state.py` — 2560 策略
- `scripts/search_vcp_optimal_state.py` — VCP 策略
- `scripts/search_bollinger_optimal_state.py` — 布林强盗策略

统一输出目录：

```text
outputs/strategy_evaluation/{strategy}_optimal_state_search_YYYYMMDD_*.json
outputs/strategy_evaluation/{strategy}_optimal_state_search_YYYYMMDD_*.md
outputs/project/{strategy}_optimal_state_search.md
```

### 2.3 精确组合 vs 模糊 bit 聚合

16x16x16 = 4096 种精确 MN1/W1/D1 State 组合存在过拟合风险。因此系统同时使用两种统计口径：

- **精确组合**：如 `MN1=14, W1=14, D1=15` → `E/E/F`。样本量小时波动大，但能发现特定结构。
- **模糊 bit 聚合**：按 bit 签名聚合，如 `S+B8T1P1V0/S+B8T1P1V0/S+B8T1P1V1`。样本量更大，结论更稳健。

候选发现以精确组合为主，规则升格以模糊 bit 或样本外验证结果为准。

### 2.4 质量闸门

任何候选 State 组合要进入正式规则，必须通过以下闸门：

| 闸门 | 要求 |
|------|------|
| 最小样本数 | 每组至少 30 个样本（可配置） |
| 超额收益显著性 | t-stat > 1.96（95% 置信度）或人工判断可接受 |
| 样本外验证 | 样本内/样本外拆分，避免过拟合 |
| 本地复现 | KIMI 结论必须用本项目 Foundation DB 重新验证 |
| 人工确认 | 任何写入 config 的结论需人工审核签字 |

## 第 3 章：三个策略的验证结论汇总

### 3.1 2560 战法 — 已固化

#### 策略定义

2560 结合趋势与成交量，寻找趋势股回到关键均线附近后重新确认的机会。

- **核心参数**：25 日均线（趋势生命线）、5 日均量线、60 日均量线
- **信号规则**：MA25 坚定向上 + 股价回踩后重新站上 MA25 + VOL5 > VOL60
- **策略 ID**：`ma2560`

#### 本地验证结论

2560 是三个策略中验证最充分、已进入系统规则层的策略。

**条件组合复现（data/project Top20 样本）**：

| 条件 | 命中率 |
|------|--------|
| VOL5 > VOL60 | 20/20 |
| 买入日低点触及 MA25 2% 区间 | 20/20 |
| 收盘在 MA25 2% 区间 | 17/20 |
| MA25 上行 | 17/20 |
| 强势评分 >= 6 | 19/20 |

**已固化的 State 组合**：

```text
{E/E/F, E/F/F, E/F/E} + ma2560_strong_hold
```

即 MN1、W1、D1 三周期均处于 E 或 F 状态，且个股满足 2560 信号规则。

**市场匹配三层口径**：

| 匹配等级 | 含义 | 全量数量 |
|----------|------|----------|
| full_match | 个股规则 + State 组合 + 行业 ETF ef_count >= 2 | 62 |
| stock_only | 个股规则 + State 组合，行业 ETF 字段缺失 | 8 |
| market_unsupported | 个股规则 + State 组合，行业 ETF ef_count < 2 | 97 |
| not_match | 个股不满足 2560 适配 State 组合 | 519 |

**配置文件**：`config/ma2560_state_market_match_rule.json`

**研究状态**：已固化，进入系统规则层。但只能作为候选质量证据，不是无条件买入信号。

### 3.2 VCP — 收缩后释放

#### 策略定义

VCP 捕捉上涨途中的休整。价格和波动率同步收缩，成交量逐步枯竭，随后价格突破支点时形成趋势延续或趋势启动信号。

- **核心参数**：50 日均线 + 200 日均线趋势背景、2-4 次收缩结构、放量突破支点
- **信号规则**：价格突破最近一次摆动高点（支点），含 `vcp_breakout`、`vcp_breakout_no_vol`、`vcp_breakout_weak_vol` 三种口径
- **策略 ID**：`vcp`

#### 本地验证结论

VCP 验证的核心发现是：**路径假设优于静态组合**。

**近期前向验证（2026-04-20 至 2026-05-01）**：2,930 个信号样本，9 个标注日期。

**关键假设对照（主观察窗口 20d）**：

| 假设 | 20d 样本 | 20d 平均超额 | 20d 胜率 | 结论 |
|------|----------|-------------|----------|------|
| D1 近5日经历收缩后释放 | 9 | +5.58% | 44.44% | 样本偏少，方向一致 |
| D1 近10日经历收缩后释放 | 34 | +3.78% | 58.82% | 优于外侧组 |
| D1 近20日经历收缩后释放 | 73 | +4.69% | 56.16% | 通过路径假设 |
| 当前扩张但近20日无收缩前兆 | 525 | +1.19% | 44.19% | 弱于路径组 |
| KIMI 静态候选 {10,12,14}/{12,14}/{4,12} | 110 | -0.45% | 40.00% | 未通过 |
| D1 in {10,12,14} | 542 | +1.44% | 44.83% | 静态状态弱于路径条件 |

注：上述数据来自近期前向验证窗口（9 个日期），样本量有限。KIMI 研究消化记录中的全量验证（2025-06-01 至 2026-05-01，43,259 个已标注样本，223 个日期）同样支持路径假设优于静态组合的结论，详见 `docs/KIMI_STATE_STRATEGY_RESEARCH_DIGEST.md` 第 7.1 节。

**精确 State 组合 Top3（VCP 信号触发时）**：

| 排名 | MN1/W1/D1 | 样本 | 20d 超额 | 胜率 |
|------|-----------|------|----------|------|
| 1 | E/E/F | 11 | +17.02% | 63.64% |
| 2 | E/E/E | 22 | +14.74% | 68.18% |
| 3 | C/0/E | 11 | +10.59% | 54.55% |

**产品化表达**：

```text
VCP 信号出现，D1 近期经历收缩后释放，属于趋势新生观察场景。
```

**研究状态**：路径假设已通过初次本地验证。KIMI 静态组合未通过，不能直接升格为规则。需进一步做样本外验证。

### 3.3 布林强盗 — volatility_bit=0 更稳

#### 策略定义

布林强盗把布林带上轨从传统的阻力参考转化为趋势突破确认位，用于捕捉价格突破后引发的趋势性行情。

- **核心参数**：SMA(50) + 1 倍标准差上轨、30 日趋势过滤、自适应均线出场（50→10 递减）
- **信号规则**：收盘价突破上轨 + 收盘价高于 30 日前最高收盘价 + 从上轨下方穿越到上方
- **策略 ID**：`bollinger_bandit`

#### 本地验证结论

布林强盗验证的核心发现是：**KIMI 候选组合未通过本地数据，波动稳定组表现更稳。**

**验证区间**：2025-06-01 至 2026-05-01，Foundation DB 含 47,219 个已标注样本，223 个标注日期。

**关键假设对照（主观察窗口 20d）**：

| 假设 | 20d 样本 | 20d 平均超额 | 20d 胜率 | 结论 |
|------|----------|-------------|----------|------|
| KIMI 候选组合 {13,14,15}/{14,15}/{12,14,15} | 6358 | -0.40% | 37.50% | 未通过 |
| KIMI 候选组合外 | 39684 | +0.37% | 39.30% | 优于候选组 |
| 三周期 E/F 共振 | 1215 | -0.02% | 39.26% | 20d 不成立，5/10d 较强 |
| D1 in {13,14,15} | 23888 | -0.09% | 38.07% | 未通过 |
| D1 volatility_bit=1 | 13879 | -0.49% | 36.96% | 未通过 |
| D1 volatility_bit=0 | 32163 | +0.59% | 39.95% | 明显更稳 |

**精确 State 组合 Top3（布林强盗信号触发时）**：

| 排名 | MN1/W1/D1 | 样本 | 20d 超额 | 胜率 |
|------|-----------|------|----------|------|
| 1 | E/F/A | 86 | +5.90% | 46.51% |
| 2 | E/-C/A | 39 | +5.15% | 46.15% |
| 3 | C/2/E | 55 | +4.98% | 63.64% |

**核心洞察**：

```text
布林强盗的触发本身是波动突破信号；
但信号出现时，D1 已经波动活跃并不一定更好。
当前样本中，D1 波动稳定组（volatility_bit=0）的 20d 表现更稳。
```

这提示布林强盗信号的最佳环境可能不是"已经波动扩张"，而是"刚从稳定进入扩张"的转折点。

**研究状态**：KIMI 候选组合未通过本地验证，暂不写入正式配置。需要进一步研究波动转折路径。

## 第 4 章：策略互补性分析

### 4.1 趋势生命周期三阶段

如果把 State 系统视为市场环境识别器，三套策略分别对应趋势生命周期中的三个阶段：

| 趋势阶段 | State 背景 | 适配策略 | 核心用途 |
|----------|------------|----------|----------|
| 趋势新生 | D1 刚脱离收缩、三周期共振新近形成 | VCP | 捕捉收缩结束后的支点突破 |
| 趋势行进 | 三周期 E/F 稳固、持续天数增加、波动稳定 | 2560 | 跟踪趋势中的回踩确认 |
| 趋势延展 | 波动从稳定转为活跃、趋势明显延续 | 布林强盗 | 捕捉波动扩张后的趋势加速段 |

### 4.2 base=0 到 base=8 的周期接力

三个策略在 State 空间中存在天然的互补关系，形成一条从收缩到扩张的周期接力路径：

```text
阶段 1：base=0（收缩）
  → 布林带宽收缩，波动率降低
  → 成交量逐步萎缩，筹码锁定
  → 这是 VCP 的压缩阶段
  → State 编码：0, 4 等收缩态

阶段 2：base 从 0 转为 8（收缩 → 扩张）
  → D1 刚脱离收缩，进入扩张
  → 支点突破，成交量放大
  → VCP 信号触发的最佳窗口
  → State 路径：0/4 → 10/12/14

阶段 3：base=8 + trend_bit=1（扩张趋势确认）
  → 三周期 E/F 同步形成
  → 趋势行进，波动相对平顺
  → 2560 回踩确认的最佳窗口
  → State 编码：E/E/F, E/F/F, E/F/E

阶段 4：volatility_bit 从 0 转 1（波动活跃）
  → 趋势进入延展或加速段
  → 波动从稳定转为活跃
  → 布林强盗的趋势突破确认窗口
  → State 编码：D, F 等波动活跃态
```

### 4.3 决策树研究口径

系统内部采用以下研究口径理解策略分工：

1. **当 State 显示收缩或刚脱离收缩时**：重点关注 VCP 类支点突破信号。
2. **当 State 显示趋势稳定、波动平顺时**：重点关注 2560 类趋势回踩确认信号。
3. **当 State 显示波动突然放大或趋势进入延展区时**：重点关注布林强盗类波动扩张信号和已有持有上下文中的防守规则。

### 4.4 协同对照表

| State 环境 | 更适合的策略 | 解释 |
|------------|-------------|------|
| 收缩、趋势孕育、刚脱离收缩 | VCP | 等待支点突破，捕捉趋势新生 |
| 扩张、趋势确认、波动稳定 | 2560 | 观察趋势行进中的回踩确认 |
| 扩张、趋势确认、波动活跃 | 布林强盗 | 捕捉波动扩张后的趋势加速 |

### 4.5 策略环境标签

系统为每个策略信号附加环境标签，描述信号触发时所处的 State 背景：

**通用标签**：

- 三周期共振新近形成
- D1 收缩充分
- D1 刚脱离收缩 / W1 刚脱离收缩 / MN1 刚脱离收缩
- 波动稳定 / 波动偏活跃
- 三周期共振延续

**策略专属标签（研究阶段）**：

- `vcp_lifecycle_stage` / `vcp_environment_fit` / `vcp_fit_reasons`
- `bb_lifecycle_stage` / `bb_environment_fit` / `bb_fit_reasons`

原则：

- 策略信号回答"是否触发"。
- 环境标签回答"触发时的背景是什么"。
- 校准统计回答"历史上类似背景表现如何"。
- 这三者都不构成操作指令。

## 第 5 章：从研究到规则的升格流程

### 5.1 五步升格流程

任何策略-环境匹配结论从研究假设到系统规则，必须经过以下五个步骤：

```text
步骤 1：KIMI 报告
  ↓
步骤 2：对账审计
  ↓
步骤 3：本地验证
  ↓
步骤 4：人工确认
  ↓
步骤 5：写入规则
```

#### 步骤 1：KIMI 报告

KIMI 集群产出研究假设，包括：

- 候选 State 组合建议
- A 股制度环境过滤项
- 回测统计数字（未经本地复现不得展示）
- 参数扫描建议

**风险提示**：KIMI 部分草稿使用了与本项目不一致的 State bit 顺序。KIMI 中的 `base=1/base=0` 二值化写法与本项目的 `base=0/base=8` 不兼容。任何 bit mask 代码禁止直接照抄。

#### 步骤 2：对账审计

将 KIMI 结论与本项目 State 契约对齐：

- State 公式以 `config/deepseek_context.md` 和 `docs/KIMI_STATE_STRATEGY_RESEARCH_DIGEST.md` 第 1 节为准。
- 策略触发条件以 `docs/STRATEGY_DEFINITIONS.md` 和 `backtest/strategy_signals/` 为准。
- 记录哪些可采纳、哪些需本地验证、哪些禁止采用。

参考文档：`docs/BOLLINGER_BANDIT_IMPLEMENTATION_AUDIT.md`

#### 步骤 3：本地验证

使用本项目 Foundation DB 和只读搜索脚本进行本地验证：

```bash
# 2560
python3 scripts/search_2560_optimal_state.py --start-date ... --end-date ... --foundation-db ...

# VCP
python3 scripts/search_vcp_optimal_state.py --start-date ... --end-date ... --foundation-db ...

# 布林强盗
python3 scripts/search_bollinger_optimal_state.py --start-date ... --end-date ... --foundation-db ...
```

或通过 Agently runner：

```bash
python3 agently_adapter/stockpool_daily_runner.py search_vcp_optimal_state \
  --start-date 2025-06-01 --end-date 2026-05-01 \
  --foundation-db outputs/p116_foundation_20260521/p116_foundation.duckdb \
  --primary-window 20 --min-samples 30
```

#### 步骤 4：人工确认

审核要点：

- 样本量是否达到最小门槛（默认 30）。
- 超额收益和胜率是否在样本外保持一致。
- t-stat 是否达到统计显著性。
- 是否存在过拟合风险（精确组合 vs 模糊 bit 聚合对比）。
- 与直觉和策略逻辑是否一致。

#### 步骤 5：写入规则

通过人工确认后，结论可写入：

- `config/*.json` — 策略配置文件（如 `ma2560_state_market_match_rule.json`）
- `docs/STRATEGY_DEFINITIONS.md` — 策略定义文档

写入前必须在文档中标注验证日期、验证区间、样本量和关键统计数据。

### 5.2 KIMI 采用状态速查

| KIMI 内容 | 处理方式 |
|-----------|----------|
| VCP 偏收缩、趋势新生 | 采纳为研究假设 |
| 布林强盗偏扩张、趋势延展 | 采纳为研究假设 |

| KIMI 中错误 bit mask 代码 | 禁止采用 |
| KIMI 中收益/胜率数字 | 未经本地复现不得展示 |
| A 股制度过滤项 | 采纳为风险标签研究方向 |

### 5.3 当前各策略升格状态

|------|----------|----------|

## 第 6 章：合规边界

### 6.1 底座只读

State 底座的计算公式、D1 视角规则、SR 关键位计算方法、确认延迟和前向填充机制是系统契约，不可被任何下游模块或 AI Agent 修改。

DeepSeek、KIMI、Agently 等 AI 层被严格禁止：

- 修改 State 公式
- 修改 D1 视角规则
- 将模型文本当作事实
- 使用未来数据参与评分

### 6.2 策略不重写

策略信号账本只记录策略模块输出的权威信号。提醒层和研究层不得：

- 自行发明策略触发条件
- 简化或重新定义策略规则
- 混淆研究假设与已验证结论

KIMI 增强建议（流动性过滤、ATR 止损、信号分级等）不得直接写入核心信号函数，应分别落入可交易性过滤、回测层或校准层。

### 6.3 提醒不越界

提醒层只展示事实，不承担执行层职责：

**可以输出**：

```text
策略信号出现。
当前环境属于某类生命周期。
该策略与该环境的适配度为高/中/弱/待观察。
历史统计仍待校准或已由本地样本验证。
```

**不得输出**：

```text
买入 / 卖出 / 加仓 / 减仓
满仓 / 空仓
确定机会 / 必涨 / 必跌
建议操作 / 推荐执行
```

### 6.4 系统语言标准

系统对外的所有语言必须保持以下结构：

```text
1. 策略信号出现（事实层）
2. 当前 State 环境描述（环境层）
3. 策略与环境的适配度评价（适配层）
4. 历史校准状态说明（校准层）
```

不得跨越事实层直接进入行动建议。

### 6.5 完整执行链条

```text
State 底座寻找先验概率大的环境
  → 策略模块提供精确触发
  → 策略信号账本记录事实
  → 提醒层展示信号和环境
  → 执行层按策略规则处理进入、离开、调整
```

任何成交、仓位、离场、调整都必须来自独立的执行规则、持有上下文或人工记录。提醒层不得承担执行层职责。

### 6.6 A 股制度特殊边界

以下 A 股制度性因素属于风险标签和可交易性过滤层，不属于 State 底座或策略触发条件：

- T+1 制度：信号日确认与次日执行之间存在滑点和跳空风险。
- 涨停板：涨停突破可能无法成交，不能把"突破涨停"简单视作可执行入场。
- 跌停板：跌停导致退出困难，所有出场研究必须考虑不可成交情形。
- 基础过滤：ST、停牌、低成交额、低市值、极端低价应进入基础可交易过滤。
- 环境标签：板块退潮、题材一日游、业绩公告窗口、减持公告、监管事件。
