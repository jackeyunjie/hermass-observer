# 外部 AI 助手公司研究回答框架

版本：v1.0
日期：2026-05-28
状态：设计稿
定位：定义 AI 助手对外回答公司研究问题的标准结构、质量要求和合规边界

---

## 核心原则

**不是卖方报告，是研究助手。** AI 助手不生成 10000 字投资价值报告，而是针对用户的具体问题，给出有数据支撑、有来源标注、有风险提示的结构化回答。

三个不可妥协的原则：
1. **每个数字有来源**——标注数据来源、报告期、更新时间
2. **每个结论有边界**——区分事实、解释、风险，标注数据充分度
3. **每个回答有免责**——历史不代表未来，不构成投资建议

## 2. 降级模块（有条件输出）

以下模块可以有，但必须标注不确定性，不能当强结论。

### 2.1 估值参考（非估值建议）

**降级原因**：估值涉及假设，不同假设得出不同结论。

**输出方式**：
```
估值参考（仅供参考，不构成建议）：
  当前 PE(TTM)：{pe}（行业均值 {industry_pe}）
  当前 PB：{pb}
  历史分位：PE 处于近 3 年 {percentile}% 分位

  可比公司估值：
    {comparable_1}：PE {pe1}
    {comparable_2}：PE {pe2}

  注意：估值受市场情绪、流动性、宏观环境等多重因素影响，
  历史分位不代表未来走势。
```

### 2.2 券商观点汇总（非推荐）

**降级原因**：券商观点有滞后性，目标价经常不准确。

**输出方式**：
```
公开市场观点（数据来源：巨潮资讯，截至 {date}）：
  评级分布：买入 {buy} 家 / 增持 {hold} 家 / 中性 {neutral} 家
  目标价区间：{low} - {high}（{n} 家机构）
  最新研报：{institution}（{date}）评级 {rating}

  注意：券商评级和目标价基于各自分析模型，存在分歧是正常的。
  以上为公开信息汇总，不代表系统判断。
```

### 2.3 盈利预测趋势（非预测）

**降级原因**：EPS 一致预期数据稳定性有限，且我们目前缺乏完整数据。

**输出方式**：
```
盈利趋势观察（基于历史数据）：
  近 4 个季度 EPS：{q1} → {q2} → {q3} → {q4}
  趋势方向：{direction}
  营收增速：{revenue_growth}%（最新报告期）

  注意：以上为历史数据的趋势描述，不构成未来盈利预测。
  一致预期数据待补充。
```

## 3. 删除模块

以下内容不适合作为 AI 助手的常规输出：

| 删除项 | 原因 |
|--------|------|
| 最终投资建议 | 合规边界——AI 不给出买入/卖出判断 |
| 明确的买入/卖出信号 | 合规边界 |
| 合理估值区间计算 | 涉及主观假设，AI 不具备估值专业资质 |
| 强制写满的长篇报告 | 数据不足时强行凑章节会降低质量 |
| 需要大量低稳定性数据的章节 | 如详细股权结构细拆（数据更新慢、人工成本高） |

## 4. 回答合同

每次 AI 助手回答公司研究问题时，必须遵循以下结构：

### 4.1 默认回答结构

```
1. 结论摘要（1-2 句话）
   - 用最简洁的语言回答用户的问题
   - 不使用"买入/卖出/推荐"等措辞

2. 事实依据（核心数据）
   - 引用具体数字
   - 标注数据来源和报告期

3. 数据充分度
   - 充足：数据完整，结论有较强支撑
   - 部分：数据有缺口，结论需谨慎参考
   - 缺失：关键数据缺失，无法给出可靠结论

4. 风险与限制
   - 列出 2-3 个主要风险点
   - 标注数据局限性

5. 免责声明（固定格式）
   - "以上为基于公开数据的研究观察，不构成投资建议。
     历史数据不代表未来表现。投资决策应由投资者独立做出。"
```

### 4.2 快速问答卡格式

适合飞书/钉钉群对话：

```
{stock_name}（{stock_code}）

📊 主营：{main_business_short}
📈 State：{mn1}/{w1}/{d1}（ef={ef}）
🏭 行业：{sw_l1} 景气 {prosperity}/10
📉 财务：营收 {revenue}亿/{growth}% | EPS {eps} | ROE {roe}%
⚠️ 风险：{risk_1}；{risk_2}
💡 策略适配：{strategy}（{fit_level}）

以上为环境观察，不构成投资建议。
```

## 5. 质量检查对象

### 5.1 数据质量（最重要）

| 检查项 | 规则 | 自动化 |
|--------|------|--------|
| 报告期一致性 | 同一回答中所有财务数据必须来自同一报告期 | 是 |
| 指标合理性 | PE < 0 或 > 1000 标注异常 | 是 |
| 来源标注 | 每个数字必须标注数据来源 | 是 |
| 数据充分度 | 标注充足/部分/缺失 | 是 |
| iFinD vs AKShare 交叉验证 | 同一指标偏差 > 20% 时告警 | 是 |

### 5.2 逻辑质量

| 检查项 | 规则 | 自动化 |
|--------|------|--------|
| 数字引用 | 回答必须引用具体数字，不能模糊描述 | 部分 |
| 事实/解释分离 | 事实用陈述句，解释用"可能"/"预计" | 部分 |
| 缺失数据声明 | 数据不足时必须明确告知，不能用默认值 | 是 |
| 逻辑一致性 | 结论不能与引用数据矛盾 | 部分 |

### 5.3 合规质量（对外服务最重要）

| 检查项 | 规则 | 自动化 |
|--------|------|--------|
| 禁止词扫描 | 买入/卖出/推荐/建议/确定/必涨/荐股 | 是 |
| 禁止指令化 | 不输出"应该买入"/"建议持有" | 是 |
| 免责声明 | 每次回答必须包含 | 是 |
| 历史不代表未来 | 涉及统计数据时必须标注 | 是 |

---

## 6. 数据充分度分级

| 等级 | 定义 | 展示规则 |
|------|------|----------|
| 充足 | 数据完整，覆盖 ≥ 3 个报告期，来源明确 | 正常展示 |
| 部分 | 数据有缺口（如缺最新季度、缺可比公司），但核心结论可支撑 | 标注"数据部分，仅供参考" |
| 缺失 | 关键数据缺失（如无财务数据、无行业数据） | 标注"数据不足，无法给出可靠结论"，不展示具体数字 |

```

📊 主营：集成电路设计，特种芯片+智能安全芯片
📈 State：MN1=E W1=E D1=E（ef=3，三周期共振）
🏭 行业：电子 景气 8.2/10（上升）
📉 财务：营收 75.2亿/+14.9% | EPS 2.39 | ROE 18.2%
⚠️ 风险：特种芯片需求受国防预算影响；估值处于历史中位
💡 策略适配：VCP（最佳适配—趋势新生环境）

数据来源：iFinD 财务数据（2023Q3）+ State 底座（2026-05-27）
以上为环境观察，不构成投资建议。
```
