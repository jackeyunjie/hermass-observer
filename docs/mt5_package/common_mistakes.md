# 常见错误与混沌值编码统一

---

## 错误 1：position 用错收盘价

**症状**：MT5 计算的 State 与 Python 不一致，尤其是 W1 和 MN1 的 position_bit。

**根因**：用了各自周期的收盘价（如 W1 position 用 W1 close），而不是视角基准价。

**正确做法**：
- D1 视角：所有周期的 position 用 D1 close
- W1 视角：所有周期的 position 用 W1 close
- H1 视角：所有周期的 position 用 H1 close

**验证方法**：用 `validation_samples.csv` 中的样本逐行对比。

---

## 错误 2：ADX 参数不一致

**症状**：trend_bit 与 Python 不一致。

**根因**：MT5 的 iADX 默认周期可能是 14，但 slope 计算窗口可能不同。

**正确参数**：
- ADX 周期：14（不可变）
- slope 窗口：3（用前 3 根 bar 的 ADX 计算斜率）
- trend_bit=1 条件：ADX 未处于"封闭"状态

---

## 错误 3：布林带宽分位计算错误

**症状**：base 位（0 或 8）与 Python 不一致。

**根因**：分位计算的窗口长度或方法不同。

**正确参数**：
- 布林带：20 期，2 倍标准差
- 带宽 = (上轨 - 下轨) / 中轨
- 分位窗口：20 期
- base=0（收缩）条件：当前带宽位于历史分位的低 20% 以下

---

## 错误 4：SR 确认延迟不对

**症状**：position_bit 在某些日期与 Python 不一致。

**根因**：分形确认的延迟根数不同。

**正确参数**：
- 分形周期：k=5（左右各 5 根 bar）
- 确认延迟：3 根 bar（分形形成后等 3 根确认）
- 前向填充：确认后的 SR 向后填充直到新的确认

---

## 错误 5：混合 D1 视角和 W1 视角

**症状**：日线图上的 W1 State 与周线图上的 W1 State 不同。

**根因**：这是正确行为，不是错误。日线图用 D1 close，周线图用 W1 close。

**正确理解**：
- 日线图 W1 State = "日线价格在周线结构中的位置"
- 周线图 W1 State = "周线自身的趋势和位置"
- 两者回答的问题不同，不能混用

---

## 混沌值编码统一方案

如果对方之前使用"混沌值"系统，以下是与 State 系统的对应关系：

| 混沌值概念 | State 系统对应 | 关键差异 |
|-----------|---------------|----------|
| 混沌值 = 综合评分 | State score = base+trend×4+pos×2+vol | State 是 4-bit 编码，不是连续值 |
| 混沌值高 = 强 | E(14)/F(15) = 最强 | 混沌值可能是连续 0-100，State 是离散 0-15 |
| 多周期混沌值 | MN1/W1/D1 State | State 有统一的 4-bit 编码，混沌值可能各周期编码不同 |
| 视角概念 | D1 视角天条 | 混沌值系统可能没有"视角"概念 |

**统一方案**：
1. 废弃混沌值编码，全部切换为 State 4-bit 编码
2. 历史数据按 State 公式重新计算
3. 11 个不可变参数以本文档为准

---

## 不可变参数清单

| 参数 | 值 | 用途 |
|------|-----|------|
| BB 周期 | 20 | 布林带计算 |
| BB 标准差倍数 | 2 | 布林带宽度 |
| BB 分位窗口 | 20 | 带宽分位计算 |
| ATR 周期 | 14 | 波动率计算 |
| ADX 周期 | 14 | 趋势判定 |
| ADX slope 窗口 | 3 | 趋势斜率 |
| 分形周期 k | 5 | SR 关键位识别 |
| 确认延迟 | 3 | SR 确认 |
| 编码位数 | 4 | State 编码 |
| base 值 | 0 或 8 | 收缩/扩张 |
| D1 视角 | 强制 | 所有周期 position 用 D1 close |
