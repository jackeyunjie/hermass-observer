## 1. 系统定位

为交易人提供板块、子板块、领头羊、核心个股的结构化研究观察。核心能力：

- AI 科技板块拆解为 8 个子板块
- 每只个股分配到：领头羊、核心股、弹性股、掉队股
- 2560 三层串行过滤：长周期框架 → 趋势方向盘 → 量能扳机
- MA144/169/200 收敛识别 + SR 区间收缩识别
- W1/D1/M30 三周期方向矩阵
- **输出必须是研究观察，不得输出买卖建议**

## 3. 指标系统

### 3.1 三层指标架构

| 层级 | 指标 | 回答的问题 | 信号权重 |
|------|------|-----------|----------|
| 长周期框架 | MA144 / MA169 / MA200 排列 + 收敛 | 大环境允不允许做多 | 一票否决（空排不观察） |
| 趋势方向盘 | MA25 斜率 + close vs MA25 | 当前方向对不对 | 必要条件（线下不观察） |
| 量能扳机 | VMA5 vs VMA60 关系 | 资金跟不跟 | 观察确认（无量不记录） |

**重要**：2560 战法的「60」指 **60 日均量线（VMA60）**，不是价格均线。MA60 若作为价格均线使用，属于本项目自身扩展，不属于 2560 原教旨。

### 3.2 核心指标定义

| 指标 | 计算方式 | 用途 |
|------|----------|------|
| MA25 | 25 日简单移动平均（收盘价） | 趋势方向 |
| MA144 | 144 日简单移动平均（收盘价） | 长周期框架 |
| MA169 | 169 日简单移动平均（收盘价） | 长周期框架 |
| MA200 | 200 日简单移动平均（收盘价） | 长周期框架 |
| VMA5 | 5 日均量线 | 短期资金敏锐度 |
| VMA60 | 60 日均量线 | 中期量能基准线 |
| ATR14 | 14 日平均真实波幅 | SR 区宽自适应基准 |

### 3.3 派生指标

| 派生指标 | 计算方式 |
|----------|----------|
| MA144/169/200 带宽 | `(max - min) / close` |
| MA144/169/200 排序 | 多头排列 / 缠绕 / 空头排列 / 混乱 |
| MA25 斜率 | MA25 近 5 日变化率 |
| 价格-MA25 位置 | `(close - MA25) / MA25` |
| VMA5-VMA60 位置 | 上穿 / 粘合 / 下穿 |
| SR 区间宽度 | `(阻力上沿 - 支撑下沿) / close` |

## 4. 收缩规则

### 4.1 三类收缩

| 收缩类型 | 定义 | 方法 | 参数 |
|----------|------|------|------|
| MA144/169/200 收敛 | 三条线两两间距 < 阈值，持续 ≥N 日 | 固定阈值 | 2%, 3 日 |
| 量能收缩 | VMA5 < VMA60 且 VMA5 连续下行 | 序贯判断 | ≥5 日递减 |
| SR 区间收缩 | 区间宽度压缩至历史中位的显著比例 | 比率自适应 | 20 日宽 / 60 日中位宽 < 0.6 |

### 4.2 设计依据

- **长周期均线收敛**：长周期均线本身变化缓慢，固定阈值（2%）已足够稳健。使用波动率自适应反而会引入噪声。
- **量能收缩**：2560 原教旨看的是成交量阶梯式萎缩的**过程**，不是绝对值。用序贯判断（连续 5 日 VMA5 下行且 < VMA60）最能还原"缩量→冲量→作量"的节奏。
- **SR 区间收缩**：不同价位股票的 SR 绝对宽度不可比，必须用比率（当前宽 / 历史中位宽）做自适应。

### 4.3 收缩与 VCP 的关系

VCP（波动率收缩模式）的核心思想是收缩的**序贯性**——不是一次收缩，而是 2-3 次递减收缩。本系统中：

- 单次收缩 → 标记"收缩中"，不作为强信号
- 连续 2 段收缩递减 → 标记"VCP 结构"，提升关注度
- 需至少 W1+D1 双周期确认，不做单周期判断

## 5. 三层串行过滤规则

### 5.1 过滤流程

```
长周期框架层
  MA144/169/200 是否多头排列 or 收缩中？
  ├── 否 → 不纳入观察清单
  └── 是 → 进入趋势层
            │
趋势方向盘层
  close > MA25 且 MA25 斜率 ≥ 0？
  ├── 否 → 标记"掉队"，仅作反向对照
  └── 是 → 进入量能层
            │
量能扳机层
  VMA5 > VMA60 且 当日收阳？
  ├── VMA5 > VMA60 + 收阳 → 纳入核心观察
  ├── VMA5 < VMA60 + 收阳 → 标记"缩量观察"
  └── VMA5 > VMA60 + 收阴（放量大阴线）→ 一票否决，诱多陷阱
```

### 5.2 关键规则

- 三层必须串行过滤，不做交叉组合信号（如"MA25 金叉 MA144"之类属于过度拟合）
- VMA5 上穿 VMA60 **必须**配合当日阳线确认；放量阴线视为诱多，信号作废
- 次日 K 线若未突破前一日高点，信号降权

## 6. SR 支撑阻力系统

### 6.1 SR 区域定义

SR 不使用单一点位，使用**区域**：

| 区域 | 定义 | 宽度 |
|------|------|------|
| 支撑区 | 下沿 = 最近有效低点 - 0.5×ATR | 1×ATR |
|  | 上沿 = 最近有效低点 + 0.5×ATR | |
| 阻力区 | 下沿 = 最近有效高点 - 0.5×ATR | 1×ATR |
|  | 上沿 = 最近有效高点 + 0.5×ATR | |

### 6.2 SR 有效性量化

| 维度 | 量化标准 | 权重 |
|------|----------|------|
| 触碰次数 | ≥3 次 → 有效；1-2 次 → 待确认 | 高 |
| 成交量配合 | 触碰时量能萎缩 → 支撑稳固；放量 → 压力大 | 高 |
| 时间衰减 | 最近触碰 > 60 日前 → 降权 50% | 中 |
| 突破确认 | 突破后次日收盘未回归 → SR 转换确认 | 高 |

## 7. 多周期共振/冲突

### 7.1 方向矩阵

三个周期的观察维度：

| 周期 | 观察内容 | 权重 |
|------|----------|------|
| D1 | 三层串行过滤 + MA144/169/200 状态 + SR 位置 | 主判断 |
| W1 | 宏观框架确认 + 长周期 MA 状态 | 宏观框架 |
| M30 | 日内结构细节 | 仅精度补充，不独立拍板 |

### 7.2 共振强度

| 共振情况 | 含义 | 处理 |
|----------|------|------|
| 3/3 同向 | 强共振 | 标记"高可信" |
| 2/3 同向 | 一般共振 | 标记"偏强/偏弱"，待第三个确认 |
| 1/3 或 0/3 | 无共振 | 不做方向判断，标记"待观察" |

### 7.3 冲突处理规则

- D1 方向优先级最高：W1/M30 与 D1 冲突时，以 D1 为准
- W1/M30 只做 D1 框架内的精度补充
- 收缩信号需 W1 + D1 双确认；仅 D1 收缩标记"待确认"

## 9. 风险提示规则

### 9.1 最容易误导的规则（已降权或改写）

| 原始规则 | 问题 | 本系统处理 |
|----------|------|-----------|
| "VMA5 金叉 VMA60 即入场" | 缺阳线确认，可能遇放量大阴线 | 必须绑阳线确认 + 次日突破前高 |
| "MA 金叉/死叉做方向" | 滞后严重，噪声多 | 降为背景色，不作为独立触发 |
| "单周期收缩即判断" | 日线收缩可能只是横盘 | 至少 W1+D1 双确认 |
| "SR 突破即方向确认" | 假突破常见 | 需次日收盘确认 + 量能配合 |
| 只输出做多方向 | 交易人会默认"看好" | 始终输出双向 SR 区间 + 失效条件 |

### 9.2 一票否决规则

以下任一触发，该个股排除出观察清单：

1. MA144/169/200 空头排列
2. close < MA25 且 MA25 向下
3. VMA5 上穿 VMA60 当日为放量大阴线
4. 属于 ST / 上市不足半年 / 股价 < 5 元
5. 近 20 日平均换手率 < 1%

## 10. 最小可行版本实现路径

### 10.1 数据依赖

| 数据 | 来源 | 周期 |
|------|------|------|
| OHLCV 日线 | p116_foundation.duckdb / 黑狼 API | D1 |
| OHLCV 周线 | p116_foundation.duckdb / 黑狼 API | W1 |

| 板块分类映射 | 本地 JSON 配置 | 静态+定期更新 |
| 个股基础信息 | p116_foundation.duckdb | 日更新 |

### 10.2 计算模块

```
1. _compute_ma           → MA25/MA144/MA169/MA200
2. _compute_vma          → VMA5/VMA60
3. _classify_ma_state    → 多头/空头/缠绕/混乱
4. _detect_ma_convergence → MA144/169/200 是否收敛
5. _detect_volume_contraction → VMA 是否阶梯缩量
6. _compute_sr_zones     → SR 支撑/阻力区域
7. _compute_sr_contraction → SR 区间是否收缩
8. _three_layer_filter   → 串行过滤 → 角色分配
9. _multi_tf_matrix      → W1/D1/M30 方向矩阵
10. _format_output       → 结构化文本输出
```

### 10.3 验收标准

任取一只 AI 科技板块股票 + 一个日期：

1. 正确输出子板块归属 + 个股角色
2. 三层串行过滤结果正确
3. 收缩判断（均线/量能/SR）对应正确
4. 多周期矩阵与 W1/D1/M30 实际数据一致
5. 输出中不含任何买卖建议措辞
