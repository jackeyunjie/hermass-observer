## 版本概述

Hermass Observer 是一套面向 A 股市场的多周期 State 策略研究系统。系统的核心思想是：**先识别市场环境，再评估策略适配度，用数据验证替代主观判断**。

V1.0 是系统 MVP 的首个正式版本。五层架构（数据层、State 底座、策略引擎、验证体系、展示层）已全部搭建完成，三个经典策略（MA2560、VCP、布林强盗）已接入信号账本并完成初步校验。

**当前状态**：系统已完成代码建设和初步验证，等待 5 月 28 日首次校准触发后进入正式运行。

**回溯数据覆盖**：2025-06-01 至今，约 11 个月、223 个交易日。

## 核心能力清单

### 1. State 底座（Layer 2）— 已上线

State 底座是系统的"只读地基"，负责将原始行情数据转化为标准化的多周期市场状态描述。

| 能力 | 说明 |
|------|------|
| P116 State 计算 | 对每只股票生成 MN1（月线）、W1（周线）、D1（日线）三层 State 值 |
| E/F 分类 | E=14、F=15，定义位置优先的符号裁决规则 |
| SR 边界计算 | 支撑/阻力位量化输出 |
| State 缓存 | 每日生成 state_cache JSON，供下游策略和验证模块消费 |
| 市场阶段识别 | 基于 State 分布判定当前市场所处阶段 |

**关键数据**：State 缓存已累计生成 1,423 个文件，覆盖 223 个交易日。

**设计原则**：底座只读，所有下游模块只消费 Layer 2 输出，不写回。

---

### 2. 策略引擎（Layer 3）— 已上线

策略引擎将经典趋势策略转化为可复现的信号模块，并为每个信号附加环境适配度评分。

| 策略 | 信号类型 | 验证状态 |
|------|----------|----------|
| **MA2560** | golden_cross / strong_hold / aligned / death_cross_exit | 部分验证通过（partially_validated） |
| **VCP** | breakout / breakout_no_vol / breakout_weak_vol | 路径假设获数据支持（path_hypothesis_supported） |
| **布林强盗** | bb_bandit_long_entry + 递减均线出场 | KIMI 候选被本地数据拒绝，D1 volatility_bit=0 更稳定 |

**附加能力**：

- **策略信号账本**：每日汇总三策略信号，记录适配度等级（最佳适配 / 适配 / 弱适配 / 不适配 / 未知）
- **宏观-产业链先验**：骨架已上线，当前为单分数输出，置信度 0.25（待宏观四维数据接入后升级）
- **W1×MN1 环境标签**：为每个信号标注大周期环境背景

### 3. 验证体系（Layer 4）— 待校准

验证体系是系统"用数据说话"的核心机制。通过前向观察账本积累信号的未来收益标签，在样本充足时触发校准，检验适配度排序是否与历史收益方向一致。

| 组件 | 功能 | 状态 |
|------|------|------|
| 前向观察账本 | 每日记录信号 + 5/10/20 日收益标签 | 已运行，当前 6 个文件 |
| 适配度观察持久化 | 将适配度评分散列到 fit_log.duckdb | 已上线 |
| 校准触发机制 | 三重门检查（时间门槛 5 天 + 样本门槛 100 条 + 变化门槛 0.10） | 已实现，等待首次正式触发 |
| Bootstrap CI | 置信区间计算，验证统计显著性 | 已实现 |
| State 组合稳定性验证 | 跨期验证 State 组合是否过拟合 | 已实现 |
| 策略环境验证编排器 | 自动化校准流程编排 | 已上线 |

## 关键验证数据

### MA2560 策略验证

- **Top20 历史回测样本**：平均收益 +13.7%（最高 +19.1%）
- **适合组合特征**：VOL5 > VOL60（20/20）、低点触及 MA25 2% 区间（20/20）、MA25 上行（17/20）
- **State 匹配规则**：E/E/F、E/F/F、E/F/E 三种组合
- **市场支持**：30 条推荐样本中 24 条有行业 ETF ef_count >= 2 的市场支持

### VCP 策略验证

- **已标注样本**：43,259 条
- **D1 压缩释放路径（近 5 日）**：命中 1,473 条，20 日平均超额 +1.66%（95% CI: [0.87%, 2.49%]），t-stat = 3.98
- **D1 压缩释放路径（近 10 日）**：命中 2,637 条，20 日平均超额 +1.62%（95% CI: [1.00%, 2.42%]），t-stat = 5.08
- **非路径样本**：20 日平均超额仅 +0.60%，路径条件有效区分了信号质量

### 布林强盗验证

- **已标注样本**：47,219 条
- **KIMI 候选 State 组合**：被本地数据拒绝，未通过验证
- **发现**：D1 volatility_bit=0 比 KIMI 候选更稳定，待进一步优化

## 当前局限

V1.0 是 MVP 版本，以下局限需在后续版本中逐步解决：

| 局限 | 影响 | 预计解决 |
|------|------|----------|
| 宏观数据仅骨架 | 宏观先验置信度低（0.25），四维评分不可用 | Phase 1 数据补全 |
| 产业链覆盖有限 | 仅 Top 3 产业链有景气度评分 | Phase 1-2 |
| 前向观察样本不足 | 校准仅 4 条样本，统计置信度有限 | 5 月 28 日后持续积累 |
| 校准为手动触发 | 需人工执行校准脚本 | Phase 3 自动化 |
| 布林强盗未找到有效 State 组合 | 该策略暂不输出环境匹配信号 | 需进一步研究 |
| 单机本地部署 | 不支持远程访问和自动推送 | Phase 3 |
| ATR 吊灯未接入账本 | 需要持仓上下文，当前架构不支持 | Phase 3 |

## 使用方式

### 环境要求

- Python 3.14+
- DuckDB 1.3+
- 依赖包：numpy、pandas、PyYAML、requests、AKShare

### 日常运行流程

```text
1. 数据更新：python scripts/download_daily.py        # 拉取最新 A 股日线
2. State 计算：python scripts/build_p116_foundation.py # 生成 State 底座
3. 信号生成：python scripts/strategy_signal_ledger.py  # 输出三策略信号
4. 查看报告：浏览器打开 public/ 目录下的 HTML 文件
```

### 关键输出目录

| 目录 | 内容 |
|------|------|
| `outputs/p116_foundation_*/` | State 底座数据库（每日一个目录） |
| `outputs/strategy_signals/` | 策略信号账本 JSON |
| `outputs/forward_observation/` | 前向观察记录 |
| `outputs/calibration/` | 校准报告 |
| `public/` | HTML 报告（本地 HTTP 服务） |
