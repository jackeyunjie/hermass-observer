## 已知限制（本轮不验收）

1. **事件类问题会降级**  
   例如“美国下跌影响”可能返回通用框架化解释。这是已知限制，本轮只验收是否有明确披露，不验收实时事件归因准确性。

2. **反馈 API 直调需要完整字段**  
   浏览器表单可用。PM 本轮只走浏览器反馈页，不需要手写 `curl` 调接口。

3. **交易建议边界**  
   系统不会给出买入、卖出、加仓、减仓、止盈、止损、荐股、目标价、收益承诺。PM 如期待此类输出，属于产品边界设计，不是缺陷。

4. **000021 研究页“证据不够”**  
   这是正常状态，不是误判。

## 禁止验收为功能缺陷的内容

以下四类内容如果出现，请先对照本说明确认，不要直接记为 P0：

| 类别 | 说明 |
| --- | --- |
| 交易建议 / 买卖点 / 目标价 / 止损价 | 系统 intentionally 不提供。若 PM 认为必须提供，请先走产品决策，不要当 bug 验收。 |
| 事件类实时归因 | 事件类问题降级为框架化解释是已知限制。 |
| 手写 API 缺字段 | PM 只验收浏览器表单，curl 缺字段属于使用方式问题。 |
| 000021 证据不够 | 单标的无足够证据时的正常输出。 |

## 反馈分级标准

| 级别 | 定义 | 处理时限 | 示例 |
| --- | --- | --- | --- |
| **P0 阻塞** | 主路径不可用、页面 500、数据日期错误、严重误导为交易建议 | 当天处理 | 首页打不开；研究页 500；市场页数据不是 2026-07-01；页面出现买入/卖出/目标价 |
| **P1 高优先级** | 主路径能用但明显影响理解或闭环 | 1-2 天内处理 | 首页不知道下一步；研究页结论看不懂；观象不给下一步入口；观察账本任务状态混乱 |
| **P2 优化** | 不影响主路径，但影响体验或表达 | 排期处理 | 按钮文案不顺；卡片信息密度偏高；某个说明可以更短 |
| **Known 已知限制** | 已在本说明中披露，本轮不作为缺陷 | 记录，不阻塞 | 事件类问题降级；手写反馈 API 缺字段；000021 证据不够；系统不给买卖点 |

## P0 判定清单

出现任一项，请直接标记 **P0**：

1. 公网页面返回非 200：`/`、`/market`、`/recommend`、`/research?stock_code=000021.SZ`、`/watchlist`、`/playbook`、`/feedback`。
2. 页面出现 `Internal Server Error`、模板错误、空白页。
3. 页面数据日期不是 2026-07-01，且没有明确说明最新有效交易日。
4. 观象增强入口 `/api/chat/query` 未认证不返回 401。
5. 页面或观象出现买入、卖出、加仓、减仓、止盈、止损、荐股、收益承诺等表达。
6. 研究页把“证据不够”错误包装成明确正向结论。
7. 候选股票无法跳转到研究页。
