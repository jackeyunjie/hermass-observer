## 1. 产品定位

Hermass 首页从「新闻门户 + 多入口导航页」收敛为 **「我的观察台」**（产品内部代号：转折雷达）。

> 以用户持仓、自选和候选标的为核心，持续观察 3D / 3W / 3M / 6M 多时间窗内的关键转折概率、证据与风险。

系统只描述状态和证据，不输出交易动作。

### 核心角色

- **我的观察台**：用户标的的转折雷达。
- **多周期概率系统**：3D / 3W / 3M / 6M 的状态变化与概率表达。
- **经典策略信号哨兵**：只在触发时亮灯，不参与主系统判断。
- **证据与风险解释层**：Agent / 结构 / 行业 / 风险证据，只解释，不建议。

### 一句话价值

> 我的持仓、自选、候选标的，在未来 3 天、3 周、3 个月、6 个月对应的结构里，是否正在接近或已经发生关键转折？

## 2. 首页信息架构

首页第一屏采用「输入 → 我的标的 → 时间窗矩阵 → 证据 → 参考池」的纵向信息流。

### 2.1 顶部观象指令栏

- 股票代码快速查询（如 `000021`、`688107`）。
- State 条件查询（如 `D1 zero`、`W1 转强`、`EF>=2`）。
- 自然语言观察问题（如「我的自选池 3 周变化」）。
- 快捷 chip：`我的自选`、`今日转折`、`全市场 Top`、`状态说明`。

### 2.2 我的标的转折雷达

- 默认展示用户自选池 / 持仓池。
- 无持仓数据时，使用自选池 + 全市场转折 Top 作为补位。
- 每行展示：标的、最新状态摘要、3D/3W/3M/6M 关键变化、证据条数、风险标记。
- 点击行进入单标的研究页 `/stock-research?code=xxx`。

### 2.3 多时间窗矩阵

对雷达中选中的标的（默认第一只），展示：

| 时间窗 | 观察含义 | 展示字段 |
|---|---|---|
| 3D | 短期变化 | 当前状态、较 3 日前变化、关键证据 |
| 3W | 结构形成 | 当前状态、较 3 周前变化、关键证据 |
| 3M | 趋势延续/失效 | 当前状态、较 3 个月前变化、关键证据 |
| 6M | 长期迁移 | 当前状态、较 6 个月前变化、关键证据 |

每个单元格输出：
- 转折类型：转强早期 / 确认转强 / 强势延续 / 转弱预警 / 确认转弱 / 结构未破坏 / 证据不足。
- 关键证据：哪个周期变化、EF / A+B / 0 / 关键位、行业共振、市场环境。
- 风险标记：假突破风险 / 数据异常 / 无。

### 2.4 经典策略信号灯

- 只在某经典策略触发时显示小标签（如 `VCP 信号`、`2560 触发`）。
- 平时不显示、不占固定区域。
- 不解释与 State 的同向 / 冲突 / 领先关系。
- 点击进入策略详情页 `/strategy-detail?code=xxx&strategy=vcp`。

### 2.5 全市场转折 Top

- 作为参考池，折叠或置于第二屏。
- 不抢「我的标的」主线。
- 展示：标的、时间窗、转折类型、证据摘要。

### 2.6 系统健康与数据新鲜度

- 数据日期（如 `2026-07-02`）。
- State Cube / State Timeline 更新状态。
- Agent 复盘状态。
- 数据同步状态（market_assets_state、forward_observation 等）。

## 6. 允许文案 / 禁止文案

### 6.1 禁止作为系统结论出现

- 买入
- 卖出
- 加仓
- 减仓
- 清仓
- 空仓
- 加杠杆
- 止盈
- 止损
- 目标价
- 收益承诺
- 任何直接交易动作建议

### 6.2 允许表达

- 转强早期
- 确认转强
- 强势延续
- 转弱预警
- 确认转弱
- 结构未破坏
- 未进入候选
- 证据不足
- 假突破风险
- 数据异常
- 转折概率上升 / 下降
- 风险增加 / 风险释放
- 关键位附近
- 多周期共振 / 背离

### 6.3 文案原则

> 系统描述「正在发生什么」，不替用户决定「应该怎么交易」。

概率类文案示例：
- 「3D 转强概率较高，关键证据来自 D1 state 变化。」
- 「3W 延续概率下降，存在假突破风险。」
- 「6M 证据不足，建议持续观察。」

## 7. 与现有页面的关系

### 7.1 `/state-observer`（State Workbench）

- **定位**：完整 State 长表与多日对比的数据视图。
- **与首页关系**：首页「状态」导航入口；用户想查看完整 State 矩阵时进入。
- **不做**：首页不替代 State Workbench，只展示浓缩的转折雷达。

### 7.2 `/research` 与 `/stock-research`（研究页）

- **定位**：单标的研究、产业链、新闻解释、Agent 证据。
- **与首页关系**：首页点击标的进入 `/stock-research?code=xxx`；研究页承载深度证据。
- **不做**：首页不展开单标的全部证据。

### 7.3 `/mystrategies`、`/strategy-editor`、`/2560-sr-observation`（经典策略页）

- **定位**：经典策略信号、规则说明、历史表现。
- **与首页关系**：首页仅展示策略触发小标签；点击进入策略详情页查看买点、失效条件、仓位规则。
- **不做**：策略信号不与 State 主概率融合，不在首页给出综合交易结论。

### 7.4 `/watchlist` 与 `/tasks`（观察账本 / 任务）

- **定位**：用户观察任务、持仓 / 自选标的、复盘记录。
- **与首页关系**：首页「我的标的转折雷达」默认读取 watchlist / user_tasks；创建观察按钮写入 `user_task_ledger.json`。
- **不做**：首页不替代 watchlist 的完整任务管理功能。

## 8. Phase 1 实施范围

Phase 1 只做产品收敛与首页原型，**不写概率引擎、不新增数据库、不改动后端核心逻辑**。

### 8.1 必做

1. 确定首页名称：「我的观察台」。
2. 收敛顶层导航为：观察、状态、研究、策略。
3. 改造 `web/templates/index.html` 首页模块：
   - 观象指令栏。
   - 我的标的转折雷达。
   - 多时间窗矩阵（3D/3W/3M/6M 状态变化）。
   - 经典策略信号灯（仅展示触发标签）。
   - 全市场转折 Top（可折叠）。
   - 系统健康与数据新鲜度。
4. 更新 `web/templates/_top_nav.html`：主导航 + 二级菜单。
5. 在 `web/main.py` 中为首页补充/复用现有数据聚合函数（不新增持久化）。
6. 制定并落地禁用词清单，首页文案走查。

### 8.2 可选

1. 首页路由保持 `/`，或新增别名 `/observation-deck` 重定向到 `/`。
2. 增加「我的自选」与「持仓」导入入口（仅前端占位，后端复用 user_tasks）。

### 8.3 明确不做

1. 不实现 Empirical Bayesian 概率引擎。
2. 不新增 DuckDB 表或字段。
3. 不改动 State Cube / State Timeline 生成逻辑。
4. 不让经典策略信号参与 State 主概率。
5. 不输出任何交易动作建议。
6. 不做实时推送 / 邮件订阅。

## 10. 风险与验收标准

### 10.1 风险

| 风险 | 说明 | 缓解 |
|---|---|---|
| 交易动作语言泄露 | 产品或开发人员在文案中误用「买入/卖出/止损」等词 | 禁用词清单 + 文案走查 + 自动化扫描 |
| 经典策略与 State 混淆 | 策略信号被用户误读为系统主结论 | 策略标签只做展示，详情页独立，文案不解释同向/冲突 |
| 数据新鲜度影响体验 | 若 State Timeline / State Cube 未更新，首页展示旧数据 | 系统健康模块明确显示数据日期和更新状态 |
| 概率字段过早泄露 | Phase 1 还没概率引擎，不能标具体概率数值 | Phase 1 用转折类型（转强早期等）替代数值概率 |
| 页面过度设计 | 首页承载过多模块，失去主轴 | 严格按「我的标的 → 时间窗矩阵 → 证据」分层 |

### 10.2 验收标准

1. 首页标题显示「我的观察台」或「Hermass — 我的观察台」。
2. 顶层导航只有 4 个主项：观察、状态、研究、策略。
3. 首页第一屏包含「观象指令栏」「我的标的转折雷达」「多时间窗矩阵」。
4. 首页无任何禁用词作为系统结论出现。
5. 经典策略信号只在触发时出现小标签，平时不显示。
6. 点击雷达行可跳转到 `/stock-research?code=xxx`。
7. 系统健康栏显示数据日期、State Cube 状态、Agent 复盘状态。
8. Phase 1 不改后端生产数据库，不新增 API 持久化。
