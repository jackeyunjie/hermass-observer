# KIMI2 任务：经典策略信号哨兵方案

日期：2026-07-02
执行者：KIMI2
任务类型：策略 Agent 边界 / 信号契约 / 页面关系

---

## 任务背景

经典策略 Agent 的定位已经明确：

> 经典策略 Agent 是安静的信号哨兵。

它不是 Hermass 主策略，不参与 State 概率计算，不和 State 输出混合结论。

平时静默；只有某个经典策略明确触发时，显示一个小标签，例如：

- `VCP 信号`
- `CANSLIM 信号`
- `趋势跟随信号`
- `均值回归信号`

---

## 你的目标

设计经典策略信号哨兵的独立契约。

回答：

1. 哪些经典策略适合第一批做哨兵。
2. 每个策略触发时输出哪些字段。
3. 首页只显示什么。
4. 点击后进入详情页显示什么。
5. 如何避免和 Hermass State 主系统混淆。

---

## 必须遵守的边界

经典策略信号层：

- 不解释 State。
- 不参与 State 概率。
- 不输出同向 / 领先 / 冲突 / 证据不足。
- 不在首页占固定区域。
- 不生成综合交易结论。
- 不影响“我的观察台”的转折概率。

首页只显示小标签：

```text
VCP 信号
CANSLIM 信号
趋势跟随信号
均值回归信号
```

没有信号就不显示。

## 请重点设计

### 1. 第一批策略清单

建议评估：

- VCP
- CANSLIM
- 趋势跟随
- 均值回归
- 突破策略
- 动量 / 相对强弱

请给出第一批建议，不要一次做太多。

### 2. 信号契约

建议字段：

```text
stock_code
state_date
strategy_name
signal_name
signal_level
triggered
trigger_price
invalid_condition
position_rule_text
evidence_items
source
```

注意：这些是经典策略自己的原始规则展示，不是 Hermass 建议。

### 3. 首页展示

首页只显示：

- 小标签。
- 可点击进入详情。
- 不显示买卖动作。
- 不显示仓位建议。

### 4. 详情页展示

详情页可以展示经典策略原始语义，但必须明确：

> 以下为经典策略规则触发说明，仅作研究观察，不构成交易建议。
