# 经典策略定义与系统边界

本文档记录当前系统重点跟踪的经典趋势策略定义，供策略信号模块、策略信号账本、研究 Agent 和人工复核引用。

核心边界：

- State 底座负责识别先验概率更高的市场环境。
- 策略模块负责产生精确、可复现的策略信号。
- 策略信号账本只记录策略模块输出的权威信号。
- 提醒层只消费账本和环境标签，不重新计算、不简化、不发明策略触发条件。
- 本文档不是交易建议，也不是提醒层输出话术。

## 1. 布林强盗

### 核心思想

布林强盗把布林带上轨从传统的阻力参考转化为趋势突破确认位，用于捕捉价格突破后引发的趋势性行情。

### 适用周期

中长线，日线及以上周期。

### 核心参数

- 布林带中轨均线周期：`50`
- 标准差倍数：`1`
- 趋势过滤周期：`30`

### 信号规则

做多 entry 信号需要同时满足：

1. 当日收盘价大于 `50日均线 + 1个标准差`，即突破布林上轨。
2. 当日收盘价大于过去 `30` 日最高收盘价，确认处于上升趋势。

满足条件后，策略模块可在下一交易日生成正式 entry 信号。

### 离场规则

采用自适应均线机制。离场条件需要同时满足：

1. 当日收盘价低于自适应均线。
2. 自适应均线低于布林上轨。

自适应均线设定：

- 刚进入持有状态时，使用 `50日均线`。
- 持有每增加一天，均线周期减 `1`。
- 最低递减至 `10日均线`。

含义：持有时间越长，防守线越灵敏。

### 系统映射

- `strategy_id`: `bollinger_bandit`
- 正式 entry 信号可进入提醒层。
- exit/risk 信号只有在策略模块完整实现且有明确持有上下文时才进入账本。

## 2. VCP 波动收缩形态

### 核心思想

VCP 捕捉上涨途中的休整。价格和波动率同步收缩，成交量逐步枯竭，随后价格突破支点时形成趋势延续或趋势启动信号。

### 趋势背景

理想的 VCP 前提：

- 价格高于上升中的 `50日均线` 和 `200日均线`。
- `50日均线` 位于 `200日均线` 之上。
- 股价距离 `52周高点` 在约 `25%` 以内。

### 形态识别

理想结构包含 `2-4` 次收缩：

- 每次回调幅度逐步减小，例如 `25% -> 15% -> 8%`。
- 收缩区间成交量逐步萎缩，体现筹码锁定。
- 突破时成交量需要显著放大。

### 信号规则

价格突破最近一次摆动高点，即支点，策略模块可生成正式 entry 信号。

### 风险控制参考

策略模块或交易执行层可参考：

- 突破前最后一次收缩低点下方。
- 或 `2 * ATR(20)`。
- 或最大约 `10%` 的硬性风险边界。

这些内容属于策略或执行层规则，提醒层不自行计算或展示为操作建议。

### 系统映射

- `strategy_id`: `vcp`
- 完成突破确认时，`signal_type`: `entry`。
- 形态识别中但尚未突破时，可记录为 `structure` 且 `research_only=true`。

## 3. 2560 战法

### 核心思想

2560 结合趋势与成交量，寻找趋势股回到关键均线附近后重新确认的机会。

### 核心参数

- `25日均线`：趋势生命线。
- `5日均量线`：短期成交动能。
- `60日均量线`：中期成交动能。

### 信号规则

做多 entry 信号需要同时满足：

1. `25日均线` 坚定向上。
2. 股价回踩或跌破后重新站上 `25日均线`。
3. `5日均量线` 位于 `60日均量线` 之上。

可作为加分的形态：

- 触发 K 线为缩量小星线。
- 出现极度缩量的坑量结构。

### 放弃或防守规则

策略定义中的关键约束：

- 股价远离 `25日均线` 后第一次回踩质量最高。
- 多次回踩后，信号质量下降。
- 若后续跌破 `25日均线`，策略模块可产生 exit 或 risk 信号。

### 成交量结构

- 冲量：`5日均量线` 刚上穿 `60日均量线`，短线稳定性不足。
- 做量：`5日均量线` 反复接近 `60日均量线` 但不下穿，波段结构较稳。
- 缩量：`5日均量线` 长期位于 `60日均量线` 上方后出现地量，通常代表筹码稳定。

### 系统映射

- `strategy_id`: `ma2560`
- 正式 entry 信号进入提醒层。
- 结构性观察可记录为 `structure` 且 `research_only=true`。
- exit/risk 信号必须由策略模块按完整规则输出，提醒层不推断。

### P116 State 与市场匹配口径

截至 2026-05-22，2560 与 P116 state 的本地复现结论为“有条件成立”。规则文档见：

- `docs/MA2560_STATE_MARKET_MATCH_RULE.md`
- `config/ma2560_state_market_match_rule.json`
- `outputs/project/2560_state_market_match.md`

当前可进入系统规则层的组合是：

```text
VOL5 > VOL60
AND 买入日低点触及 MA25 2% 区间
AND (收盘在 MA25 2% 区间 OR 强势评分 >= 6)
AND 止跌代理成立
```

与正式 P116 state 对齐后，`ma2560_strong_hold` 可先限定在：

```text
{E/E/F, E/F/F, E/F/E}
```

市场匹配层只在行业 ETF `ef_count >= 2` 时认定为完整匹配。行业 ETF 支持字段按以下顺序取得：

1. 优先读取推荐输出中的 `macro_etf_ef_count`。
2. 如果推荐输出没有覆盖该股票，则用全市场 iFinD 行业映射 `sw_l1` 关联行业 ETF。
3. 再读取 `outputs/market_assets_state/market_assets_state_YYYYMMDD.csv` 中的 ETF `ef_count`。

因此：

- `full_match`：个股规则成立 + P116 state 组合匹配 + 行业 ETF `ef_count >= 2`。
- `stock_only`：个股规则和 state 组合成立，但行业尚未配置可用 ETF。
- `market_unsupported`：个股规则和 state 组合成立，但行业 ETF `ef_count < 2`。
- `not_match`：个股自身不满足 2560 适配 State 组合。

2026-05-22 已用黑狼 ETF 清单扩展行业覆盖，新增交通运输、传媒、公用事业、建筑材料、房地产、机械设备、环保、石油石化、计算机、钢铁 10 个直接行业 ETF。扩展后全量 `ma2560_strong_hold` 分布为：

| 匹配等级 | 数量 |
|----------|----:|
| `full_match` | 62 |
| `stock_only` | 8 |
| `market_unsupported` | 97 |
| `not_match` | 519 |

剩余 `stock_only` 主要是轻工制造、社会服务、商贸零售、美容护理等缺少直接行业 ETF 的代理问题，不应强行用宽泛消费类 ETF 认定为完整市场共振。

全量前向验证使用 `strategy_signal_daily` 中所有 `ma2560_strong_hold` 研究样本，
不是每日提醒层子集。脚本为：

```bash
python3 scripts/analyze_ma2560_market_match_forward.py --date YYYY-MM-DD
```

## 4. ATR 吊灯策略

### 核心思想

ATR 吊灯策略用 State 环境过滤筛选入场时机，用 ATR 跟踪止损让利润奔跑。其理论基础来自《超级交易员》中验证的结论：**随机入场 + ATR 吊灯出场在长期具有正期望值**。这意味着出场逻辑比入场逻辑更重要——只要环境过滤足够好、出场规则足够灵活，即使入场不完美也能获得正期望。

### 理论来源

《超级交易员》（Super Trader）中的经典实验：在随机入场条件下，使用 ATR 吊灯跟踪止损（从最高点回撤 N 倍 ATR 即离场），长期回测仍为正期望。这证明了趋势跟踪出场的数学优势。

### 适用周期

日线及以上周期。

### 与 VCP/2560/布林强盗的定位差异

| 维度 | VCP | 2560 | 布林强盗 | ATR 吊灯 |
|------|-----|------|---------|---------|
| 趋势阶段 | 新生 | 行进 | 延展 | **全程** |
| 入场核心 | 支点突破 | 均线回踩 | 上轨突破 | **State 环境过滤** |
| 出场核心 | 移动止损 | 均线跌破 | 递减均线 | **ATR 跟踪止损** |
| 入场精确度 | 高 | 高 | 高 | **中（依赖环境而非形态）** |
| 出场灵活度 | 中 | 中 | 中 | **高（自适应 ATR）** |

### 入场条件

ATR 吊灯的入场不依赖价格形态，而是依赖 State 环境过滤——当三周期 State 均处于扩张或正值状态时触发。

**月线 MN1 允许集合**：

```text
{E, F, 2, 3, 6, 7, 10, 11}
```

含义：排除所有负值状态（负向 = 月线破位），只要月线为正值即可。

**周线 W1 允许集合**：

```text
{E, F, 2, 3, 6, 7, 10, 11}
```

含义：同月线，排除负值状态。

**日线 D1 允许集合**：

```text
{E, F, 2, 6, 7, 3}
```

含义：比月/周线更严格——排除无趋势的扩张初期（8/9/A/B）和收缩态（0-7 中的非突破项），只要求 D1 处于有突破或有趋势的状态。

**触发条件**：MN1 ∈ 允许集合 AND W1 ∈ 允许集合 AND D1 ∈ 允许集合 → 触发 entry 信号。

### 出场规则

#### 初始止损

```text
止损价 = 入场后最高收盘价 - 3 × ATR(20)
```

- ATR(20) 为入场当日的 20 日平均真实波幅
- 每日更新：用持仓期间的最高收盘价替代入场价
- 止损价只上移，不下移（跟踪止损）

#### 盈利收紧

```text
R = 入场时的 ATR(20)  （初始风险单位）

当浮盈 >= 4R 时：
  止损价 = 最高收盘价 - 1.6 × ATR(20)
  （从 3ATR 收紧为 1.6ATR）
```

**设计理由**：

- 3×ATR 初始止损：给予足够的波动空间，避免正常波动触发止损
- 4R 盈利后收紧为 1.6×ATR：趋势已大幅运行，需要保护利润
- 1.6×ATR 仍然比 1×ATR 宽，避免过早被震出

#### R 计算示例

```text
入场价：50.00
ATR(20) = 2.00
R = 2.00

初始止损 = 50.00 - 3 × 2.00 = 44.00（-12%）

第 10 天：最高价 56.00，浮盈 = 6.00 = 3R
  止损 = 56.00 - 3 × 2.00 = 50.00（保本）

第 15 天：最高价 58.00，浮盈 = 8.00 = 4R → 触发收紧
  止损 = 58.00 - 1.6 × 2.00 = 54.80（锁定 +9.6%）

第 20 天：最高价 60.00
  止损 = 60.00 - 1.6 × 2.00 = 56.80（锁定 +13.6%）

第 25 天：收盘价 56.50 < 止损 56.80 → 出场
  最终收益 = (56.50 - 50.00) / 50.00 = +13.0%
```

### 系统映射

- `strategy_id`: `atr_chandelier`
- 正式 entry 信号可进入提醒层。
- 出场由执行层管理（ATR 跟踪止损需要每日更新最高价），提醒层不出场指令。
- 需要持仓上下文（最高价、入场 ATR），因此出场信号不在信号账本中独立生成。

### 与其他策略的关系

ATR 吊灯与其他三个策略不竞争，而是覆盖它们无法覆盖的阶段：

- 当 VCP/2560/布林强盗都因环境不匹配而无法触发时，ATR 吊灯可能仍可触发（只要 State 不为负值）
- 当趋势从一个阶段过渡到另一个阶段时（如 VCP 结束、2560 尚未触发），ATR 吊灯提供连续覆盖
- ATR 吊灯的出场规则可以作为其他策略的出场参考

## 策略与环境标签

环境标签不是策略信号，也不替代策略规则。它们只描述策略信号出现时所处的 State 背景，用来帮助使用者过滤不适合自身节奏的场景。

当前环境标签示例：

- `三周期共振新近形成`
- `D1收缩充分`
- `D1刚脱离收缩`
- `W1刚脱离收缩`
- `MN1刚脱离收缩`
- `波动稳定`
- `波动偏活跃`
- `三周期共振延续`

原则：

- 策略信号回答“是否触发”。
- 环境标签回答“触发时的背景是什么”。
- 校准统计回答“历史上类似背景表现如何”，但只有质量闸门通过后才展示真实数字。

## State-策略匹配总纲

如果把 State 系统视为市场环境识别器，四套策略分别对应趋势生命周期的不同阶段：

| 趋势阶段 | State 背景 | 适配策略 | 核心用途 |
|----------|------------|----------|----------|
| 趋势新生 | `D1刚脱离收缩`、`三周期共振新近形成` | VCP | 捕捉收缩结束后的支点突破 |
| 趋势行进 | 三周期 E/F 稳固、持续天数增加、`波动稳定` | 2560 | 跟踪趋势中的回踩确认 |
| 趋势延展 | 波动从稳定转为活跃，或趋势已明显延续 | 布林强盗 | 捕捉波动扩张后的趋势加速段 |
| 趋势全程 | 三周期不为负值即可，覆盖新生/行进/延展 | ATR 吊灯 | State 环境过滤入场 + ATR 跟踪止损让利润奔跑 |

这不是精确预测框架，而是场景过滤框架。State 提供先验概率更高的环境，策略模块提供精确触发，提醒层只展示事实。

### VCP 与趋势新生

VCP 的最佳场景是市场刚从收缩进入扩张：

- State 出现 `D1刚脱离收缩`。
- 或三周期 E/F 刚同步形成，即 `三周期共振新近形成`。
- D1 的收缩过程对应 VCP 中的弹簧压缩。
- 放量突破支点时，策略模块可生成正式 entry 信号。

研究理解：

- 收缩区域成交量逐步萎缩，代表供应减少。
- 突破日成交量放大，代表需求重新主导。
- 在 A 股环境中，这类结构常对应机构建仓后期的静默阶段结束。

### 2560 与趋势行进

2560 的最佳场景是趋势已确认但仍保持健康节奏：

- 三周期 E/F 状态已经持续数日。
- D1 趋势不再只是首次突破，而是进入行进状态。
- 环境标签出现 `波动稳定` 时，说明趋势运行相对平顺。

研究理解：

- 25 日均线是趋势生命线。
- 第一次或第二次回到 25 日均线附近的质量通常高于多次回踩。
- 5 日均量线位于 60 日均量线之上，可作为成交动能仍在的确认。
- 这类结构适合观察趋势延续，而不是寻找最低点。

### 布林强盗与趋势延展

布林强盗的最佳场景是趋势进入波动扩张阶段：

- 波动状态从稳定转为活跃。
- 或价格离开原有 SR 区间，进入趋势延展区域。
- 此时机会和风险同时放大。

研究理解：

- 上轨突破不是阻力解释，而是趋势扩张确认。
- 过去 30 日最高收盘价过滤用于降低假突破。
- 自适应均线让策略在趋势早期保留空间，在持有时间增加后提高敏感度。

### 决策树口径

系统内部可采用以下研究口径理解策略分工：

1. 当 State 显示收缩或刚脱离收缩时，重点关注 VCP 类支点突破信号。
2. 当 State 显示趋势稳定、波动平顺时，重点关注 2560 类趋势回踩确认信号。
3. 当 State 显示波动突然放大或趋势进入延展区时，重点关注布林强盗类波动扩张信号和已有持有上下文中的防守规则。
4. 当三周期均不为负值但其他策略未触发时，ATR 吊灯可作为全覆盖策略，用 ATR 跟踪止损管理持仓。

执行边界：

- 这套决策树只用于研究解释和策略模块分工。
- 提醒层不得输出具体操作指令。
- 任何执行动作必须来自策略模块、执行系统或人工交易日志。

## 系统化执行边界

完整链条是：

```text
State 底座寻找先验概率大的环境
  -> 策略模块提供精确触发
  -> 策略信号账本记录事实
  -> 提醒层展示信号和环境
  -> 执行层按策略规则处理进入、离开、调整
```

提醒层不得承担执行层职责。任何成交、仓位、离场、调整都必须来自独立的执行规则、持有上下文或人工记录。
