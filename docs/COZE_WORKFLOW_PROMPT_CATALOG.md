- **可复用性**: 高

### 2.2 公司概况节点 (二、公司概况)

#### node_id: 175309 | 公司概况分析
- **输入**: Top10_holders, Main_Business, Report_2025Q3, Report_2024, Report_2023, Report_2022
- **输出**: `Company_Overview`
- **模型**: DeepSeek-V3.1
- **prompt_summary**: 
  - 角色：严谨、证据导向的证券高级分析师
  - 核心约束：禁止废话、直接输出、信源零容忍（禁止股吧/论坛/自媒体）
  - 信源分级：一级（政府/交易所）、二级（头部券商/Wind）、三级（官媒）、废弃（股吧等）
  - 输出结构：公司简介、发展历程、股权结构、主营业务、财务状况（含表格）
  - 字数：1000-1200字
- **风险**: 角色设定为"证券分析师"，但输出内容以事实描述为主，无明确投资建议
- **可复用性**: **最高** — 信源分级标准、数据清洗规范、表格结构可直接复用

## 4. 可复用 Prompt 专区

### 4.1 高价值可复用结构

#### 信源分级标准（来自 175309）
```
一级（核心）: 政府、监管机构官网（证监会、交易所）、上市公司正式公告
二级（高信度）: 头部券商（中信、中金）、主流财经终端（Wind、巨潮资讯）
三级（一般建议）: 官媒（财新、证券时报、中国证券报）
废弃信源: 任何含有"股吧"、"自媒体"、"营销号"字样的搜索片段
```
**复用建议**: 直接纳入当前系统的 `source_map` 规范，作为 `source_confidence` 的分级依据。

#### 数据清洗规范（来自 175309）
```
1. 解析变量，剔除SQL杂质
2. 完成亿元单位换算
3. 所有金额统一转为"亿元"，保留2位小数
4. 多年份引用原则：按当前系统 `period_rows` 配置动态提取最新可用期核心指标，禁止硬编码年份
```
**复用建议**: 纳入 `external_research_evidence.py` 的数据预处理逻辑。

#### 反幻觉约束（来自多个节点）
```
- 严禁引用"股吧"、"论坛"、"自媒体"、"个人博客"
- 严禁使用"网友称"、"据传"、"爆料"等词汇
- 来源域名包含guba、weibo、zhihu等必须立即舍弃
- 严禁编造数据，无确切来源时改用定性描述
```
**复用建议**: 作为所有LLM调用的全局system prompt约束。

### 4.2 可复用的分析框架

#### 产业链分析框架（来自 182242）
- 上游、中游、下游结构拆解
- 价值锚点：哪个环节利润最丰厚、话语权最强
- 集中度分析：CR3/CR5数据引用
- 竞争态势：价格战 vs 良性竞争

#### 财务健康度评估框架（来自 191599）
- 成长性：营收/利润复合增长、最新季度同比增速
- 盈利质量：毛利率/净利率趋势、ROE杜邦拆解
- 财务健康：经营现金流净额、净现比、资产负债率

#### 商业模式拆解框架（来自 194560）
- 解决方案：客户群体（ToB/ToC/ToG）及痛点
- 运营模式：轻资产撮合 vs 重资产全产业链
- 核心竞争力：成本/技术/规模/转换成本/牌照
