## 二、改进原则（三不三动）

1. **不动计算层**：State Hex 编码 bit-exact 不变，回归测试不受影响。
2. **动表达层**：用户看到的 State 从 Hex 降维到「6 命名 + 6 节奏 + 共振度」。
3. **不动数据库Schema**：基于现有 d1_perspective_state 表做视图转换。
4. **动前端展示**：缺失数据模块直接隐藏，不渲染空壳。
5. **不动AI底层模型**：通过提示词工程（Prompt Engineering）提升输出质量。
6. **动Agent编排**：引入 Agent 集群分工，替代单一大模型直接回答。

## 三、优先级与排期

### 🔴 P0：本周必须完成（潘杰原话：「争取这一周让大家觉得好用」）

| 序号 | 任务 | 责任人 | 验收标准 |
|------|------|--------|---------|
| P0-1 | **首页一句话判断**：删除三模式切换，首屏只展示「今日判断句+重点行业+动作按钮」 | 前端+AI Agent | 用户打开首页 3 秒内看到结论 |
| P0-2 | **结论去模糊化**：所有 AI 生成结论必须经过「判官Agent」二次过滤，禁用「谨慎乐观」「结构追踪」等词汇，改用「机会/观望/回避」或概率 | Prompt Engineer | 抽检 20 条结论，模糊词汇出现率 <5% |
| P0-3 | **缺失数据隐藏**：深度卡中「暂无」「—」模块前端不渲染，只展示有数据模块 | 前端 | 页面无「暂无」字样 |
| P0-4 | **修复个股重复描述 bug**：研究队列中同一只股票的周期描述去重/统一 | 后端/AI | 同一 symbol 只出现一种共振描述 |
| P0-5 | **AI 助手入口显性化**：从右下角浮窗改为首页第一屏 C 位 + 每行股票「问观象」按钮 | 前端 | 首页可见，表格行可操作 |

### 🟡 P1：两周内完成（体验升级）

| 序号 | 任务 | 责任人 | 验收标准 |
|------|------|--------|---------|
| P1-1 | **State 经纬系统上线**：内部 Hex → 用户层 6 命名（天时/地利/人和/蓄力/冬眠/逆位） | 后端+前端 | 全站 State 列显示中文命名+emoji |
| P1-2 | **共振度主展示**：三周期平铺 → ef_count 为主，二级展开具体 Hex | 前端 | 列表页只展示共振度，详情页展开 |
| P1-3 | **板块细分标签**：电力设备 → 光伏/风电/储能/电网设备等二级标签 | 数据+AI | 重点行业带 2-3 个细分标签 |
| P1-4 | **用户分层首页**：注册时询问用户类型（方向/研究/执行），后续只展示对应板块 | 前端+后端 | 不同类型用户看到不同首屏 |
| P1-5 | **State 叙事节奏层**：在 6 命名之上叠加「冬藏/萌芽/破土/生长/秋收/逆风」6 节奏 | AI Agent | 每个股票卡片显示节奏标签 |

### 🟢 P2：一个月内完成（深度建设）

| 序号 | 任务 | 责任人 | 验收标准 |
|------|------|--------|---------|

## 四、关键假设与验证方式

| 关键假设 | 验证方式 | 负责人 |
|---------|---------|--------|
| 目标用户（50-300万，5年经验）喜欢「一句话结论」胜过「三段分析」 | A/B 测试：首页版本A（一句话）vs 版本B（三模式），看停留时长和点击率 | 产品 |
| 6 命名 State 能降低教育成本 | 用户测试：给 5 个老用户看 Hex 和 6 命名，测试理解准确率 | 产品 |
| 共振度（ef_count）比三周期平铺更有决策价值 | 追踪用户从列表页到详情页的展开率 | 数据 |
| 数据缺失隐藏比显示「暂无」更能保留信任 | 问卷：用户看到隐藏版 vs 空壳版的信任度打分 | 产品 |

## 五、风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| 合规红线：结论太明确可能被认定为「荐股」 | 产品下架/法律 | 结论模板加入风险提示；明确标注「系统分析，不构成投资建议」；概率化表达而非绝对化 |
| AI Agent 幻觉导致数据错误 | 用户信任崩塌 | 所有数据类回答必须带出处链接；数值类回答优先查 DuckDB 再生成 |
| State 表达层改造引入前端 bug | 回测系统数据不一致 | 表达层只做「显示映射」，不改数据库和 API 返回的原始 Hex |
| 一周时间太紧，P0 无法全部完成 | 用户体验未达预期 | P0 再分「P0a（3天内）」和「P0b（本周内）」，确保首页和结论先上 |
