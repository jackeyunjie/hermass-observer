## 2. Hermass 设计需求画像

### 2.1 业务特性
- **高数据密度**：表格、数字、状态标签、资金流向
- **决策时效**：交易员需要 3 秒内抓到重点
- **长时间盯盘**：深色模式友好 > 浅色模式
- **专业可信**：金融属性，拒绝花哨/卡通
- **中文为主**：中文排版、数字字体、千分位分隔

### 2.2 用户反馈中的设计痛点
1. **信息过载** → 需要强信息层级、留白、折叠
2. **页面有点散** → 需要统一的网格系统和视觉锚点
3. **术语看不懂** → 需要颜色编码 + 图标 + Tooltip
4. **结论模糊** → 需要强烈的视觉状态区分（红绿黄灰）

## 3. 组件库选择分析

| 组件库 | 优势 | 劣势 | 适合度 |
|--------|------|------|--------|
| **shadcn/ui** | Tailwind 底层，轻量可定制；深色模式原生支持；数据表格/Dialog/Sheet 组件丰富；现代 SaaS 标准 | 需自行组装复杂表格 | ⭐⭐⭐⭐⭐ **首选** |
| **Spark Design** | 阿里云出品，中文企业级优化；表单/表格/仪表盘成熟 | 风格偏传统企业后台；定制自由度低 | ⭐⭐⭐ 备选 |
| **Ant Design** | 中文生态最完善；ProComponents 有高级表格/仪表盘 | 包体积大；风格偏老派后台；深色模式一般 | ⭐⭐⭐ 备选 |
| **不指定** | AI 自由发挥，可能更贴合描述 | 组件一致性难保证；需后期重构 | ⭐⭐ 快速原型 |

### 推荐结论

**主方案：shadcn/ui + 自定义金融组件**

理由：
- Hermass 不是传统「企业后台」，而是「数据密集型交易工作台」
- shadcn/ui 的 `Table`、`Badge`、`Card`、`Sheet`、`Tooltip` 刚好覆盖 Watchlist/深度卡/侧边栏需求
- 深色模式（`dark` class）对交易员友好
- 可以在此基础上封装 `StateBadge`、`ResonanceCard`、`CapitalFlowTag` 等金融专用组件

**备选方案：Spark Design**
- 如果团队更熟悉阿里云生态，或需要快速搭建标准后台
- 适合 B 端管理后台，但交易前台可能不够「锐」

## 4. 风格参考选择分析

从 161 种风格中，筛选最适合金融数据/交易台的 6 个候选：

### 候选 1：Linear ⭐⭐⭐⭐⭐（最推荐）

| 维度 | 评价 |
|------|------|
| **信息密度** | 极高，issue 列表紧凑但不乱 |
| **深色模式** | 原生，专业深色（#0E0E10 背景） |
| **状态表达** | 颜色+图标+标签三位一体 |
| **中文适配** | 字体需换为 PingFang SC / Inter |
| **适合页面** | Watchlist 表格、首页 Dashboard |

**为什么选 Linear：**
- 交易员看 Watchlist 就像工程师看 Issue 列表——需要快速扫描状态
- Linear 的「优先级标签」直接对应我们的「共振度 Badge」
- 侧边栏（Sheet）交互适合「问观象」展开

### 候选 2：Stripe ⭐⭐⭐⭐⭐（并列推荐）

| 维度 | 评价 |
|------|------|
| **信息密度** | 高，Dashboard 数据卡片布局经典 |
| **数据可视化** | 图表+指标卡片的行业标杆 |
| **信任感** | 金融支付背景，天然可信 |
| **适合页面** | 首页「今日判断」、共振度分布、行业卡片 |

**为什么选 Stripe：**
- Stripe Dashboard 就是给商人看钱的——数字要准、要大、要有变化指示
- 「今日判断」卡片直接参考 Stripe 的 Balance/Overview 卡片

### 候选 3：Notion ⭐⭐⭐⭐

| 维度 | 评价 |
|------|------|
| **信息层级** | 极强，Toggle/折叠/嵌套页面 |
| **文字排版** | 长文本可读性最好 |
| **适合页面** | 个股深度卡、研究页长文 |

**为什么选 Notion：**
- 深度卡的长文本分析需要强排版层次
- Toggle 块适合「展开看 Hex 详情」的交互

### 候选 4：Vercel ⭐⭐⭐⭐

| 维度 | 评价 |
|------|------|
| **极简** | 留白多，重点突出 |
| **深色** | 极佳（Vercel Dashboard） |
| **适合页面** | 首页首屏、登录页 |

### 候选 5：Supabase ⭐⭐⭐

| 维度 | 评价 |
|------|------|
| **数据表** | 数据库浏览界面，表格强大 |
| **开发者感** | 略偏技术，可能不够亲民 |

### 候选 6：Bloomberg Terminal（如果有）⭐⭐⭐⭐⭐

截图中未明确看到 Bloomberg 风格，但如果有：
- 黑色背景+荧光文字是交易终端的经典
- 但教育成本高，不适合 50-300 万散户

## 5. 推荐组合方案

### 方案 A：现代交易台（推荐）

```
组件库：shadcn/ui
风格参考：Linear（主）+ Stripe（数据卡片）+ Notion（文本排版）
模式：高保真
颜色体系：
  - 背景： slate-950 (#020617) 深色为主，浅色为辅
  - 卡片： slate-900 (#0f172a)
  - 强调：
    - 天时 🔥：red-500 (#ef4444)
    - 地利 ☀️：orange-500 (#f97316)
    - 人和 🌤：yellow-500 (#eab308)
    - 蓄力 🌥：blue-500 (#3b82f6)
    - 冬眠 🌧：slate-500 (#64748b)
    - 逆位 ⚡：purple-500 (#a855f7)
  - 资金流入：emerald-500  资金流出：rose-500
字体：Inter（数字）+ PingFang SC（中文）
```

**适用场景：** 面向有经验交易者，长时间盯盘，追求专业感

### 方案 B：企业级后台（备选）

```
组件库：Spark Design
风格参考：Ant Design / Corporate
模式：高保真
颜色体系：阿里云标准色板，蓝色为主
```

**适用场景：** 如果未来要扩展为机构版，需要更稳重的感觉

### 方案 C：快速验证原型（本周内）

```
组件库：不指定
风格参考：Clean / Minimal
模式：线框图
用途：先把首页结构和信息层级跑通，再细化视觉
```

## 6. QoderWork 提示词模板（可直接使用）

### 6.1 首页「今日判断」Dashboard

```markdown
请为「Hermass 多周期观测台」设计一个金融数据首页 Dashboard。

**产品定位**：面向 A 股交易者的多周期状态观测平台
**核心功能**：展示今日市场整体判断、重点行业速览、共振度分布
**设计约束**：
- 使用 shadcn/ui 组件库
- 参考 Linear 的深色模式信息密度 + Stripe 的数据卡片
- 首屏必须包含：
  1. 顶部导航栏（Logo + 观象AI入口 + 用户头像）
  2. 「今日判断」主卡片：大字体结论 + 具体数字 + 2个动作按钮
  3. 3-5 个行业速览卡片（横向排列）
  4. 共振度分布迷你条形图
- 颜色编码（必须使用）：
  - 天时 🔥 #E74C3C
  - 地利 ☀️ #F39C12
  - 人和 🌤 #F1C40F
  - 蓄力 🌥 #3498DB
  - 冬眠 🌧 #95A5A6
  - 逆位 ⚡ #9B59B6
- 中文界面，数字使用千分位分隔
- 移动端首屏可正常阅读

**避免**：
- 不要使用渐变色背景
- 不要使用卡通插图
- 不要使用大面积彩色块，用标签/徽章表达状态
```

### 6.2 执行页 Watchlist 表格

```markdown
请设计一个金融股票筛选表格页（Watchlist）。

**产品定位**：交易者每日查看候选股票的执行页面
**设计约束**：
- 使用 shadcn/ui 的 Table + Badge + Button 组件
- 参考 Linear 的 issue 列表：紧凑行高、强状态标签、hover 效果
- 列结构：
  - 股票名称/代码
  - 共振度（彩色 Badge：🔥天时/☀️地利/🌤人和/🌥蓄力/⚡逆位）
  - 节奏（标签：生长季/秋收期/破土期/萌芽期/逆风期/冬藏期）
  - 资金流向（小圆点+文字：流入/中性/流出）
  - 突破状态（真突破/假突破/刚突破）
  - 持续天数（数字）
  - 操作（💬 问观象 按钮）
- 顶部有共振度快捷筛选标签栏
- 深色模式，背景 #0E0E10，卡片 #1A1A1D
- 行 hover 时显示微光效果
- 每行高度 56px，确保一屏显示 10-12 行

**避免**：
- 不要用斑马纹（用 hover 区分别行即可）
- 不要显示 MN1/W1/D1 三个 Hex 值（隐藏到二级展开）
```

### 6.3 个股深度卡

```markdown
请设计一个个股深度分析卡片（Depth Card）。

**产品定位**：展示单只股票的多周期分析、资金流、产业链信息
**设计约束**：
- 使用 shadcn/ui Card + Tabs + Badge 组件
- 参考 Notion 的文本层级 + Stripe 的数据指标排版
- 结构：
  1. 头部：股票名称 + 共振度大标签 + 节奏标签 + 建议动作按钮
  2. 多周期定位区：MN1/W1/D1 三个小卡片，显示命名状态（点击展开 Hex）
  3. State 解读区：限高 120px，超出折叠
  4. 资金流区：5日流入/流出柱状迷你图 + 确认标签
  5. 产业链区（可选）：只有有数据时才显示
  6. 底部：固定免责声明条
- 缺失数据模块完全不渲染，不留空白占位
- 深色模式，文字对比度符合 WCAG AA
- 中文排版，段落最大宽度 65ch

**避免**：
- 不要用「暂无数据」占位符
- 不要显示两个矛盾的周期描述
```

### 6.4 AI 助手「观象」侧边栏

```markdown
请设计一个 AI 助手对话侧边栏（Sheet/Drawer）。

**产品定位**：交易者随时提问的 AI 助手「观象」
**设计约束**：
- 从屏幕右侧滑入，宽度 420px
- 顶部：助手头像 + 名称「观象」+ 关闭按钮
- 中部：对话气泡，用户右对齐，AI 左对齐
- AI 回答必须三段式显示：
  - 【结论】（加粗，彩色标签）
  - 【依据】（正常文本）
  - 【下一步】（带 ⚠️ 风险提示）
- 底部：输入框 + 快捷问题标签（如「这只股票什么状态？」「现在能进场吗？」）
- 深色模式，与主页面风格一致
- 支持 Markdown 渲染（表格、列表）

**避免**：
- 不要全屏覆盖，要侧边栏形式
- 不要用闪烁光标等过度动画
```
