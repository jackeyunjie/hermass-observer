> 版本：V2.0  
> 日期：2026-05-30

## 1. 文档目标

将第9次会议反馈和 State 数据分析结论，转化为可执行的产品需求。

## 2. 用户画像与分层

| 用户类型 | 特征 | 核心诉求 | 首屏展示 |
|---------|------|---------|---------|
| **方向型** | 50-300万资金，看大趋势，交易频率低 | 「现在能不能做？做什么方向？」 | 市场环境判断 + 重点行业 |
| **研究型** | 基金经理/深度研究者，需要多因子分析 | 「这只股票什么状态？值不值得深入研究？」 | 个股深度卡 + 多周期拆解 |
| **执行型** | 短线/趋势跟踪交易者，需要明确信号 | 「今天跟踪哪只？真突破还是假突破？」 | 执行页候选列表 + 盯盘提醒 |

**注册流程增加 1 步：「你更关注？」**  
选项：【方向/研究/执行】→ 后续登录默认进入对应首页，其他入口折叠到导航栏。

### 3.3 需求细节

**FR-01 今日判断卡片**
- 数据来源：`d1_perspective_state` 表 `state_date='今日'`
- 计算逻辑：
  - `ef≥2 股票数 / 全市场股票数` = 活跃占比
  - 活跃占比 >20% → 「进攻环境」
  - 活跃占比 10-20% → 「震荡选择环境」
  - 活跃占比 <10% → 「防守/等待环境」
- 文案模板（由 AI 判官 Agent 生成）：
  - 必须包含：环境定性 + 具体数字 + 建议动作 + 1-2 个重点行业
  - 禁用词库：「谨慎」「乐观」「结构追踪」「密切关注」「适当参与」「中长期看好」
  - 可用词：「机会/观望/回避」「进攻/防守/等待」「跟踪/建仓/减仓」「顺风/逆风」

**FR-02 重点行业速览**
- 展示 3-5 个行业卡片
- 每个卡片显示：
  - 行业名称
  - 该行业平均共振度（天时/地利/人和/蓄力/冬眠/逆位）
  - 资金流向标签：流入/中性/流出
  - 点击 → 跳转到该行业成分股执行页

**FR-03 共振度分布条**
- 小型可视化，让用户一眼看出今天市场整体在什么阶段
- 6 种状态用固定颜色：
  - 天时 🔥：#E74C3C（红）
  - 地利 ☀️：#F39C12（橙）
  - 人和 🌤：#F1C40F（黄）
  - 蓄力 🌥：#3498DB（蓝）
  - 冬眠 🌧：#95A5A6（灰）
  - 逆位 ⚡：#9B59B6（紫）

### 4.3 需求细节

**FR-04 共振度主列**
- 列表页只展示一个综合判断：`ef_count` 映射 + 主导周期
- 映射规则：
  - ef=3 → 🔥 天时共振
  - ef=2 → ☀️ 地利共振
  - ef=1 → 🌤 单一周期
  - ef=0 → ❄️ 无共振
  - 若三周期均为负值 → ⚡ 逆位共振（单独标签）

**FR-05 节奏标签（基于实际字段）**

> 确认来源：`d1_perspective_state` 表中实际存在 `ef_count`, `d1_state_score`, `d1_state_magnitude`, `d1_position_bit` 字段。`sustained_days` / `boundary_distance` 不存在，已用等效字段替换。

| 节奏 | 判断条件（DuckDB/SQL） | 含义 | 建议动作 |
|------|----------------------|------|---------|
| **生长季** | `ef_count >= 2 AND d1_state_score > 0 AND d1_state_magnitude >= 14` | 多周期共振向上，三周期至少两个处于正向 E/F，日线最强 | 标准跟踪 |
| **秋收期** | `ef_count >= 2 AND d1_state_score > 0 AND d1_state_magnitude IN (12, 13)` | 多周期共振但日线非最强（C/D），对应高位延展 | 防守/减仓 |
| **破土期** | `d1_state_score > 0 AND d1_position_bit = 2 AND d1_state_magnitude >= 10` | 日线向上且处于突破位（人和及以上：A/B/E/F） | 跟踪确认 |
| **萌芽期** | `d1_state_score > 0 AND d1_state_magnitude >= 4 AND d1_state_magnitude <= 7` | 日线向上但处于蓄力状态（4-7），有趋势无突破 | 观察 |
| **过渡期** | `d1_state_score > 0 AND d1_state_magnitude IN (8, 9) AND d1_position_bit = 0` | 日线人和（8/9），有趋势但无突破，单一周期未共振 | 观察 |
| **冬藏期** | `d1_state_score >= 0 AND d1_state_magnitude <= 3` | 日线极弱或零状态，无有效信号 | 等待 |
| **逆风期** | `d1_state_score < 0` | 日线向下（所有逆位：-A 到 -F，-2/-3） | 暂不推进 |

**字段说明：**
- `ef_count`: INTEGER，MN1/W1/D1 中正向 E/F（state_score>0 且 magnitude>=14）的个数
- `d1_state_score`: INTEGER，正值=向上，负值=向下，绝对值=magnitude
- `d1_state_magnitude`: INTEGER，结构强度（0-15）
- `d1_position_bit`: INTEGER，0=无突破/边界状态，2=突破/延展状态
- `d1_state_hex`: VARCHAR，原始编码（如 'E', '-F', 'C', '8' 等），仅用于展示保留

**FR-06 每行「问观象」按钮**
- 点击后弹窗/侧边栏，AI 用固定模板输出：
  ```
  【当前结论】一句话，yes/no风格
  【为什么】2-3 个理由，用数据说话
  【下一步】建议动作 + 风险提示
  ```
- 禁止输出超过 300 字。

**FR-07 突破确认标签**
- 真突破：price > upper_boundary + volume > MA20 + 资金流入
- 假突破：price > upper_boundary + volume < MA20 或 资金流出
- 刚突破：上突 + distance ≤ 1%，标记为「破土期」

### 5.3 需求细节

**FR-08 数据缺失处理**
- 规则：任何模块的数据为空/暂无/— 时，前端不渲染该模块的标题和占位符
- 例外：「多周期定位」和「State 解读」必须有，属于核心功能

**FR-09 周期描述去重**
- 后端 API 返回前，对同一 symbol 的多个周期描述做合并：
  - 若 AI 生成多个描述，取 ef_count 最高的那个为主描述
  - 若描述矛盾（如一个说强势一个说弱势），触发人工 review 标记

**FR-10 AI 生成约束**
- 字数限制：State 解读 ≤ 200 字，整体卡片 ≤ 800 字
- 每句必须有数据支撑，禁止纯定性描述
- 必须包含「来源」标注（同花顺/黑狼/AI估算/手动维护）

## 7. 非功能性需求

| 类别 | 要求 |
|------|------|
| **性能** | 首页加载 ≤ 2s；Watchlist 表格 ≤ 1000 行时滚动不卡顿 |
| **兼容性** | 支持 Chrome/Safari/Edge 最新两版；移动端自适应（先保证可读） |
| **安全** | 用户密码加密存储；AI 回答必须带免责声明；禁止展示具体买入价位 |
| **可维护** | State 命名映射表抽离为 JSON 配置，不改代码即可调整 |
| **监控** | 首页「今日判断」每日生成后写入日志表，便于审计和回滚 |

## 9. 合规需求

- **FR-C01**：任何涉及具体股票的结论下方必须显示固定文案：「本分析由 AI 基于多周期模型生成，仅供参考，不构成投资建议。市场有风险，决策需独立。」
- **FR-C02**：禁止生成具体买入价位、目标价位、持仓比例。
- **FR-C03**：概率化表达优先：「历史上类似 State 环境下，该策略胜率约 X%」→ 允许；「这只股票会涨」→ 禁止。
