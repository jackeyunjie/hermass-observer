## 3. 宏观象限合成判断

### 3.1 坐标计算

```
横轴（增长周期）= 0.60 × S_growth + 0.40 × S_inflation
                = 0.60 × 4.25 + 0.40 × 4.20
                = 2.55 + 1.68
                = 4.23

纵轴（货币信用周期）= 0.55 × S_liquidity + 0.45 × S_credit
                    = 0.55 × 5.75 + 0.45 × 5.00
                    = 3.16 + 2.25
                    = 5.41
```

### 3.2 象限判定

| 判定条件 | 值 | 结果 |
|----------|-----|------|
| 增长周期 >= 5.0？ | 4.23 < 5.0 | **增长偏弱** |
| 货币信用周期 >= 5.5？ | 5.41 < 5.5 | **货币信用偏紧（严格）** |

**严格判定：滞胀**（增长弱 + 货币信用紧）

### 3.3 边界分析

纵轴 5.41 非常接近阈值 5.5，处于**边界区域**：

- 若将 S_liquidity 向上微调 0.2（考虑 M2=8.6% 实际偏松），则纵轴 = 5.52 >= 5.5
- 此时象限变为**衰退**（增长弱 + 货币信用宽）

**更合理的描述**：处于**衰退/滞胀边界**，数据不足以做出可靠区分。

### 3.4 与直觉的对比

| 维度 | 数据信号 | 直觉判断 |
|------|----------|----------|
| 增长 | PMI 49.4（收缩） | 偏弱 |
| 通胀 | CPI 0%, PPI -3.6% | 通缩压力，非滞胀 |
| 流动性 | M2 8.6% | 中性偏松 |
| 信用 | 社融 6245 亿 | 无法判断 |

**矛盾点**：PPI 深度负值（-3.6%）更符合**衰退/通缩**特征，而非滞胀（高通胀）。  
**原因**：框架的象限划分基于评分值而非原始数据，评分转换可能丢失了通缩信号。

### 4.1 新模型置信度

```
confidence = coverage_factor × history_factor × freshness_factor

coverage_factor = 5 / 22 = 0.227
history_factor = min(1.0, 0 / 24) = 0.000
freshness_factor = 0.50  (AKShare 数据 9 个月前)

confidence = 0.227 × 0.000 × 0.50 = 0.000
```

| 因子 | 值 | 说明 |
|------|-----|------|
| coverage_factor | 0.227 | 5/22 指标可用 |
| history_factor | 0.000 | 无指标有 12 个月+历史 |

### 5.2 四维到策略的加权映射（进阶版）

```
VCP macro_adj = (0.30×(4.25-5)/5 + 0.35×(5.75-5)/5 + 0.20×(5.0-5)/5 + 0.15×(4.2-5)/5) × 15
              = (0.30×(-0.15) + 0.35×0.15 + 0.20×0 + 0.15×(-0.16)) × 15
              = (-0.045 + 0.0525 + 0 - 0.024) × 15
              = (-0.0165) × 15
              = -0.25

MA2560 macro_adj = (0.20×(-0.15) + 0.25×0.15 + 0.35×0 + 0.20×(-0.16)) × 15
                 = (-0.03 + 0.0375 + 0 - 0.032) × 15
                 = (-0.0245) × 15
                 = -0.37

布林强盗 macro_adj = (0.25×(-0.15) + 0.35×0.15 + 0.20×0 + 0.20×(-0.16)) × 15
                   = (-0.0375 + 0.0525 + 0 - 0.032) × 15
                   = (-0.017) × 15
                   = -0.26
```

**结果**：所有策略加成接近 0，符合"数据不足时不参与策略调整"的原则。

### 5.3 实际建议

**由于置信度为 0.00，不应基于当前宏观数据调整策略权重。**  
建议保持默认中性加成（macro_adj = 0），待数据质量改善后再启用四维评分。
