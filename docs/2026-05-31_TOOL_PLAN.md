# 2026-05-31 工具链规划

版本：v1.0  
日期：2026-05-31  
作者：Codex  
目标：将测试、部署、代码质量三条工具链定义为本地可执行的最小闭环。

---

## 限制说明

本规划只定义“做什么、交付什么、如何验收”，不写入业务逻辑（不涉及 Agent、场景、行情规则等），也不输出可运行代码。实现阶段的代码由后续开发任务完成。

---

## 板块 1：测试与回归工具链

### 问题
- 本地无 `pytest`，`tests/smoke/test_agently_chains.py` 无法直接执行。
- 没有统一入口区分“需要 API key 的冒烟测试”和“不需要 key 的单元测试”。

### 方案
- 新增 `scripts/run_tests.sh` 作为测试入口。
- 新增 `.env.test` 存放测试所需的本地变量（仅含非敏感占位或本地 key 路径）。
- 新增 `tests/smoke/README.md` 说明如何显式激活 `.venv` 并传入 `HERMASS_DEEPSEEK_API_KEY`。

### 文件清单
- `scripts/run_tests.sh`
- `.env.test`
- `tests/smoke/README.md`

### 里程碑
- M1：`scripts/run_tests.sh unit` 可跑所有 unit tests（mock，无需 key）。
- M2：`scripts/run_tests.sh smoke` 可跑 3 个核心冒烟用例（需 key）。

### 验收标准
- `scripts/run_tests.sh unit` 正常结束且输出包含 `passed`。
- `scripts/run_tests.sh smoke` 在设置 `HERMASS_DEEPSEEK_API_KEY` 后可正常运行。
- 任一测试子集失败时，脚本打印失败用例名称和退出码。

### 脚本约定
- 输入：`unit` 或 `smoke` 参数。
- 输出： pytest 标准输出；失败时在末尾打印最后错误摘要。
- 退出码：`0` 通过，`1` 失败，`2` 参数错误。

## 板块 2：部署与回滚工具链

### 问题
- 当前部署依赖手工 `rsync` + `py_compile` + `systemctl restart` + `curl` 冒烟，步骤分散，失败后无自动诊断。

### 方案
- 新增 `scripts/deploy.sh`：一键代码同步、语法检查、重启服务、冒烟验证、失败回滚。
- 新增 `scripts/rollback.sh`：一键代码回退、重启服务、自动跑 smoke。
- 新增 `scripts/deploy_env.sh`（只读配置）：统一读取服务器 IP、路径、服务名。

### 文件清单
- `scripts/deploy_env.sh`
- `scripts/deploy.sh`
- `scripts/rollback.sh`

### 里程碑
- M1：`scripts/deploy.sh` 可成功执行一键部署并输出“部署完成 ✅”。
- M2：`scripts/rollback.sh` 可成功回滚并输出“回滚完成 ✅”。

### 验收标准
- 部署脚本遇到 `rsync`、`py_compile`、`systemctl restart`、`curl` 中任一步失败时立即中止并打印最后成功步骤。
- 冒烟测试通过后，脚本打印服务状态摘要（active/running）。
- 回滚脚本执行后，版本符合作业前状态且 smoke 测试通过。

### 脚本约定
- `scripts/deploy.sh` 输入：可选的 rsync 源路径；无参数时默认本机相对路径。
- `scripts/deploy.sh` 输出：分步日志；最终输出服务状态和 HTTP/状态码。
- `scripts/deploy.sh` 退出码：`0` 成功，`3` 同步失败，`4` 语法检查失败，`5` 重启失败，`6` 冒烟失败。
- `scripts/rollback.sh` 输入：无。
- `scripts/rollback.sh` 输出：git 回退摘要 + smoke 结果。
- `scripts/rollback.sh` 退出码：`0` 成功，`7` 回滚失败，`8` smoke 失败。

## 板块 3：代码质量与可观测性

### 问题
- `_llm_chat_answer()` 函数已超过 100 行，异常处理只用 `pass`，排查成本高。
- 当前没有统一的静态检查与运行日志机制。

### 方案
- 在 `pyproject.toml` 中引入 `ruff` 配置（格式 + 静态检查）。
- 在 `web/main.py` 内为数据预取异常补充 `logging.warning` 并记录失败来源。
- 新增 `web/routes_debug.py`，暴露只读调试接口 `/api/debug/memory?session_id=xxx`，返回脱敏后的 `_build_memory_context()` 结果，帮助验证代词解析与场景消费。

### 文件清单
- `pyproject.toml`（追加 `[tool.ruff]` 配置）
- `.pre-commit-config.yaml`（可选、后续启用）
- `web/routes_debug.py`
- `scripts/lint.sh`

### 里程碑
- M1：`ruff` 检查通过，`scripts/lint.sh` 可一键执行。
- M2：数据预取异常有可追溯日志，`_build_memory_context()` 可通过调试接口读取。

### 验收标准
- `ruff check` 与 `ruff format` 对修改文件通过。
- 模拟数据预取异常后，日志包含来源字段（`market_data` / `industry_distribution` / `value_prompt_pack`）。
- 调用 `/api/debug/memory?session_id=xxx` 能返回 `recent_turns` 和 `recent_stock_codes` 列表。

### 脚本约定
- `scripts/lint.sh` 输入：`check` / `format` 参数。
- `scripts/lint.sh` 输出： ruff 检查结果；失败时打印首个错误文件/行号。
- `scripts/lint.sh` 退出码：`0` 通过，`9` lint 检查失败。
