# COZE Prompt 资产迁移优先级（Phase 1）

版本：v1.0  
日期：2026-05-28  
范围：A 股 External Research Response（Research-Only）

---

## 结论

Phase 1 最值得先迁的 2 个 COZE prompt 资产是：

1. `175309 | 公司概况分析`
2. `194560 | 商业模式与核心竞争力分析`

选择标准不是“哪段 prompt 最华丽”，而是：

- 能直接提升当前 `evidence -> formatter -> API/飞书` 主链质量
- 合规风险低
- 与现有数据源契合度高
- 不会把系统带回长篇投研报告主线

---

## 优先级 1：175309 | 公司概况分析

### 为什么它最值钱

这是当前 COZE 资产里最适合直接沉淀进系统底层的一份 prompt 资产。它最有价值的不是“生成一段公司介绍”，而是：

- 信源分级标准
- 数据清洗规范
- 事实性公司介绍的组织结构
- 反幻觉约束

这些能力不依赖卖方语气，也不依赖目标价/评级，和当前 `external_research_evidence.py` 的职责高度一致。

### 当前系统的对应缺口

当前研究链已经能输出：

- `company_profile`
- `financial_trend`
- `industry_state`
- `state_core`

但还缺：

- 更明确的 `source_confidence` / 信源分级口径
- 更统一的公司概况叙述模板
- 面向 LLM 解释层的“事实优先、无幻觉”全局约束

### 迁移目标

优先迁移 4 类资产：

1. 信源分级标准
2. 数据清洗规范
3. 公司概况字段组织顺序
4. 反幻觉约束

### 迁移落点

- `external_research_evidence.py`
  - 继续吸收数据清洗与字段整理规则
- `EXTERNAL_RESEARCH_RESPONSE_EVIDENCE_CONTRACT.md`
  - 补强 `source_map` 的信源等级语义
- 未来 LLM 解释层全局 system prompt
  - 吸收“禁止股吧/自媒体/据传/网友称”等规则
- `deep card`
  - 参考其公司概况字段顺序，而不是照抄长篇章法

### 禁止迁移的部分

- “证券高级分析师”角色设定
- 1000-1200 字长文要求
- 卖方/投研报告语气

### Phase 1 产出形式

不是单独新增一个“公司概况生成器”，而是：

- 一组全局规则
- 一组字段组织规则
- 一组 source confidence 规则

## 优先级 2：194560 | 商业模式与核心竞争力分析

### 为什么它排第二

当前系统最明显的短板不是“没有事实”，而是“缺少一层高质量的解释组织”。  
`194560` 正好补这个缺口。

它的价值在于：

- 业务模式拆解框架
- 客户/场景/运营模式的分析结构
- 核心竞争力维度（成本/技术/规模/转换成本/牌照）
- 强制 peer 对标思路

这类内容非常适合变成 `deep card` 的一个“解释型扩展模块”。

### 当前系统的对应缺口

现在的 `deep card` 有：

- 公司概况
- 财务趋势
- 行业景气 / State 环境
- 估值参考
- 券商观点
- 风险与限制

但缺：

- “为什么这家公司有竞争力”的结构化解释层
- “商业模式属于哪一类”的简洁判断层

### 迁移目标

把 `194560` 迁成一个低风险、短文本、基于 evidence 的解释模块：

- 模块名建议：
  - `公司竞争力观察`
  - 或 `商业模式与核心竞争力`
- 输出位置：
  - `deep card` 新增子章节
  - 后续可扩成按需触发的研究问答子模块

### 迁移输入

只允许使用当前已有证据层可稳定提供的数据：

- `company_profile.main_business`
- `company_profile.main_product_names`
- `company_profile.main_product_types`
- `company_profile.comparable_companies`
- `company_profile.sw_l1/l2/l3`
- 必要时配合 `financial_trend` 的历史表现

### 禁止迁移的部分

- “二级市场行业研究员”角色扮演
- 明确 Alpha / 超额收益 / 交易机会语义
- 为凑长文本而生成大段空泛赞美

### Phase 1 产出形式

不是生成一整章“公司分析”，而是：

- 一个可插入 `deep card` 的短模块
- 1 到 3 段解释文本
- 必须由 evidence 驱动
- 必须带数据充分度与风险边界

---

## 为什么不是别的节点

### 不是 181473（市场观点总结）

原因：

- 已有 `market_views` evidence
- 风险高，容易把评级/目标价重新带回默认输出
- 适合后移，不适合做第一批解释层资产

### 不是 179899 / 113460（估值链）

原因：

- 合规风险最高
- 和当前 Research-Only 边界冲突
- 容易把系统重新带回“合理估值 -> 目标价 -> 建议”链路

### 不是 165674（股价催化剂）

原因：

- “交易级机会”语义太强
- 适合以后降级成“事件观察”
- 不适合第一批进入对外解释层

### 不是 121060 / 121443（盈利预测）

原因：

- 当前系统优先做历史趋势，不做前瞻预测
- 这类资产更适合放到第二批，作为“历史趋势解释模板”处理

---

## 迁移顺序

### Step 1

先迁 `175309` 的非生成型资产：

- 信源分级
- 数据清洗
- 反幻觉规则
- 公司概况字段组织顺序

### Step 2

再迁 `194560` 的解释框架：

- 商业模式拆解
- 核心竞争力维度
- peer 对标结构

### Step 3

等这两项稳定后，再考虑第二梯队：

- `191599 | 盈利质量与财务健康分析`
- `182242 | 行业产业链拆解与竞争格局`

---

## 对实现层的要求

这 2 个资产的迁移都必须遵守：

- Evidence First
- 只服务 A 股
- 不恢复长篇报告主线
- 不引入投资建议、目标价、评级输出
- 默认服务 `deep card` 和研究问答，不服务完整报告

---

## 一句话结论

Phase 1 先迁：

1. `175309` 的规则资产
2. `194560` 的解释框架资产

前者提升底层可信度，后者提升对外回答质量。  
这两项最符合当前 Hermass 的 Research-Only 主架构。
