消费方：`scripts/generate_daily_trading_sop.py`

---

## 核心理念

**"趋势好时多买，趋势差时少买或不买"** — 将市场环境信息转化为具体的仓位指导。

固定 2% 单笔风险忽略了市场本身的涨跌倾向。在市场扩张期（trending / progression）和收缩期（contraction），同样的信号质量面临完全不同的胜率环境。仓位应该动态反映这个差异。

---

## 1. 仓位调节公式

```text
建议仓位 = 基础仓位 × 市场阶段系数 × 策略加成系数 × 宏观系数
```

| 因子 | 含义 | 来源 |
|------|------|------|
| 基础仓位 | 总资金的 50%（可配置） | 默认值 |
| 市场阶段系数 | 当前市场处于哪个生命周期 | `MARKET_PHASE_IDENTIFICATION.md` → 阶段分类 |
| 策略加成系数 | 当前策略在此时段的历史适配度 | `MARKET_PHASE_FACTORS`（`daily_research_brief.py`） |
| 宏观系数 | 宏观象限对整体仓位的调节 | 衰退/复苏/过热/滞胀 |

### 1.1 市场阶段系数

| 阶段 | 系数 | 逻辑 |
|------|------|------|
| 趋势新生（emergence） | 1.0 | 标准配置，弹簧释放初期 |
| 趋势行进（progression） | 1.2 | 趋势确定，加仓窗口 |
| 趋势延展（extension） | 0.7 | 波动放大，降低敞口 |
| 收缩期（contraction） | 0.3 | 防御姿态，极小仓位 |
| 风险释放（risk_release） | 0.0 | 禁开新仓 |
| 未分类（undetermined） | 0.5 | 半仓观望 |

### 1.2 策略加成系数

直接复用 `daily_research_brief.py` 中的 `MARKET_PHASE_FACTORS`，按策略和阶段二维查表：

| 阶段 | VCP | 2560 | 布林强盗 |
|------|-----|------|---------|
| 收缩期 | 0.90 | 0.80 | 0.80 |
| 趋势新生 | 1.15 | 1.00 | 0.90 |
| 趋势行进 | 1.00 | 1.10 | 1.00 |
| 趋势延展 | 0.90 | 1.00 | 1.15 |
| 风险释放 | 0.80 | 0.90 | 0.80 |
| 未分类 | 1.00 | 1.00 | 1.00 |

### 1.3 宏观系数

| 宏观象限 | 系数 | 逻辑 |
|----------|------|------|
| 复苏 | 1.0 | 经济上行，标准配置 |
| 过热 | 0.9 | 通胀压力，适度谨慎 |
| 衰退 | 0.8 | 经济收缩，降低敞口 |
| 滞胀 | 0.5 | 最差环境，大幅降低 |

### 1.4 计算示例

**场景 A：趋势行进 + 布林强盗（最佳适配）+ 复苏**
```
50% × 1.2 × 1.15 × 1.0 = 69% 建议仓位
```

**场景 B：趋势延展 + VCP（弱适配）+ 过热**
```
50% × 0.7 × 0.9 × 0.9 = 28.4% 建议仓位
```

**场景 C：收缩期 + 任意策略 + 滞胀**
```
50% × 0.3 × 0.9 × 0.5 = 6.8% 建议仓位
```

## 2. 单笔风险预算动态调整

单笔风险不固定为 2%，而是基于市场阶段 × 适配度二维查表：

### 2.1 风险速查表

| 市场阶段 | 最佳适配 | 适配 | 弱适配 | 待观察 |
|----------|---------|------|--------|--------|
| 趋势新生 | **2.5%** | 2.0% | 1.0% | 0.5% |
| 趋势行进 | **2.0%** | 1.5% | 1.0% | 0.5% |
| 趋势延展 | **1.5%** | 1.0% | 0.5% | 0.3% |
| 收缩期 | **1.0%** | 0.5% | 0.0% | 0.0% |
| 风险释放 | **0.0%** | 0.0% | 0.0% | 0.0% |
| 未分类 | **1.5%** | 1.0% | 0.5% | 0.3% |

### 2.2 设计逻辑

- **趋势新生 + 最佳适配 = 2.5%**：刚脱离收缩，弹簧释放初期，统计验证有正超额，略高于基准 2%
- **趋势行进 + 最佳适配 = 2.0%**：趋势健康但已运行一段时间，标准风险预算
- **趋势延展或弱适配 = 1.0%**：环境转弱或信号质量下降，减半
- **收缩期 / 风险释放 = 0% 或不建议开新仓**：不追逐弱势环境中的信号

## 4. 与现有 SOP 脚本的衔接

### 4.1 Python 函数接口

```python
from position_sizing import calculate_dynamic_position

result = calculate_dynamic_position(
    market_phase="progression",       # 市场阶段
    strategy_boost=1.15,              # 策略加成系数
    macro_quadrant="复苏",            # 宏观象限
    fit_level="最佳适配",             # 策略-环境适配度
)

# 返回：
# {
#     "total_allocation_pct": 0.69,       # 69% 建议仓位
#     "per_trade_risk_pct": 2.0,          # 单笔 2% 风险
#     "max_positions": 10,                # 最多 10 只
#     "allow_new_positions": True,        # 允许开仓
#     "reason": "趋势行进+20% × 策略加成=1.15",  # 调整原因
# }
```

### 4.2 在 `generate_daily_trading_sop.py` 中的集成

SOP 脚本已完整接入动态仓位模型：

1. **环境层面**：`build_sop()` 中调用一次，产出环境层面的总仓位建议
2. **个股层面**：Top 5 执行参数中，每只候选单独计算 `per_trade_risk_pct`，替代固定 2%
3. **展示层面**：MD/HTML 输出中新增「动态总仓位」「动态单笔风险」「仓位模型」字段

## 5. 安全钳机制

为防止极端参数组合导致仓位失控：

- 总仓位上限硬编码为 100%
- 总仓位下限硬编码为 0%
- `risk_release` 阶段强制 `per_trade_risk=0`, `allow_new_positions=False`
- `calculate_dynamic_position` 返回 `allow_new_positions: bool`，调用方必须遵守

## 6. 后续扩展点

- **资金曲线反馈**：连续回撤期自动降仓位（凯利公式变体）
- **波动率调整**：ATR 百分比纳入 `per_trade_risk` 计算
- **行业集中度**：同一行业多只候选时，降低单笔风险
- **连胜/连败调节**：连续 N 次胜出可适度上浮风险预算
