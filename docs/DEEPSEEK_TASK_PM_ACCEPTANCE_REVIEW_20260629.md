## 11. 已知限制（不需要验收，但影响评分）

以下为当前已知限制，PM 测试时应当知悉：

1. **观象事件类问题降级**：当实时数据源未接入时，观象会返回 `framework_with_disclosure`（分析框架 + 缺口说明），这是设计行为，不是 bug。
2. **观察任务跨浏览器不共享**：当前观察任务基于浏览器会话，不同浏览器/设备不互通。这是当前架构限制。
3. **数据新鲜度**：系统数据取决于本地数据管线是否正常运行，非实时数据可能在非交易日数小时内陈旧。
4. **个股覆盖**：研究页只覆盖 A 股，不支持港股/美股。

## 12. Go / No-Go 标准

审查者请在审查完成后，按以下标准给出总体结论：

### Go 条件（全部满足才算 Go）

- [ ] 9 个页面全部 HTTP 200（增强接口 401 符合预期）
- [ ] 首页路径路由 + 候选池正常展示
- [ ] 研究页裁决逻辑存在且可读
- [ ] 观察账本可创建、可查看、可取消
- [ ] 观象免登录可用，返回结构化回答
- [ ] 赚钱秘籍覆盖使用说明书全部要点
- [ ] 完整闭环 9 步全部走通
- [ ] 无痕浏览器从头到尾不遇到 401 阻断

### 可放行但需标注的问题（Weak Go）

- 观象个别问题回答偏空或不精准 → 标注具体问题类型
- 候选池为空 → 标注日期与数据状况
- 某页面加载偏慢 → 标注加载时间
- 移动端布局有细微偏移 → 标注设备和现象

### No-Go 条件（任一命中即 No-Go）

- 首页、研究页、观察账本任一 HTTP 非 200
- 观象免登录入口完全不可用
- 研究页裁决结论完全不出现
- 创建观察完全失败
- 完整闭环在步骤 3 之前就断裂
