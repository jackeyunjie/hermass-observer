# AI 助手 MVP 规格

## 1. 目标

在现有 `Hermass 多周期观察台` 上增加一个 **AI 原生对话入口**，提升首次使用体验与停留时长。

定位不是“新建一个纯聊天产品”，而是：

- 以页面和确定性底座为主
- 以 AI 助手为解释层、导航层、调度层
- 用自然语言把现有 `market / industry / watchlist / research / backtest` 串起来

一句话：

**页面是底座，AI 助手是入口。**

## 2. MVP 边界

### 2.1 要做

1. 网站右下角增加对话入口
2. 支持 4 类问题
3. 回答基于现有确定性数据
4. 回答后给出明确跳转入口
5. 所有回答保留研究参考边界

### 2.2 不做

1. 不做自由 SQL
2. 不做自动生成交易建议
3. 不做自动下单
4. 不做多 Agent 长链推理
5. 不做“替用户决定仓位”

## 3. 支持的问题类型

### 3.1 市场问答

示例：

- 现在能不能做
- 今天市场怎么样
- 当前更适合等待还是试错

数据来源：

- `market_phase`
- `daily_snapshot`
- `market_assets_state`
- `macro_chain_prior`

落点：

- `/market`

### 3.2 行业方向问答

示例：

- 今天先看什么方向
- 电子行业怎么样
- 哪些行业先少看

数据来源：

- `industry_rotation`
- `industry_position_summary`
- `market_assets_state`

落点：

- `/industry`

### 3.3 个股研究问答

示例：

- 000021 怎么看
- 深科技现在是什么位置
- 这只是刚突破还是高位延展

数据来源：

- `research evidence`
- `foundation db`
- `unified_view`
- `reward_risk`
- `forward_observation`

落点：

- `/research?stock_code=...`

### 3.4 导航与任务问答

示例：

- 我应该先看哪里
- 今天先做什么
- 我想找高质量候选

数据来源：

- 当前页面上下文
- `watchlist`
- `quant summary`

落点：

- `/watchlist`
- `/market`
- `/research`

## 4. 回答合同

每次回答统一为 6 段：

1. `当前结论`
2. `为什么`
3. `多周期环境`
4. `单周期位置`
5. `暂时不用看什么`
6. `下一步去哪里`

示例：

> 当前不适合全面铺开，更适合先做选择题。  
> 原因是多周期结构并非同步强势，当前更偏局部改善。  
> 多周期环境上，月线与周线未形成全面共振，日线改善更明显。  
> 单周期位置上，当前更像活跃推进与局部修复，不是全面主升起点。  
> 暂时少看高位延展样本，不把所有突破都当成早期机会。  
> 下一步先看市场页，再去执行观察页筛候选。

## 5. 页面交互

### 5.1 入口

- 右下角悬浮按钮：`Hermass 助手`
- 点击后展开侧边对话面板

### 5.2 快捷问题

首屏给 4 个快捷按钮：

- 现在能不能做
- 今天先看什么方向
- 帮我看一只股票
- 我应该先去哪一页

### 5.3 回答后的操作

每条回答底部附 1-2 个动作按钮：

- 打开市场页
- 打开行业页
- 打开执行观察
- 打开研究卡

## 6. 技术结构

### 6.1 前端

新增：

- 对话浮层组件
- 消息列表
- 输入框
- 快捷问题按钮

### 6.2 后端

新增接口：

- `POST /api/chat/query`

输入：

```json
{
  "message": "现在能不能做",
  "page_context": "market",
  "stock_code": null
}
```

输出：

```json
{
  "answer": "当前不适合全面铺开，更适合先做选择题。",
  "why": "多周期结构并非同步强势，当前更偏局部改善。",
  "multi_cycle_view": "先看 MN1/W1/D1 是否同步共振。当前更像大周期未全面同步、日线局部改善的环境。",
  "single_cycle_position": "当前单周期节奏更偏日线活跃推进与局部修复，适合先观察结构延续性。",
  "avoid": "暂时少看高位延展样本。",
  "next_actions": [
    {"label": "打开市场页", "url": "/market"}
  ],
  "sources": ["market_phase", "daily_snapshot"],
  "freshness_note": "市场阶段与快照按 2026-05-29 口径展示。"
}
```

## 7. 回答生成原则

### 7.1 必须

1. 基于确定性底座回答
2. 引用已存在模块
3. 语气简洁
4. 结论可执行
5. 能跳回页面

### 7.2 禁止

1. 买入卖出建议
2. 目标价
3. 必涨必跌
4. 编造数据
5. 把低频背景数据伪装成当日结论

## 8. 与现有系统的连接

### 8.1 使用已有能力

- `web/main.py` 的聚合逻辑
- `research evidence`
- `watchlist` 的多因子判断
- `market` 的阶段与节奏判断

### 8.2 不新增新底座

AI 助手只消费：

- 已有页面数据
- 已有研究数据
- 已有状态判断

不新增新的交易引擎或策略引擎。

## 9. MVP 成功标准

1. 新用户 10 秒内知道从哪里开始
2. 用户能用自然语言问出第一个问题
3. 回答能把用户导向正确页面
4. 回答不越过研究参考边界
5. 不影响现有页面主路径

## 10. Phase 2

MVP 稳定后再考虑：

1. AI 对话记忆
2. 个股连续追问
3. 内测版 NL→SQL
4. 每日 AI 叙事摘要
5. 对话式策略实验入口
