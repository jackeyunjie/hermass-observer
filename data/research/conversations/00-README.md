## 核心结论汇总

### 产品内核
Hermass Observer 的内核不是 EF 共振筛选，而是**多周期 Agent 市场态运转系统**。
D1 Agent 提供执行信号，W1 Agent 提供趋势质量过滤，MN1 Agent 提供大方向许可。

### 三周期权重（基于 2026-06-01 生产数据）
- D1：50-60%（与次日回报相关性 0.205，最强）
- W1：25-30%（独立 EF 率最低=最严格过滤器）
- MN1：15-20%（方向最稳定，与短期回报相关性最低）

### 行业维度（基于 2026-06-01 生产数据）
- 全市场 EF2=10.7%，但行业间 EF2 率差异达 0%-90%
- 电子行业 80 只股票占全部 EF 信号的 37.4%
- 同属"电子"，半导体 EF2=85.2% vs 元件 EF2=73.1%→二级行业内部也有差异
- 行业应作为独立权重维度（15-25%），与三周期 Agent 并列

### 信息过载病灶（5个）
1. 首页面板爆炸（10+面板）
2. 双设计系统分裂（浅色 CSS vs 深色 Tailwind）
3. 策略编辑器两套版本
4. 执行观察表 14 列
5. 研究页信息层叠 12+ 面板

### 自组织多 Agent 运营（最终架构方向）
- 不推翻现有底座，在底座上加 Agent 层
- 资金流保留在 D1 Agent 内部
- 目标：先验概率 + 场景化 AI 应用流（压制检测→突破跟踪→持仓管理）
- Agent 四个成长维度：记忆、见识、稳定性、创新性
- 五步渐进路径：记忆 → 场景库 → 概率化 → 协作 → 强化学习闭环
- 现有 7 个 Agent 已具备基础框架，需增加：判断→结果自反馈、场景库、概率输出

### 孙子兵法·知胜有五（交易哲学根基）
- 知可以战与不可以战者胜 = 市场环境判断
- 识众寡之用者胜 = 仓位管理
- 上下同欲者胜 = 策略-环境匹配
- 以虞待不虞者胜 = 提前预期
- 将能而君不御者胜 = Agent 自主（核心！）

### 强化学习因子与评价体系
- 时间因子：季节、节气、月份、周几——State 不变但市场节奏按日历变化
- 事件因子：重要会议、宏观报告、央行动态、财报季——已有 ifind_event_radar 骨架
- 多视角评价：同一信号在 D1/W1/MN1 视角下含义不同，RR、储能、胜率分别计算
- 先验概率 = 历史统计 × 时间修正 × 事件修正 × 行业修正
- 后验校准 = 实际 vs 先验偏差 → 超过阈值自动触发校准
- Agent 速度设定：快(D1/每日)、中(W1/每周)、慢(MN1/每月)——各看各的时间尺度
- **因子自发现机制**：Agent 检测异常偏差 → 扫描关联特征 → 提出假设 → 回溯验证 → 入候选池
- **权重自修正机制**：每月末按因子分解偏差 → 对比实际贡献 vs 设定权重 → 指数平滑修正
- 因子池生命周期：发现→验证→激活→休眠→归档
- 人做框架（因子类型/初始权重/校准周期），Agent 做填充和修正
- 多 Agent 交叉验证：快慢 Agent 独立验证同一因子，防止单一 Agent 自欺

### 多 Agent 体系构建验证与持续运行（最终整合框架）
- **可构建吗**：已有 7 个 Agent 生产代码 + 测试体系（37个测试文件）+ 多周期底座 + 校准触发器
- **三步升级路径**：AgentMemory → AgentBus 消息总线 → 自组织循环
- **构建验证**：契约层→单元层→集成层→端到端 四层金字塔
- **运行时验证**：Q1产出状态→Q2方向回溯→Q3趋势改善→Q4策略效益 四象限
- **三层评价**：系统自评(每4h)+Agent互评(每日)+人类审阅(每日)
- **自组织**：自调度(AgentScheduler)+自协作(AgentBus pub/sub)+自进化(权重/因子自修正)
- **记忆**：AgentMemory.duckdb(判断记录/回溯结果/场景库/权重历史/进化日志)
- **沟通**：AgentBus实时pub/sub + 共享数据库异步读写
- **复盘**：日复盘(自动化)→周复盘(Agent主导)→月复盘(人类+Agent)
- **改进**：闭环验证(改进→观察窗口→确认有效→固化/无效→回滚)
- **持续运行**：Assume Failure 原则 + 四级降级路径 + AgentSupervisor 监控
- **量变到质变**：场景样本+判断记录→因子自发现+准确率突破+Agent互补判断+人退出日常操作
- 核心原则：不是等完美再上线——"合抱之木，生于毫末"

### 外部方案深度研究（节奏加速 + 业界对标）
- 评价节奏：AI自评每4h、Agent互评每日、人类评价至少每日一次
- **NousResearch/hermes-agent**：自学习闭环+Skill自动创建+FTS5搜索+Cron调度+子Agent委托+ACP协议 → 最值得借鉴的架构
- **agentmemory**：BM25+Vector+Graph混合检索(R@5=95.2%)+置信度评分+四级生命周期 → AgentMemory设计的参照标杆
- **Harness Engineering**：OpenAI 0行手写代码实践，七大核心原则——Context Engineering、机械约束、Agent反馈循环、熵管理、深度优先
- **Karpathy Wiki + Obsidian**：三层架构(Ingest→Query→Lint)+渐进式披露 → 对应 Hermass 的 docs/wiki/
- 所有方案与Hermass的设计方向高度一致（agentmemory生命周期↔Hermass因子池、HE原则↔将能而君不御）
