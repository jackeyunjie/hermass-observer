# 信息过载诊断

日期：2026-06-01
主题：前端信息过载，需要聚焦最核心的产品内核

## 项目定位确认

P116 A 股多周期结构观察平台，核心能力链路：

```
原始数据 → State 计算 (MN1/W1/D1) → EF 共振筛选 → 策略信号生成 → 每日观察输出
```

## 5 个信息过载病灶

### 病灶 1：首页面板爆炸（index.html）
首页包含 10+ 个独立面板，既有用户视角的'今日判断'，又有运维视角的'产物状态''定时任务'。

### 病灶 2：双设计系统分裂
自建 CSS 与 TailwindCSS 两套风格并存，视觉连贯性断裂。

### 病灶 3：策略编辑器两套版本
mystrategies.html 与 strategy-editor.html 是同一功能的两套 UI。

### 病灶 4：执行观察表 14 列
用户需要水平滚动才能看到所有列，认知负荷极高。

### 病灶 5：研究页信息层叠 12+ 面板
用户需要滚动很久才能看完一只股票。

## 聚焦建议（按优先级）

### P0：首页做减法
保留：今日判断 Hero + 精简数据条 + '从哪里开始'入口
移除：术语说明、信号数据提示、每日产物、运行时、研究预览

### P0：统一设计系统
journal 和 strategy-editor 改为浅色 CSS 系统

### P1：执行观察表精简
14 列 → 6 列核心 + 折叠详情

### P1：研究页收束
12+ 面板 → 三段折叠结构：收束层 → 证据层 → 详情层

### P2：策略编辑器合并
删除 strategy-editor.html，保留 mystrategies.html

### P2：运维面板独立入口
将运维面板从核心页面移除，单独做 /admin 或运维入口
