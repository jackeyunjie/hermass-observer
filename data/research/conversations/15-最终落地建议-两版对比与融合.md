## 一、两版核心差异

| 维度 | 14 号计划（Claude） | Codex 计划 |
|------|-------------------|-----------|
| **起点** | 直接进入工程阶段 | 先定义"我们到底在做什么"（漏斗层） |
| **结构** | 6 阶段、12 周 | 3 波、1-2 个月 |
| **粒度** | 细：具体到每个表的字段和函数 | 粗：按能力域分组，给验收标准 |
| **部署** | 没有部署模板 | 三段可直接发服务器的部署命令 |
| **前端** | 6 列精简、三段折叠 | L1/L2/L3 认知分级 + 调试层权限隔离 |
| **权重** | 动态自修正（因子级） | **档位切换**（震荡/趋势/极端三档） |
| **外部方案** | 明确分阶段引入 | **先不引入 Hermes Skill**，等第三波稳定 |
| **讨论输出** | 开放式 6 个问题 | **结论模板**（会后可直接填） |

## 二、Codex 计划更好的 5 个地方

### 1. 漏斗层——先定方向再开工

Codex 一上来就问"我们到底在做什么"，给了 A1/A2/A3 三个口径选项。14 号计划直接跳进了工程细节，没做这个对齐。如果团队对"这是 EF 筛选器还是运转系统"没有共识，后续所有前端设计都会反复拉扯。

**采纳**：波 0 必须先做。

### 2. 三波比六阶段更适合团队节奏

Codex 的 3 波更聚拢——AgentMemory 建表和先验引擎可合并到同一波。14 号过度拆分了。

**采纳**：合并为 4 波（波 0 + 波 1-3）。

### 3. 档位切换比动态自修正更安全

Codex 的权重用三档（震荡市/趋势市/极端分叉）切换，而非让 Agent 每月自由调。解决了 14 号计划中未解决的风险：权重自修正频率太高会导致不稳定。

**采纳**：波 1 用档位，波 3 再加动态自修正。

### 4. 部署模板直接可用

Codex 给了 A/B/C 三个部署版本，每个都有完整 shell 命令。

**采纳**：每次提交后直接使用。

### 5. 结论模板让会议产出可控

10 个讨论问题 + 决策表模板 = 开完会一定有结论。

**采纳**：合并两版的讨论问题。

Codex 第三波主要是降级和消息总线，因子自发现、权重自修正、免疫稳态、五条红线没展开。

**保留**：波 3 的自进化机制。

### 3. 依赖关系图

**保留**：依赖图作为波次推进的前提检查。

## 波 0：漏斗层对齐（1 天）

**不开工任何代码。由人在会上确认方向。**

### 0.1 口径选择 → A2

| 选项 | 内容 | 选择 |
|------|------|------|
| A1 纯内核 | 多周期运转系统 | ✗ |
| **A2 分层包装** | 内核对内=运转系统；EF 对用户=信号入口 | **✓ 推荐** |
| A3 偏保守 | 仍以 EF 为首页第一信号 | ✗ |

理由：对工程师能做任务拆分（"给运转系统加收缩观测能力"），对用户有直观入口（"今天 EF 信号有多少"），两面都能说通。

### 0.2 首页保留/移除

**保留 4 块**：当前周期位置 + 策略触发计数 + Agent 脉搏摘要 + 行业 State 流

**移除**：多层装饰页、账本调试信息、原始价格图表/技术指标公开版

### 0.3 认知陷阱清单（会前确认）

- 不把"展示 EF 股票列表"混同"只展示 EF 股票列表"
- 动态权重必须有规则窗口与熔断，不变成每天自由浮动
- 自组织 Agent 不可改策略结构（五条红线之一）

### 0.4 必须做出的决策

| 决策点 | 选项 |
|--------|------|
| 数据质量异常值策略 | 废弃 / 标 DEGRADED 仍计入 / 保留标记 |
| 四重突破确认深度 | 四选三 / 二重满足+延迟确认 / 等价得分 |
| 权重切换权限 | 自动 / Agent 提案+人类确认 / 仅人类 |
| 一级 vs 二级行业角色 | 一级仅展示 / 二级进入信号修正 / 两者都进 |

### 0.5 死锁退出机制

波 0 讨论限时 **1 小时**。若超时仍有分歧：
- 默认走 **A2** 口径（对内运转系统 + 对外 EF 入口）
- 4 周后（波 1 验收时）回溯是否要切换口径
- 决策表里至少填入"唯一下一步动作"和负责人

## 波 3：自进化闭环 + 前端认知卸载（4 周+）

### 目标
权重自修正 + 因子自发现 + 质变检测 + 五条红线；首页变为决策分级器。

### 任务清单

| # | 任务 | 来自 | 交付物 | 验收标准 |
|---|------|------|--------|---------|
| 3.1 | 因子权重自修正 | 14号 | 指数平滑：新权重=旧×0.7+实际贡献×0.3 | 幅度≤20% 自动生效，超限需人类确认 |
| 3.2 | 因子免疫稳态 | 14号+辩论加分 | 每个因子±10%耐受带，超带连续N日才触发调整 | 避免过度修正 |
| 3.3 | 因子自发现 | 14号 | 异常偏差→扫描特征→回溯验证→候选池 | 第一个自发现因子通过人类审核 |
| 3.4 | 场景库自动构建 | 14号 | ContractionObserver 提取 State 组合模式 | 场景模板 > 20 |
| 3.5 | 质变检测 | 14号 | 贝叶斯变点+Chow检验+Simpson检查 | 20日滚动方向正确率趋势 p<0.05 |
| 3.6 | 五条红线硬编码 | Claude方案 | 代码层拦截（配置文件已于波1建立） | 红线绕过任何路径都无法通过 |
| 3.7 | 首页 L1/L2/L3 分级 | Codex | 3 个数字 + 红黄绿灯 | 3 秒可读完 |
| 3.8 | 执行观察表精简 | 14号 | 14列→6列核心+折叠 | 无需水平滚动 |
| 3.9 | 研究页三段折叠 | 14号 | 收束层→证据层→详情层 | 默认只看收束层 |

### 五条红线（Claude 方案，不可协商）

| # | 红线 | 实现方式 |
|---|------|---------|
| 1 | **止损/止盈不可自动执行** | 交易执行永远需要人类确认 |
| 2 | **策略结构不可变** | VCP/2560/Bollinger 的信号定义不允许 Agent 修改 |
| 3 | **数据可信性不可自行判断** | 数据异常必须提交人类，Agent 不做"我觉得这个数据大概对" |
| 4 | **仓位上限不可自动突破** | 最大仓位比例由人设定，Agent 只在限制内调整 |
| 5 | **人类可随时拔线** | Admin API 一键暂停所有 Agent 自进化 |

### 波 3 验收

```
□ 至少一个因子完成自动权重修正（人类审核通过）
□ 因子免疫稳态就位
□ 质变检测产出统计显著结果
□ 五条红线无法被 Agent 绕过
□ 首页 L1/L2/L3 可用，3 秒可读完
```

## 七、风险与应对

| 风险 | 应对 |
|------|------|
| 团队对口径未对齐就开始编码 | **波 0 不完成不进入波 1**；死锁超时 1h 默认走 A2 |

## B.4 网站与 Agent 后台的关系

```
网站（用户可见）               Agent 后台（用户不可见）
─────────────                ─────────────────────
首页 L1/L2/L3                ← ContractionObserver 产出收缩信号
执行观察队列                  ← StrategyAdvisor 消费收缩信号，调整优先级
行业旋转页                    ← MarketAnalyst 行业分布判断
个股研究页                    ← 所有 Agent 的判断汇总 + AgentMemory 回溯数据
AI 助手（观象）               ← AgentBus 驱动的实时消息
管理后台 /api/admin           ← kill-switch / Agent 状态 / 红线状态
```

**关键**：用户看到的每个页面变化，背后都对应一个或多个 Agent 的运行产出。前端收敛不是单纯的 UI 美化——它是 Agent 体系从"查询型"升级到"自组织型"后，信息密度降低、决策精度提高的自然结果。
