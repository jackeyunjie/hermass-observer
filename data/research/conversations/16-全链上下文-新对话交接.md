## 项目是什么

A 股多周期市场态运转系统。内核不是 EF 共振筛选，而是多周期 Agent 市场态运转系统。

**基础数据**：
- 5510 只 A 股，MN1（月线）/ W1（周线）/ D1（日线）三周期 State 编码
- 全市场 EF2 率 10.7%，行业差异 0%-90%
- D1 score 与次日涨跌幅 r=0.205（最强），W1 r=0.082，MN1 r=0.061
- 三个策略：VCP 收缩突破、2560 趋势跟踪、Bollinger Bandit

**网站**：https://console.supertrader.world
**服务器**：8.130.125.201，FastAPI:8020，systemd: hermass-console
**项目路径**：`/opt/hermass`（服务器）/ `/Users/lv111101/Documents/hermass-observer-product`（本地）

## 部署方式

SSH 到服务器执行：`git pull → py_compile → restart → smoke check`。

部署脚本模板（复制到本地终端）：
```bash
cat > /tmp/hmass_deploy.sh << 'EOF'
set -euo pipefail
ssh -o StrictHostKeyChecking=no root@8.130.125.201 'bash -s' <<'REMOTE'
set -euo pipefail
cd /opt/hermass
git pull
source .venv/bin/activate && python -m py_compile web/main.py
sudo systemctl restart hermass-console
sudo systemctl status hermass-console --no-pager || true
HTTP=$(curl -s -o /dev/null -w "%{http_code}" http://localhost:8020/)
echo "HTTP_CODE=$HTTP"
REMOTE
EOF
bash /tmp/hmass_deploy.sh
```

### 波 2：收缩观测 + Agent 协作（已完成）
- `hermass_platform/agents/contraction_observer.py`：三重交叉验证（BB 带宽 + 枢轴带宽 + ATR 比率，三选二触发收缩信号）
- `hermass_platform/bus/agent_bus.py`：Agent 消息总线，6 个标准 topic（contraction_extreme/market_phase_change/false_breakout/weight_adjusted/review_needed/data_stale），JSON schema 校验
- `scripts/build_stock_percentiles.py`：每只股票独立 BB/枢轴/ATR 分位阈值，不依赖全市场一刀切
- 混沌演练通过：kill ContractionObserver → 服务正常恢复

### 波 3：自进化 + 前端（已完成）
- `scripts/factor_homeostasis.py`：因子免疫稳态（±10% 耐受带，连续 5 日超带触发，5 天观察期内不再调整）
- `scripts/build_scenario_library.py`：场景库自动构建，样本 < 20 标 low_confidence，每周五自动运行
- **五条红线**（`config/redlines.yaml` + `hermass_platform/red_lines.py`）：
  1. 止损/止盈不可自动执行
  2. 策略结构不可变（VCP/2560/Bollinger/composite）
  3. 数据异常必报人类
  4. 仓位上限 25%/单行业 40%，不可覆盖
  5. Admin kill-switch（24h 自动过期）
