日期：2026-06-01
主题：时间/事件因子体系、多周期评价系统、Agent 快慢视角设定

## 一、因子体系总览

### 核心理念

市场的运转不是匀速的。同一只股票在月初和月末、节气和节气之间、会议前和会议后，同样的 State 组合含义不同。因子不是取代 State，而是在 State 底座之上加时间/事件修正层。

```
State 底座（不变量）        因子层（修正量）           输出
─────────────────    ×    ───────────────    =    ──────
MN1/W1/D1 State          时间因子（季节/节气/月初末/周几）
资金流证据               事件因子（会议/报告/公告）
行业 State                自然因子（农历/节假日）
策略信号                  宏观因子（CPI/PMI/信贷发布日）
                        ─────────────────
                        复合修正系数
```

## 二、时间因子

### 2.1 季节因子

| 时段 | 标签 | 历史特征 | 权重影响 |
|------|------|---------|---------|
| 1-3月 | 春季行情 | 节后资金回流+年报窗口 | 突破信号 +5% 置信度 |
| 4-6月 | 业绩验证期 | 一季报+年报落地后的重新定价 | 回归策略→谨慎，趋势策略→中性 |
| 7-9月 | 中报季 | 夏季流动性收缩+中报预期 | 中报超预期个股→独立行情 |
| 9-10月 | 秋季行情 | 国庆后资金回补 | "金九银十"结构性行情 |
| 11-12月 | 年末效应 | 机构排名+年末资金紧张 | 低位蓄力→春季埋伏 |
| 5-6月 | 五穷六绝 | 传统避险月份 | 趋势策略→降权 |

### 2.2 节气因子

二十四节气是中国资本市场独特的时间节奏——节气变换当日及前后1日容易出现转折。

| 关键节气 | 时间 | 交易特征 | 映射 |
|---------|------|---------|------|
| **立春**（2月初） | 春季行情开端 | 资金开始回流 | D1 State 回暖概率 +8% |
| **夏至**（6月下旬） | 阳气盛极转衰 | 趋势可能见顶 | ADX>50 + 夏至 = 高位延展警告 |
| **秋分**（9月下旬） | 阴阳平衡 | 行情方向选择 | 震荡后方向选择窗口 |
| **冬至**（12月下旬） | 阴极转阳 | 资金布局来年 | 低位蓄力信号→加权 |

节气是定点的，不需要提取历史规律——它是固定的日历节点，直接作为权重窗口用。

### 2.3 月份因子

| 月份因子 | 特征 | 影响 |
|---------|------|------|
| **月初效应**（每月1-5日） | 月初资金面宽松，PMI 发布 | 月初信号+3% |
| **月中效应**（15日前后） | 月度中段，方向延续 | 中性 |
| **月末效应**（最后5日） | 资金面收紧，机构结算 | 月末新信号→谨慎 |
| **期权/期货交割日** | 每月第三个周五 | 大波动日，不适合开新仓 |

### 2.4 周几因子

| 星期 | 特征 | 影响 |
|------|------|------|
| 周一 | 周末信息消化+周一情绪 | 容易出现大阳大阴 |
| 周二-四 | 正常交易日 | 信号可信度最高 |
| 周五 | 博周末消息+避险平仓 | 周五信号需过滤 |

### 2.5 是否交易日

| 类型 | 处理 |
|------|------|
| 节前最后一日 | 交易量萎缩，信号降权 30% |
| 节后首日 | 跳空风险，等待确认后再入 |
| 长假期（五一、国庆、春节） | 节前 3 日信号→仅观察 |

### 3.3 单个事件的影响模式

```
事件发生
    │
    ├── 事前：预期定价期（T-3 至 T-1）→ 权重按预期方向 +10%
    │
    ├── 事件当日：波动放大（T）→ 不产生新信号，仅观察
    │
    ├── 事后消化期（T+1 至 T+3）→ 信号再确认
    │
    └── 衰减期（T+4 至 T+10）→ 影响按指数衰减
```

## 四、评价体系

### 4.1 评价视角：放在哪个周期

同一个信号在不同周期视角下，含义不同：

| 评价视角 | 适合什么策略 | 评价指标 |
|---------|------------|---------|
| **D1 视角**（快） | 短线突破、日内确认 | 次日涨跌幅、3日方向正确率 |
| **W1 视角**（中） | 中线趋势跟踪、2560 | 5日/10日超额收益、最大回撤 |
| **MN1 视角**（慢） | 长线布局、月线共振 | 20日/60日超额收益、胜率稳定性 |

**核心原则**：评价周期必须与策略的持仓周期对齐，不能用一个视角评价所有策略。

### 4.2 储能长短

指从信号产生到市场反应的时间差——"蓄力→释放"的时间跨度。

| 储能类型 | 典型 State 组合 | 释放时间 | 适用策略 |
|---------|---------------|---------|---------|
| **短储能**（1-3天） | D1 刚出收缩即突破 | 次日-3日 | VCP 日内突破 |
| **中储能**（3-10天） | W1 收缩+D1 突破确认 | 3-10日 | 2560 趋势确认 |
| **长储能**（10-20天+） | MN1+W1+D1 多周期共振 | 10-60日 | 长线趋势持有 |

储能长短的评价：
- 短储能 → 看 3 日 RR 和胜率
- 中储能 → 看 10 日超额收益
- 长储能 → 看 20 日+超额、最大回撤路径

### 4.3 先验概率

在未看到今日数据之前，基于历史统计的先验判断：

```
先验 P(突破成功) = 
    基础先验（该 State 组合的历史胜率）
    × 时间修正（季节/周几/是否月初）
    × 事件修正（是否在会议前/报告前）
    × 行业修正（该行业当前 EF2 密度 vs 全市场）
```

**先验概率的来源**：
- State 组合后验统计（已积累：forward_observation 145→持续增长）
- 策略信号综合追踪（已积累：7625 样本，胜率 51.4%）
- 分层回测（适配度 A/B/C 各层的胜率和 RR）

### 4.4 后验变化

后验 = 实际结果出来后，对比先验的偏差。

```
后验偏差 = 实际结果 - 先验概率

如果 |后验偏差| > 阈值：
    → 触发校准：该 State 组合的先验概率需要更新
    → 触发异常标记：该信号所处环境出现了新模式
```

**后验变化的跟踪维度**：

| 维度 | 指标 | 阈值 |
|------|------|------|
| 方向偏差 | |实际方向 - 先验方向| | > 15% → 检查 |
| 强度偏差 | |实际涨幅 - 先验预期涨幅| | > 2σ → 标记 |
| 时间偏差 | |实际储能长度 - 预期储能长度| | > 1倍 → 标记 |
| 转移偏差 | 实际 State 转移 vs 预期转移 | 不同 → 更新转移矩阵 |

### 4.5 不同周期视角的盈亏比

| 视角 | RR 计算方法 | 优势 | 劣势 |
|------|-----------|------|------|
| D1 视角 RR | D1 SR 支撑/阻力 | 快速、敏感 | 波动大、假信号多 |
| W1 视角 RR | W1 SR 支撑/阻力 | 更稳定、过滤短波噪音 | 反应慢、可能错过 |
| MN1 视角 RR | MN1 SR 支撑/阻力 | 最稳定、大方向明确 | 不适用于短期信号 |

**当前系统**：`estimate_reward_risk.py` 的 RR 计算是基于 D1 收盘价 × 各周期 SR，实际是 D1 视角下的三周期 RR。

**建议补充**：
- W1 视角 RR（W1 收盘价 vs W1 SR，用于中长线策略的盈亏比）
- MN1 视角 RR（MN1 收盘价 vs MN1 SR，用于方向许可和仓位上限）

### 4.6 不同概率场景的策略结果预测和统计

| 概率场景 | 策略选择 | 预期结果基准 | 历史样本 |
|---------|---------|-----------|---------|
| **高概率**（EF3 + 最佳适配） | 三种策略均积极 | 20日超额 +1.6-1.7% | VCP 4955 样本 |
| **中概率**（EF2 + 适配） | 匹配的策略正常执行 | 20日超额 +0.5-1.0% | 7625总量样本 |
| **低概率**（EF1 + 弱适配） | 仅观察，不开仓 | 标的降权 | - |
| **压制概率 > 50%** | 等待，不追突破 | 假突破概率高 | - |
| **压制概率 < 30%** | 可积极，突破概率高 | VCP/2560 均可行 | - |

**核心**：不是直接输出"做/不做"，而是输出"当前场景下，历史类似条件下的胜率、RR、储能长度"，让 Agent 做概率判断。

## 五、Agent 快慢视角设定

### 5.1 核心概念

不同的 Agent 看不同的时间尺度，同一时间不同 Agent 的判断可能相反——这是设计特性，不是错误。

```
快 Agent（D1 视角）   → "今天 D1 已突破阻力，可以出手"
慢 Agent（W1 视角）   → "周线还在收缩中，等一周再看"
超慢 Agent（MN1 视角） → "月线方向向上，当前回踩是健康的"

三个 Agent 都对，但回答的是不同时间尺度的问题。
```

### 5.2 三层 Agent 速度设定

| Agent 速度 | 对应周期 | 刷新频率 | 角色 | 决策类型 |
|-----------|---------|---------|------|---------|
| **快 Agent** | D1 | 每交易日 | 执行判断：今天能不能动 | 入场/出场/止损 |
| **中 Agent** | W1 | 每周五 | 趋势过滤：这个行情有没有骨架 | 是否值得持仓/加仓 |
| **慢 Agent** | MN1 | 每月末 | 方向许可：大方向上放多大仓位 | 仓位上限/行业选择 |

### 5.3 Agent 快慢视角的评价体系

| 评价维度 | 快 Agent（D1） | 中 Agent（W1） | 慢 Agent（MN1） |
|---------|-------------|-------------|-------------|
| 判断后评估窗口 | 次日/3日 | 5日/10日 | 20日/60日 |
| 正确率要求 | 不作过高要求（55%+） | 60%+ | 70%+（低频高胜） |
| 储能长度 | 1-3天 | 3-20天 | 20-60天+ |
| 先验概率权重 | D1 State 历史分布 | W1+D1 联合分布 | MN1+W1 联合分布 |
| 后验校准周期 | 每日 | 每周 | 每月 |
| RR 视角 | D1 SR 支撑/阻力 | W1 SR 支撑/阻力 | MN1 SR 支撑/阻力 |
| 关键事件因子 | 周几、是否节前 | 月中末、是否财报季 | 季度会议、年度政策 |

### 5.4 Agent 间的协作模式

```
慢 Agent（MN1）——每个月说一次话
    │
    │  "本月方向向上，建议仓位 70-80%"
    │  "偏好行业：电子、建筑材料"
    ▼
中 Agent（W1）——每周五说一次话
    │
    │  "本周周线仍在收缩，但有蓄力迹象"
    │  "2560 策略本周适配度为适配"
    ▼
快 Agent（D1）——每个交易日工作
    │
    │  "今日 D1 已突破阻力 ±2%，资金流确认"
    │  "VCP 突破确认，建议入场"
    │  "但 MN1 Agent 提示仓位不超过 80%"
    ▼
  执行层
```

### 5.5 快慢 Agent 的分歧处理

当快 Agent 和慢 Agent 判断相反时：

| 快 Agent | 慢 Agent | 处理方式 |
|----------|---------|---------|
| 可做 | 不可做 | 降仓位、仅观察 |
| 不可做 | 可做 | 等待 D1 确认后再入场 |
| 可做 | 可做 | 全力执行（概率最高） |
| 不可做 | 不可做 | 完全防守 |

```
"快慢分歧度" = |快 Agent 评分 - 慢 Agent 评分|

分歧度低：Agent 一致，置信度高 → 按标准流程
分歧度高：Agent 冲突，置信度低 → 降低仓位，等待确认
```

## 八、Agent 迭代补充与纠正：因子自发现和权重自修正机制

### 核心理念

上面的因子和权重是**人预设的**——人定义了节气重要、月初重要、会议重要。但市场会变——某一年节气因子失效了、某个新出现的产业链事件比旧因子更有影响力、某个权重分配错了——人不可能每次都及时发现并手动调整。

Agent 必须有**自发现机制**来补充人遗漏的因子，和**自修正机制**来纠正人配错的权重。

### 8.1 Agent 因子自发现：异常检测驱动的特征挖掘

```
Agent 每天对比：先验预测 vs 实际结果

如果某天偏差 > 2σ（显著异常）：
    │
    ├── Agent 自动扫描关联特征
    │   ├── 今天是什么月份？是否是特殊月份？
    │   ├── 今天是周几？是否是特殊日历日？
    │   ├── 今天是否有事件？（会议/报告/公告）
    │   ├── 行业是否有异常？（某行业突然爆发/暴跌）
    │   ├── 宏观数据今天是否发布？
    │   └── 全市场 State 分布是否有异动？
    │
    ├── Agent 提出假设
    │   例："本周五是中报最后披露日+立秋节气+月末，三个因子叠加
    │        → 之前的模型对周五+月末权重偏低"
    │
    ├── Agent 回溯验证
    │   回查历史上类似因子组合的天数
    │   → 统计聚合偏差是否系统性地指向同一个方向
    │   → 如果是（> 70% 的历史类似日也有类似偏差）→ 标记为新因子候选
    │
    └── Agent 提案放入"候选因子池"
        ├── 状态: pending_review
        ├── 继续跟踪 20 个交易日
        ├── 如果 20 日内偏差持续 → 自动提升为正式因子
        └── 如果失效 → 丢弃
```

### 8.2 因子权重自修正

每个因子不是固定权重——Agent 持续跟踪每个因子的实际贡献度：

```
因子权重的自动校准周期 = 每月

每月末，Agent 回溯本月所有信号的预测偏差：
    │
    ├── 按因子分解偏差来源
    │   实际收益 = 基础先验 + 时间因子贡献 + 事件因子贡献 + 行业因子贡献 + 残差
    │
    ├── 对每个因子计算：实际边际贡献 vs 设定权重
    │   如果 |实际贡献 - 设定权重| > 阈值：
    │       → 该因子权重需要修正
    │
    ├── 权重修正规则
    │   新权重 = 旧权重 × 0.7 + 实际边际贡献 × 0.3
    │   （指数平滑，避免单月数据大幅翻转）
    │
    └── 修正后的权重写入 Agent 记忆，下月生效
```

**关键**：Agent 不能单日推翻权重——必须经过完整周期验证（月度是最小校准周期），避免被单日噪音带偏。

### 8.3 具体场景举例

**场景：节气因子权重过高**

```
立秋当日，Agent 按 8% 的节气权重修正了先验概率
→ 实际结果与先验偏差极小（仅 1.2%）
→ Agent 发现：本月节气因子贡献 < 设定权重的 30%
→ Agent 自动降低节气因子权重：8% → 6%
→ 下个月再看是否继续降
```

**场景：发现遗漏因子——"ETF期权交割日"**

```
某周三，实际 vs 预测偏差达 3.5σ
→ Agent 扫描当日特征：
  - 不是节气日 ✗
  - 没有重要会议 ✗
  - 不是月末 ✗
  - 是周三 ✓  → 但周三历史偏差仅 0.3σ，不够解释
  - 期权交割日 ✓ → 回查历史：过去 6 次交割日有 5 次出现 >2σ 偏差
→ Agent 自动提案：ETF期权交割日 → 新增为因子
→ weight = 0.05（初始权重，保守）
→ 继续跟踪确认
```

### 8.4 因子池生命周期

```
发现期（proposed）
    │  Agent 发现偏差 + 回溯确认
    ▼
验证期（validating）
    │  持续跟踪 20 个交易日
    │  统计正反例比
    │  计算平均边际贡献
    ▼
激活期（active）
    │  正式参与因子修正
    │  每月自动校准权重
    ▼
休眠期（dormant）
    │  连续 3 个月边际贡献 < 阈值
    │  权重自动降至 0
    ▼
归档（archived）
    Agent 不再使用，但保留历史记录
    如果市场环境变化，可以被重新发现
```

### 8.5 Agent 发现新因子后怎么办

```
Agent 发现新因子 → 不是直接加入计算 → 上报给人类
    │
    ├── 紧急（偏差 > 3σ，连续出现）→ 立即通知 + 自动降级信号置信度
    ├── 重要（单次 > 2σ + 回溯确认）→ 下次运行时提醒
    └── 常规（持续偏差但未触发阈值）→ 月末汇总时报告
```

### 8.6 多 Agent 交叉验证：防止单一 Agent 自欺

快速 Agent 和慢速 Agent 对同一个因子的判断应该独立验证：

```
快 Agent（D1）发现：节气因子在短线判断中贡献度高
慢 Agent（MN1）验证：同一个节气因子在长线判断中贡献度如何？

如果两者一致 → 因子稳健，可以同时用于两个视角
如果只有一方有效 → 该因子只对该速度的 Agent 生效，不对全局
如果双方都无效 → 该因子可以休眠
```

### 8.8 人设 vs Agent 自发现的边界

| 什么由人定义 | 什么由 Agent 自发现 |
|------------|-------------------|
| 因子的**类型框架**（时间/事件/行业） | 因子的**具体实例**（哪个季节、哪个会议） |
| 因子的**初始权重** | 因子的**实际权重**（持续修正） |
| 校准的**最小周期**（月） | 校准的**具体数值** |
| Agent 的**速度设定**（快/中/慢） | Agent 的**具体判断逻辑** |
| 异常检测的**阈值**（2σ/3σ） | 异常后的**具体归因** |
| 因子池的**生命周期规则** | 具体因子在生命周期中的**迁移** |

**一句话**：人做框架，Agent 做填充和修正。人的框架是"哪些类型的因子可能存在"，Agent 的使命是"在当前市场中，这些因子实际权重是多少，还有哪些人没想到的因子在起作用"。
