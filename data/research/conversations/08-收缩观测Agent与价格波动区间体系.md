# 收缩观测 Agent 与价格波动区间体系

日期：2026-06-01
主题：多周期视角下的价格波动区间收缩→突破质变机制，以及专职收缩观测 Agent 设计

---

## 一、核心概念

### 为什么需要一个专职的「收缩观测 Agent」

市场 State 运转中有一个核心现象：**价格波动区间从收缩到突破，是从量变到质变的关键窗口**。这个窗口的捕捉直接决定了交易策略的成败——VCP 收缩突破、布林 Squeeze 爆发、2560 回踩确认——本质上都是从"波动率收缩"到"趋势释放"的质变过程。

但当前的 System 中：

| 已有 | 缺失 |
|------|------|
| `compression` 三值分类（contracting/expanding/neutral） | **收缩的深度——不仅是"在收缩"，而是"收缩到什么程度"** |
| `bb_width_q20_20` 固定阈值 | **不同品种、不同周期的收缩阈值不同**——不能一刀切 |
| `contraction_duration` 计数 | **收缩期间的价格区间变化趋势**——是在持续收窄还是在横盘？ |
| SR 关键位距离 → `sr_distance_pct` | **多指标交叉验证**——BB 带宽 vs 枢轴带宽 vs ATR 比率，三者一起看 |

**一个专职的「收缩观测 Agent」就专门做这一件事：监测全市场每一只股票在不同周期上的波动区间，记录收缩的深度、速度、持续性，判断是否达到了"极值"状态，并在质变发生时触发策略衔接。**

## 二、观测指标体系

### 2.1 三个核心维度 + 多参数变体

| 维度 | 标准参数 | 为何需要多参数 |
|------|---------|-------------|
| **布林带宽** | 20 周期（已有） | 短窗口适合日线敏感度，长窗口适合 W1/MN1 判断 |
| **枢轴带宽** | SR 阻力距 - SR 支撑距 / 价格 | 分形 SR 代表的"市场共识区间"，比 BB 更稳定 |
| **ATR 比率** | ATR14 / 收盘价 × 100（已有） | 波动率的直接量化，与 BB 带宽互相校验 |

### 2.2 布林带宽——多参数跟踪

```
当前系统只有 bb_width_q20_20（20 周期 × 20bar 分位）。

需要补充的：
┌───────────────────┬──────────────────┬──────────────────────┐
│ 参数组合          │ 适用周期          │ 作用                 │
├───────────────────┼──────────────────┼──────────────────────┤
│ BB20（已有）       │ D1 日线判断       │ 短线收缩敏感性最高    │
│ BB50               │ W1 周线判断       │ 中期波动结构          │
│ BB100              │ MN1 月线判断      │ 长线波动结构          │
└───────────────────┴──────────────────┴──────────────────────┘

每种参数需要自己的统计分位：
  bb_width_q20_BB50   ← 50周期BB宽度的20%分位
  bb_width_q80_BB50   ← 50周期BB宽度的80%分位
  bb_width_q20_BB100  ← 100周期BB宽度的20%分位
  bb_width_q80_BB100  ← 100周期BB宽度的80%分位

当前收缩判断只用了 BB20——但 A 股的 W1 级别收缩需要 BB50 才能准确捕捉。
```

### 2.3 枢轴带宽——分形 SR 距离

```
枢轴带宽 = (sr_resistance - sr_support) / close

这个指标有三层含义：
  1. 它代表"市场共识的价格运行空间"
  2. 枢轴带宽缩小 → 多空力量趋于均衡 → 即将选择方向
  3. 枢轴带宽极大 → 价格在宽幅震荡，不适合做突破策略

三周期枢轴带宽同时缩小 = 最强收缩共振信号：
  MN1 枢轴带宽 < 历史 Q20
  + W1 枢轴带宽 < 历史 Q20
  + D1 枢轴带宽 < 历史 Q20
  → "三周期共识收缩"，突破后的质变力度最大
```

### 2.4 ATR 比率——波动率的直接度量

```
ATR 比率 = ATR14 / close × 100

与 BB 带宽的区别：
  - BB 带宽反映的是"价格离散度"（基于布林带）
  - ATR 比率反映的是"真实波动幅度"（基于每日振幅）

两者通常是同向的，但偶尔会背离：
  BB 带宽窄 + ATR 比率高 → 价格在窄通道内剧烈抖动，不是真收缩
  BB 带宽窄 + ATR 比率也低 → 真正的低波动蓄力状态

三指标共振检查：
  ✓ BB 带宽 < Q20
  ✓ 枢轴带宽 < Q20
  ✓ ATR 比率 < Q20
  → "完全收缩"——一旦突破，质变力度最大
```

## 三、收缩极致的统计阈值

### 3.1 痛点：不能一刀切

当前系统用固定阈值：
- `bb_width_q20_20` → BB 带宽 ≤ 历史 20% 分位 = 收缩

**问题：**
- 小盘成长股的 BB 带宽天然比大盘蓝筹股宽 3-5 倍
- 同一只股票在牛市和熊市的 BB 带宽基准完全不同
- D1 的 Q20 阈值对 W1 来说可能是常态

### 3.2 三层阈值：股票级 → 行业级 → 全市场级

```
阈值计算逻辑 = min(
    per_stock_percentile(近期60bar, Q20),    ← 第一优先级：股票自身
    per_industry_percentile(同行业, Q20),    ← 第二优先级：行业横向对比
    per_market_percentile(全市场, Q20)       ← 兜底：全市场基准
)
```

#### 股票级阈值（最精确）

对每只股票，每个周期视角，用其自身过去 60/120 bar 的分位数：

```
股票 688107 的 D1 BB20 带宽：
  过去 120bar 的 Q20 = 6.2%
  过去 120bar 的 Q10 = 4.8%（更严格的极端收缩阈值）

当日 BB20 = 5.5%
→ 5.5% < 6.2% → 处于收缩区
→ 但 5.5% > 4.8% → 未达到极端收缩
→ 收缩度 = (6.2 - 5.5) / (6.2 - 4.8) = 0.5 → 中等收缩
```

#### 行业级阈值（稳健校验）

同一行业所有股票的 BB 带宽分位：

```
电子行业（80只）D1 BB20 带宽 Q20 = 5.8%
688107 当日 BB20 = 5.5%
→ 5.5% < 5.8% → 在行业内也属于收缩——双重确认
```

#### 全市场级阈值（兜底）

当股票数据不足（次新股 < 120bar），用全市场统计做兜底。

### 3.3 收缩度分级

| 收缩度 | 条件 | 含义 | 策略含义 |
|--------|------|------|---------|
| **散逸** | BB > Q80 | 波动扩张，不适合做收缩类策略 | Bollinger Bandit 趋势延展期 |
| **正常** | Q50 < BB ≤ Q80 | 正常波动 | 不特别关注 |
| **蓄力** | Q20 < BB ≤ Q50 | 波动收敛中，蓄力阶段 | VCP 观察、2560 关注 |
| **收缩** | Q10 < BB ≤ Q20 | 明显收缩，接近突破前夜 | 策略信号准备 |
| **极致收缩** | BB ≤ Q10 | 极度收敛，随时可能突破 | 优先级最高，质变概率最大 |
| **窒息** | BB ≤ Q5 且 ≥ 20 bar | 长期极端收缩后的"弹簧压缩" | 突破后力度最大，RR 最高 |

## 四、突破确认：量变到质变的关键时刻

### 4.1 质变的六重确认

收缩后的突破不是一次性的——它需要多重确认来排除假突破：

```
收缩极致    突破触发        确认层
  │            │               │
  │  Q5-Q10    │  收盘突破SR    ├── 第一层：价格确认（收盘 > 阻力）
  │  收缩状态  │               │
  │            │               ├── 第二层：量能确认（VOL > 基准 × 1.5）
  │            │               │
  │            │               ├── 第三层：带宽确认（BB宽度从Q10跳升到 > Q50）
  │            │               │         ↑ 这是"质变"的核心信号
  │            │               │
  │            │               ├── 第四层：ADX确认（ADX从<25回升到>25 且斜率>0）
  │            │               │
  │            │               ├── 第五层：行业确认（同行业 ≥ 3只也在突破）
  │            │               │
  │            │               └── 第六层：资金流确认（主动净买 > 0）
  ▼            ▼               ▼
量变阶段      触发日        质变确认阶段
└─────────── 收缩蓄力 ──────────┘└───── 突破释放 ─────┘
```

### 4.2 各策略的质变窗口特征

| 策略 | 收缩特征 | 质变信号 | 质变后跟踪周期 |
|------|---------|---------|-------------|
| **VCP** | D1 BB20 ≤ Q10 + 枢轴带宽加速缩小 | BB宽度单日跳升 50%+ + 收盘突破 SR + 量能 A 级 | 3-5天确认真突破 |
| **2560** | W1 BB50 ≤ Q20 + D1 回踩 MA25 | 回踩后放量确认 + ADX 回升 | 5-10天趋势确认 |
| **Bollinger Bandit** | BB20 ≤ Q5 极端 Squeeze | 收盘突破上轨 + BB 带宽从 Q5 跳到 Q80 | 沿上轨行走期 |

### 4.3 质变失败：假突破的早期信号

收缩观测 Agent 不仅看突破，还要在突破后持续监控"质变是否失败"：

```
突破后第 3 天仍无明显 BB 带宽扩张 → 假突破概率 70%
突破当天量能 C 级 → 假突破概率 60%
突破后立即回到 SR 区间内 → 假突破确认
同行业无其他股票同步突破 → 孤立信号，降权
```

## 五、收缩观测 Agent 设计

### 5.1 专职角色

| 属性 | 内容 |
|------|------|
| **Agent 名称** | ContractionObserver（收缩观测者） |
| **职责** | 专职监控全市场每一只股票在三周期视角下的波动区间收缩状态 |
| **输入** | timeframe_indicators（BB/ATR/ADX）+ SR数据 + 统一视图 |
| **输出** | 收缩度评分 + 质变触发信号 + 假突破预警 |
| **运行频率** | 每日（D1视角）+ 每周五（W1视角）+ 月末（MN1视角） |

### 5.2 与其他 Agent 的协作

```
ContractionObserver              其他 Agent
      │                              │
      │  发现"极致收缩"状态           │
      ├────────────────────────────→ StrategyAdvisor
      │   "000636.SZ D1达到窒息收缩   │   → 优先分配给VCP/Bollinger检查
      │    BB20=3.2%  Q5阈值=3.5%"    │
      │                              │
      │  发现"质变突破"               │
      ├────────────────────────────→ 【全系统】
      │   "600483.SH 突破确认          │   → 该信号优先级+2
      │    6重确认通过/6"             │   → risk_guardian计算仓位
      │                              │
      │  发现"假突破"                 │
      ├────────────────────────────→ 【全系统】
      │   "002281.SZ 突破3日后         │   → 该信号降为观察
      │    BB带宽未扩张"              │   → 暂停开仓推荐
      │                              │
      │  行业级警报                   │
      ├────────────────────────────→ MarketAnalyst
      │   "电子行业80只中               │
      │    45只处于收缩蓄力→可能批量突破"│
```

### 5.3 与策略的深度衔接

ContractionObserver 不直接推荐买卖——它输出的是"波动状态"，策略根据波动状态决定行为：

```
ContractionObserver 输出          策略的动作

"窒息收缩 + D1 + VCP结构"    →  VCP策略：重点监控，等突破确认
"BB带宽从Q5跳到Q80，量能A"    →  Bollinger Bandit：入场信号+1
"收缩但尚未突破"              →  2560策略：关注回踩MA25质量
"枢轴带宽在三周期同时缩小"     →  全局警报：即将有方向性行情
"突破后BB带宽持续扩张"         →  持仓策略：沿上轨继续持有
"BB带宽从扩张重回收缩"         →  止盈信号：波动率回归
```

### 5.4 收缩状态记忆

与之前讨论的 Agent 记忆机制一致，ContractionObserver 需要自己的记忆：

```
每只股票的收缩历史：
  ├── 历史上的极致收缩次数
  ├── 每次收缩的持续时间
  ├── 收缩后突破的胜率（真突破 vs 假突破）
  ├── 该股票在不同季节的收缩-释放模式
  └── 该股票历史上的"假突破频率"

这些记忆用于后验校准：
  "688107 过去3次BB20 ≤ Q10的收缩后，
   2次真突破平均+18%，1次假突破-6%
   → 当前第4次收缩，历史成功率 67%，偏积极"
```

## 七、场景举例

### 场景 1：三周期同时极致收缩 → 大行情前夜

```
2026-06-10 收盘后，ContractionObserver 输出：

股票：000636.SZ 风华高科
行业：电子 > 被动元件

收缩状态：
  MN1 BB50带宽 = 4.5%（股票历史 Q10=5.0%）  → 超越极致收缩！
  W1  BB50带宽 = 3.2%（行业 Q10=4.0%）      → 超越极致收缩！
  D1  BB20带宽 = 2.8%（行业 Q5=3.0%）       → 窒息级收缩！

  MN1枢轴带宽 = 8.2%（Q5=10.0%）            → 月线级极度压缩
  W1 枢轴带宽 = 5.1%（Q10=6.5%）            → 周线级极度压缩
  D1 枢轴带宽 = 4.3%（Q5=5.0%）             → 日线级极度压缩

综合判定：⭐⭐⭐⭐⭐ 五颗星（历史类似情况出现12次，其中10次在接下来
         20日内产生显著行情，平均幅度 +22%，仅2次假突破）

→ 通知 StrategyAdvisor：该票进入最高监控级别
→ 通知 risk_guardian：突破后建议仓位可提高至上限
```

### 场景 2：突破确认的六重验证

```
2026-06-12 收盘后，ContractionObserver 输出：

股票：603663.SH 三祥新材
状态：昨日触发了突破（收盘 > W1 SR阻力）

突破确认检查：
  ✓ 第一层（价格）：收盘 59.53，突破 W1 SR 阻力 55.00 → 确认
  ✓ 第二层（量能）：VOL = 1.8 × 20日均量 → A级量能 → 确认
  ✓ 第三层（带宽）：BB20 从昨日的 3.1% 跳升到今日 8.5%
                    跳升幅度 174% → Q10→Q70 → 质变确认！
  ✓ 第四层（ADX）：ADX 从 22 回升到 27，斜率 +1.7 → 趋势启动
  ✗ 第五层（行业）：同行业仅 1 只在突破 → 孤立信号
  ✓ 第六层（资金流）：主动净买 6.6 亿，资金流确认为"流入" → 确认

6重确认中5/6通过。唯一未通过的是行业共振→不是致命缺陷，
但降一级置信度。总体判定：真突破概率 85%。
```

### 场景 3：假突破早期预警

   → 该信号从"执行队列"移入"观察队列"

##八、结论

收缩观测 Agent 是连接"多周期 State 运转底座"与"交易策略执行"之间的关键桥接层。它的核心价值是：

1. **把静态的 State 代码变成动态的波动状态描述**
2. **把固定的阈值变成每只股票自己的统计阈值**
3. **把孤立的指标变成交叉验证的决策链条**
4. **把一次性的突破判断变成持续的质变跟踪**
